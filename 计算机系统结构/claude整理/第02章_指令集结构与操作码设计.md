# 第02章 · 指令集结构与操作码设计

> **考试重点**：操作码设计（定长/Huffman/扩展编码）、指令字格式设计、带标志位数据表示、CISC/RISC对比
> **题型**：选择、填空、**大题**（考纲大题#2、#3）——设计+计算
> **建议学习时间**：3-4小时

---

## 2.1 指令集结构的分类

### 2.1.1 按操作数存储位置分类

| 类型 | 操作数来源 | 特点 | 实例 |
|------|-----------|------|------|
| **堆栈型** | 栈顶（隐含） | 指令最短，但效率低 | Java虚拟机 |
| **累加器型** | 累加器 + 内存 | 一条指令一个操作数隐含 | 早期计算机 |
| **通用寄存器型** | 寄存器 / 内存 | **现代主流** | MIPS, x86 |

### 2.1.2 通用寄存器型的三种子类

| 子类型 | ALU操作数 | 说明 | 指令长度 | 代码量 | 实例 |
|--------|----------|------|---------|--------|------|
| **RR（Load-Store）** | 全在寄存器 | 仅load/store访存 | 固定 | 较多 | MIPS, SPARC, ARM |
| **RM（Register-Memory）** | 一个在内存 | 允许ALU操作直接访存 | 可变 | 少 | x86, 68000 |
| **MM（Memory-Memory）** | 全在内存 | 最灵活但最慢 | 可变 | 最少 | VAX（已淘汰） |

### 2.1.3 运算 C = A + B 在四种结构中的实现

| 堆栈型 | 累加器型 | RM型 | RR型（Load-Store） |
|--------|---------|------|-------------------|
| PUSH A | LOAD A | LOAD R1, A | LOAD R1, A |
| PUSH B | ADD B | ADD R1, B | LOAD R2, B |
| ADD | STORE C | STORE R1, C | ADD R3, R1, R2 |
| POP C | | | STORE R3, C |

## 2.2 CISC 与 RISC

### 2.2.1 CISC（复杂指令集计算机）

**设计目标**：增强指令功能，减少程序指令数。

**存在的问题**：
1. 指令使用频率差异极大（20%指令占80%执行时间）
2. 指令集庞大复杂
3. 指令执行速度慢
4. 规整性差，不利于流水线（变长指令、复杂寻址方式）

### 2.2.2 RISC（精简指令集计算机）

**设计原则**：

| 原则 | 说明 |
|------|------|
| 1. 指令条数少而简单 | 仅包含最高频、最有用的指令 |
| 2. 简单统一的指令格式 | 32或64位定长，寻址方式少 |
| 3. 单周期执行 | 依靠流水线实现 |
| 4. **仅load和store指令访问存储器** | RR型 / Load-Store架构 |
| 5. 大多数指令用**硬连逻辑**实现 | 不使用微程序 |
| 6. 优化编译器 | 发挥流水线效率 |
| 7. 充分利用流水线 | 规整指令格式便于流水 |

### 2.2.3 对比表

| 特征 | CISC | RISC |
|------|------|------|
| 指令数量 | 多（数百条） | 少（几十条） |
| 指令格式 | 可变长度 | 定长（如32位） |
| 寻址方式 | 多种 | 少数 |
| 访存方式 | 多种指令可访存 | 仅Load/Store访存 |
| 实现方式 | 微程序控制 | 硬连逻辑控制 |
| 流水线 | 实现困难 | 天然适应 |
| CPI | 高（~10） | 低（~1.2） |
| 代码密度 | 高 | 较低 |

## 2.3 操作码设计（考纲大题#2——三种方法）

### 2.3.1 方法一：定长操作码

所有指令的操作码长度相同。

$$操作码位数 = \lceil \log_2 N \rceil$$

其中 $N$ 为指令总数。

**优点**：译码简单，规整性好
**缺点**：信息冗余度高，编码效率低

---

### 2.3.2 方法二：Huffman编码

**原理**：使用频率高的指令用短操作码，频率低的用长操作码。

**操作码平均长度**：

$$l = \sum_{i=1}^{n} p_i \cdot l_i$$

其中 $p_i$ 为指令 $i$ 的使用频率，$l_i$ 为其操作码长度。

**信息熵（理论最短平均码长）**：

$$H = -\sum_{i=1}^{n} p_i \cdot \log_2 p_i$$

**编码效率**：

$$\text{信息冗余度} = 1 - \frac{H}{l}$$

**Huffman编码步骤**：
1. 将指令按频率从大到小排列
2. 每次合并最小的两个概率
3. 给合并后的两条路径分别赋值0和1
4. 从根到叶的路径即为编码

**例题**：某模型机有7条指令，使用频率如下。设计Huffman编码，并求平均码长和信息冗余度。

| 指令 | I1 | I2 | I3 | I4 | I5 | I6 | I7 |
|------|-----|-----|-----|----|----|----|----|
| 频率 | 0.45 | 0.30 | 0.15 | 0.05 | 0.03 | 0.01 | 0.01 |

**解**：

构造Huffman树：

```
步骤1: 合并0.01+0.01=0.02
步骤2: 合并0.02+0.03=0.05
步骤3: 合并0.05+0.05=0.10
步骤4: 合并0.10+0.15=0.25
步骤5: 合并0.25+0.30=0.55
步骤6: 合并0.55+0.45=1.00
```

得到编码（取决于每次给0还是1的约定）：

| 指令 | 频率 | Huffman码 | 长度 |
|------|------|-----------|------|
| I1 | 0.45 | 0 | 1 |
| I2 | 0.30 | 10 | 2 |
| I3 | 0.15 | 110 | 3 |
| I4 | 0.05 | 1110 | 4 |
| I5 | 0.03 | 11110 | 5 |
| I6 | 0.01 | 111110 | 6 |
| I7 | 0.01 | 111111 | 6 |

$l = 0.45 \times 1 + 0.30 \times 2 + 0.15 \times 3 + 0.05 \times 4 + 0.03 \times 5 + 0.01 \times 6 + 0.01 \times 6$
$= 0.45 + 0.60 + 0.45 + 0.20 + 0.15 + 0.06 + 0.06 = 1.97$

$H = -(0.45\log_2 0.45 + 0.30\log_2 0.30 + 0.15\log_2 0.15 + 0.05\log_2 0.05 + 0.03\log_2 0.03 + 2 \times 0.01\log_2 0.01)$
$\approx -(0.45\times 1.152 + 0.30\times 1.737 + 0.15\times 2.737 + 0.05\times 4.322 + 0.03\times 5.059 + 0.02\times 6.644)$
$\approx 1.95$

信息冗余度 = $1 - 1.95/1.97 \approx 1\%$

**Huffman编码的问题**：
- 操作码长度差异大（1~6位），不规整
- 译码困难（不知道每条指令操作码是几位）
- 实际中很少直接使用，多采用扩展编码

---

### 2.3.3 方法三：扩展编码法（考试重点！）

**基本思想**：将操作码分为几段等长部分，用某个特殊码作为扩展标志。

**等长扩展**：如 4-8-12（操作码分别为4位、8位、12位）

**4-8-12等长扩展**（以16位指令字长为例）：

| 格式 | 操作码 | 说明 |
|------|--------|------|
| 格式1 | 4位操作码 | 可编码 15 条指令（留一个全1码做扩展标志） |
| 格式2 | 8位操作码（4位全1 + 4位扩展） | 可编码 15 条指令 |
| 格式3 | 12位操作码 | 可编码 16 条指令 |

**4-8-12 15/15/15 编码方案**：

| 操作码位数 | 编码范围 | 指令条数 |
|-----------|---------|---------|
| 4位 | 0000 ~ 1110 | 15条 |
| 8位 | 1111 0000 ~ 1111 1110 | 15条 |
| 12位 | 1111 1111 0000 ~ 1111 1111 1111 | 16条 |

**不等长扩展**：如 4-6-10

| 方案 | 说明 |
|------|------|
| 4-6-10 (15/3/8) | 4位：15条 + 1个扩展码；6位：3条(2位) + 1个扩展码；10位：8条(4位) 或更多 |

**经典例题（历年真题）**：

某模型机共有9条指令，使用频率如下。要求用扩展编码法，最多两种码长，设计最优编码。计算平均码长。

| 指令 | ADD | SUB | LOD | STO | JMP | SHR | ROL | MOV | STP |
|------|-----|-----|-----|-----|-----|-----|-----|-----|-----|
| 频率 | 30% | 24% | 6% | 7% | 7% | 2% | 3% | 20% | 1% |

**解**：

两种码长 → 一种短码、一种长码。频率最高的几条用短码（如4位），其余用长码（如8位）。

4位可以表示16个编码。留1个做扩展标志 → 可以用15个4位码。

将频率从大到小排列：ADD(30%), SUB(24%), MOV(20%), STO(7%), JMP(7%), LOD(6%), ROL(3%), SHR(2%), STP(1%)

设用短码（如4位）的前k条指令：
- 若k=3（ADD,SUB,MOV）：覆盖74%，剩余26%用8位
- 若k=4（加STO）：覆盖81%

取k=3（ADD、SUB、MOV用短码）：
- ADD: 0000(4位), SUB: 0001(4位), MOV: 0010(4位)
- 其余6条用扩展：1111 0000 ~ 1111 0101 (8位)

平均码长 = $0.30 \times 4 + 0.24 \times 4 + 0.20 \times 4 + 0.07 \times 8 + 0.07 \times 8 + 0.06 \times 8 + 0.03 \times 8 + 0.02 \times 8 + 0.01 \times 8$
$= 1.20 + 0.96 + 0.80 + 0.56 + 0.56 + 0.48 + 0.24 + 0.16 + 0.08 = 5.04$ 位

（注意实际设计时还需要考虑操作数地址码的位数约束，这将在指令字格式设计中处理。）

## 2.4 指令字格式设计（考纲大题#2第二部分）

### 2.4.1 指令字的基本组成

$$\text{指令字} = \text{操作码} + \text{地址码(多个)}$$

### 2.4.2 地址码个数的影响

地址码个数 → 影响每个地址码的位数（在总指令字长固定的情况下）

| 地址个数 | 0地址 | 1地址 | 2地址 | 3地址 |
|---------|-------|-------|-------|-------|
| 需要的地址位 | 无 | 1× | 2× | 3× |
| 每条地址码可分配的位数 | 最多 | 多 | 少 | 最少 |

### 2.4.3 指令格式设计例题

**例题**：某机指令字长16位，有8个通用寄存器（需3位表示），主存64KB（按字节编址，需16位）。要求设计支持以下类型的指令格式：

1. RR型（寄存器-寄存器）：2地址
2. RM型（寄存器-存储器）：2地址
3. 无地址指令

**解**：

主存地址需要16位（64KB）。
寄存器号需要3位（8个通用寄存器）。

(1) **RR型指令**：操作码 + 2个寄存器号
- 16 - 3 - 3 = 10位操作码 → 最多1024条RR指令

(2) **RM型指令**：操作码 + 1个寄存器号 + 1个主存地址
- 16 - 3 - 16 → 不够！需要变通

解决方案：使用**扩展操作码**或减少主存地址位（如使用偏移量寻址，仅需少量偏移位）。

(3) **无地址指令**：16位全是操作码 → 最多65536条

实际设计需要考虑各种指令数量的需求来平衡。

## 2.5 带标志位的数据表示（考纲大题#3）

### 2.5.1 两种数据表示方式

**不带标志位**：
- 每个数据字就是纯数据
- 数据的类型由访问该数据的**指令操作码**决定
- 存储空间利用率高

**带标志位**：
- 每个数据字除了数据部分外，还有标识该数据类型的**标志位**
- 硬件可以直接判断数据类型
- 简化指令集（不需要区分定点加法和浮点加法）
- 但增加了存储开销

### 2.5.2 容量比较计算

**例**：某程序需要存储以下数据：
- 1000个32位定点数
- 500个64位浮点数
- 200个8位字符

比较带标志位和不带标志位的存储容量。

**解**：

不带标志位：程序按数据类型分别存储在不同存储区，总容量 = $1000 \times 32 + 500 \times 64 + 200 \times 8 = 32000 + 32000 + 1600 = 65600$位

带标志位：每个数据字需要额外的标志位（如3位标志位可区分8种数据类型）
- 32位定点数：3位标志 + 32位数据 = 35位 × 1000 = 35000
- 64位浮点数：3位标志 + 64位数据 = 67位 × 500 = 33500
- 8位字符：3位标志 + 8位数据 = 11位 × 200 = 2200
- 总容量 = $35000 + 33500 + 2200 = 70700$位

标志位额外开销 = $(70700 - 65600) / 65600 \approx 7.8\%$

### 2.5.3 数据表示 vs 数据结构

| | 数据表示 | 数据结构 |
|---|---|---|
| **谁处理** | 硬件 | 软件 |
| **定义** | 硬件能直接识别和处理的数据类型 | 软件处理的数据组织方式 |
| **例子** | 定点数、浮点数、字符 | 链表、树、队列、图 |
| **关系** | 数据表示是数据结构的基础 | 数据结构需通过软件映射到数据表示 |

## 2.6 MIPS指令集（了解级别）

> 考纲要求：2.6节能读懂后续章节中的程序即可。

**32个64位通用寄存器**：R0~R31，R0恒为0

**寻址方式**：仅立即数寻址和偏移量寻址两种

**三种指令格式（均为32位）**：

| 格式 | 结构 | 用途 |
|------|------|------|
| **I型** | OP(6) + rs(5) + rt(5) + immediate(16) | Load/Store/立即数/分支 |
| **R型** | OP(6) + rs(5) + rt(5) + rd(5) + shamt(5) + funct(6) | 寄存器间ALU操作 |
| **J型** | OP(6) + offset(26) | 跳转 |

## 2.7 课后思考题参考答案要点

1. **三类指令集结构**：堆栈型（操作数隐含在栈顶）、累加器型（一个操作数隐含）、通用寄存器型（操作数在寄存器或内存）
2. **RR与RM区别**：RR仅load/store访存，RM允许ALU操作直接访存；RR代码量大但规整，RM代码量小但不规整
3. **Load-Store结构**：所有ALU操作数必须在寄存器中，只有load和store指令可以访存
4. **CISC vs RISC**：设计出发点不同，CISC追求功能强、代码密，RISC追求规整、高效流水
5. **操作码三种设计方法**：定长（简单低效）、Huffman（最优但不规整）、扩展（折中实用）
6. **指令格式设计制约**：指令总位数 = 操作码位 + 各地址码位
