## A.2 常用互连函数（$N=2^n$个结点）

设输入/输出编号用 $n$ 位二进制表示：$x = x_{n-1}x_{n-2}...x_1x_0$

### A.2.1 交换置换（Exchange / Cube）

$$Exchange_k(x_{n-1}...x_k...x_0) = x_{n-1}...\overline{x_k}...x_0$$

即将第 $k$ 位取反。

**星形网络**：用Cube函数组合可实现任意两个处理器之间的通信。

### A.2.2 混洗置换（Perfect Shuffle）

$$Shuffle(x_{n-1}x_{n-2}...x_1x_0) = x_{n-2}...x_1x_0x_{n-1}$$

即**循环左移1位**。

**反混洗**（Inverse Shuffle）：循环右移1位。
$$Shuffle^{-1}(x_{n-1}x_{n-2}...x_1x_0) = x_0x_{n-1}...x_2x_1$$

### A.2.3 蝶式置换（Butterfly）

$$Butterfly(x_{n-1}x_{n-2}...x_1x_0) = x_0x_{n-2}...x_1x_{n-1}$$

即将**最高位与最低位交换**。

### A.2.4 PM2I函数

$$PM2_{+i}(x) = x + 2^i \pmod{N}$$
$$PM2_{-i}(x) = x - 2^i \pmod{N}$$

其中 $0 \leq i \leq n-1$。

### A.2.5 例题

8个结点（$n=3$，编号 0~7 用3位二进制表示）：

**Q**：结点6(110)经过混洗置换后输出到哪个结点？

$$Shuffle(110) = 101 = 5$$

**Q**：结点6(110)经过Cube2置换后输出到哪个结点？

$$Cube_2(110) = 010 = 2$$

## A.3 多级互连网络（MIN）设计（考纲大题#11）

### A.3.1 Omega网络

**结构**：$n = \log_2 N$ 级，每级 $N/2$ 个 $2 \times 2$ 开关。

**连接规律**：级间用**混洗连接**

**路由方法**：用目的地址的各位从高到低（或从低到高）逐级控制开关。

例如8×8 Omega网络（3级，每级4个开关）：

```
Stage 0          Stage 1          Stage 2
输入 ──[S0]──   ──[S0]──   ──[S0]── 输出0
  0  ──[  ]──   ──[  ]──   ──[  ]── 1
  1  ──[S1]──   ──[S1]──   ──[S1]── 2
  2  ──[  ]──   ──[  ]──   ──[  ]── 3
  3  ──[S2]──   ──[S2]──   ──[S2]── 4
  4  ──[  ]──   ──[  ]──   ──[  ]── 5
  5  ──[S3]──   ──[S3]──   ──[S3]── 6
  6  ──[  ]──   ──[  ]──   ──[  ]── 7
  7
```

每个 $2 \times 2$ 开关有两种状态：
- **直通（Straight）**：上入→上出，下入→下出
- **交换（Exchange）**：上入→下出，下入→上出

**例题**：8×8 Omega网络，要求输入5(101)连接到输出0(000)。

路由方法：目的地址为000。

- Stage 0：目的地址最高位 = 0 → 直通（上出）
- Stage 1：目的地址次高位 = 0 → 直通（上出）
- Stage 2：目的地址最低位 = 0 → 直通（上出）

检查是否阻塞：同时传输(5,0)和(6,3)。

输入5(101)→000：Stage0→直通→Stage1→直通→Stage2→直通→0 ✓
输入6(110)→011(3)：Stage0→直通→Stage1→直通→Stage2→交换→3 ✓

这两条路径不冲突（经过不同的开关）。

但如果传输(5,0)和(1,4)：
输入5(101)→000：Stage0上出
输入1(001)→100(4)：Stage0上出

两条路径在Stage 0的同一个开关上都要走上出 → **阻塞！**

### A.3.2 Omega网络的特点

- **阻塞网络**：不是所有置换都能同时实现
- 开关结构简单（均为 $2 \times 2$）
- 单一路径（自路由——目的地址确定唯一路径）

## A.4 阻塞网络与非阻塞网络（考纲大题#12）

### A.4.1 阻塞网络

**阻塞（Blocking）**：两个同时进行的通信请求试图使用同一物理链路或开关端口时发生。

**Omega网络是阻塞网络**的原因：不同的置换可能共享中间的链路或开关。

**阻塞成因分析**：
- 网络无法同时实现所有可能的无冲突置换
- 中间链路资源的竞争

### A.4.2 非阻塞网络

**严格无阻塞（Strict-sense Non-blocking）**：无论网络中已有何种连接，总能在任意空闲的输入和输出之间建立连接。

**可重排无阻塞（Rearrangeable Non-blocking）**：通过重新安排已有连接，总可以实现任何置换。**Benes网络**属于此类。

### A.4.3 非阻塞排列网络

一个N×N的**排列网络（Permutation Network）**能实现所有N!种置换中至少一部分。

**Clos网络**：
- 三级Clos网络可以用较少的交叉点实现非阻塞
- 条件：中间级开关数 $m \geq 2n-1$（其中 $n$ 是第一级每个开关的输入数）

## A.5 Benes网络（考纲大题#13——画出拓扑）

### A.5.1 Benes网络结构

**Benes网络**是可重排无阻塞网络，由 $2\log_2 N - 1$ 级构成。

**递归构造**：
- $N \times N$ Benes网络 = 输入级（$N/2$个 $2 \times 2$ 开关）+ 两个 $N/2 \times N/2$ Benes子网络 + 输出级（$N/2$个 $2 \times 2$ 开关）

**8×8 Benes网络拓扑**（$2 \times 3 - 1 = 5$ 级）：

```
Stage 0    Stage 1    Stage 2    Stage 3    Stage 4
输入 ──[S0]────────────[S0]────────────[S0]── 输出0
0  ──[  ]──┐      ┌──[  ]──┐      ┌──[  ]── 1
           │      │        │      │
1  ──[S1]──┤  ┌──[S1]──┐  ├──[S1]──┤  ┌── 2
   ──[  ]──┘  │  [  ]──┘  │  [  ]──┘  │
              │            │           │
2  ──[S2]────┼──[S2]──────┼──[S2]─────┼── 3
   ──[  ]──┐ │  [  ]──┐   │  [  ]──┐  │
           │ │        │   │        │  │
3  ──[S3]──┘ ├──[S3]──┤   ├──[S3]──┤  ├── 4
              │  [  ]──┘   │  [  ]──┘  │
4  ──[S4]────┼─────────────┼──[S4]─────┼── 5
   ──[  ]──┐ │             │  [  ]──┐  │
           │ │             │        │  │
5  ──[S5]──┤ ├─────────────┤  [S5]──┤  ├── 6
           │ │             │        │  │
6  ──[S6]──┘ ├─────────────┤  [S6]──┤  ├── 7
              │             │        │  │
7  ──[S7]────┘             └──[S7]──┘  └──
```

### A.5.2 Benes网络特点

- $2\log_2 N - 1$ 级
- **可重排无阻塞**：任意置换都可通过合适的开关设置实现
- 开关总数：$(N/2) \times (2\log_2 N - 1)$
- 与Omega网络相比：更多级数、更复杂路由、但无阻塞

## A.6 Delta网络 — 开关数量减少

**Delta网络**：任何输入到输出的路径经过相同的开关级数，且自路由。

**开关节省**：
- N×N交叉开关（Crossbar）：需要 $N^2$ 个交叉点
- N×N Delta网络：需要 $\frac{N}{2} \times \log_2 N$ 个 $2 \times 2$ 开关

**例**：64×64网络
- Crossbar：$64^2 = 4096$ 个交叉点
- Delta：$(64/2) \times \log_2 64 = 32 \times 6 = 192$ 个 $2 \times 2$ 开关
- 每个 $2 \times 2$ 开关 ≈ 4个交叉点 → 约 $768$ 个交叉点

节省了约 $81\%$。

## A.7 间接二进制n方体网络

### A.7.1 结构特点

- 等同于Omega网络的逆网络（或等效于蝶形网络的某种排列）

**特点**：

- $n = \log_2 N$ 级
- 每级 $N/2$ 个 $2 \times 2$ 开关

## A.8 消息路由

### A.8.1 常见路由方式

| 方式 | 说明 |
|------|------|
| **存储转发** (Store-and-Forward) | 整条消息接收完再转发 |
| **虫蚀路由** (Wormhole Routing) | 消息分成微片(flit)，流水转发 ← **重点** |
| **虚拟直通** (Virtual Cut-Through) | 缓冲足够时直接转发 |

### A.8.2 E-cube路由（超立方体网络重点）

**E-cube（Dimension-Ordered Routing）**：在n维超立方体中，沿维度从低到高依次修正地址。

**算法**：从源地址到目的地址
1. 比较源地址和目的地址，找出所有不同的位
2. 每次修正最低的1位不同位
3. 直到到达目的地址

**例**：4维超立方体，源S(1011)，目的D(0100)

S = 1011
D = 0100

不同位：第0位(S=1,D=0)、第1位(S=1,D=0)、第3位(S=1,D=0)

按修正最低不同位的顺序：
1. 1011 → 1010（修正第0位）
2. 1010 → 1000（修正第1位）
3. 1000 → 0000（修正第3位）
4. 0000 → 0100（修正第2位...等等，目的第2位是1）

重新计算：
S = 1011, D = 0100
不同位：第0位(1→0), 第1位(1→0), 第2位(0→1), 第3位(1→0)

按维序（从低到高依次修正）
1. 修正第0位：1011 → **1010**
2. 修正第1位：1010 → **1000**
3. 修正第2位：1000 → **1100**
4. 修正第3位：1100 → **0100**

最短路径经过的中间结点：1010, 1000, 1100, 0100

## 模拟大题 1：CPU性能公式 + 方案比较（10分）

**题目**：某计算机指令混合如下。时钟频率2GHz。

| 指令类型 | 占比 | CPI |
|---------|------|-----|
| 整数ALU | 40% | 1 |
| Load | 25% | 2 |
| Store | 15% | 2 |
| 分支 | 15% | 2 |
| 浮点运算 | 5% | 4 |

方案A：保持原设计不变
方案B：改进浮点单元，将浮点运算CPI从4降为2，但时钟频率降为1.8GHz
方案C：改进分支预测，将分支CPI从2降为1，时钟频率不变

问：（1）计算原方案的CPI和MIPS
（2）哪个方案最优？计算加速比证明。

<details><summary>答案与解析</summary>

（1）CPIA = 0.40×1 + 0.25×2 + 0.15×2 + 0.15×2 + 0.05×4
= 0.40 + 0.50 + 0.30 + 0.30 + 0.20 = 1.70

MIPSA = 2GHz / 1.70 = 2000/1.70 ≈ 1176 MIPS

（2）方案B：CPIB = 0.40×1 + 0.25×2 + 0.15×2 + 0.15×2 + 0.05×2 = 1.60
时钟频率 = 1.8GHz
CPUTimeB = IC × 1.60 / 1.8GHz = IC × 0.889 ns

方案C：CPIC = 0.40×1 + 0.25×2 + 0.15×2 + 0.15×1 + 0.05×4 = 1.55
时钟频率 = 2GHz
CPUTimeC = IC × 1.55 / 2GHz = IC × 0.775 ns

方案A：CPUTimeA = IC × 1.70 / 2GHz = IC × 0.85 ns

加速比：
B vs A = 0.85/0.889 = 0.956（B更慢！）
C vs A = 0.85/0.775 = 1.097（C更快约9.7%）

**方案C最优**。方案B虽然CPI降低了，但降频抵消了收益。

</details>

## 模拟大题 3：Cache AMAT + 两级Cache（12分）

**题目**：某系统有L1和L2两级Cache。
- L1：命中时间=1周期，不命中率=5%
- L2：命中时间=12周期，局部不命中率=40%
- L2不命中开销=120周期
- 基准CPI（全命中）=1.5，每条指令平均访存1.2次

（1）求平均访存时间AMAT
（2）求全局不命中率
（3）求包含Cache不命中的实际CPI

<details><summary>答案与解析</summary>

（1）AMAT = 1 + 5% × (12 + 40% × 120)
= 1 + 0.05 × (12 + 48)
= 1 + 0.05 × 60
= 1 + 3 = **4周期**

（2）全局不命中率 = L1不命中率 × L2局部不命中率
= 5% × 40% = **2%**

（3）MemoryStallCycles = IC × 1.2 × 5% × (12 + 40% × 120)
= IC × 1.2 × 0.05 × 60
= IC × 3.6

实际CPI = 1.5 + 3.6 = **5.1**

</details>

周期4：流出SUB.D → ADD2, Qj=Qi[F4]=MULT1, Qk=Qi[F10]=0, Vk=F10值, Qi[F0]=ADD2（覆盖了之前的Load1值，实现了WAW消除）

ADD1执行完成(周期5)：CDB广播F2新值，所有等待F2的保留站更新

## 模拟大题 5：互连网络设计（12分）

**题目**：有8个处理器编号0~7，需要实现的通信配对为：(5,0), (1,4), (6,3), (2,7)。

（1）用8×8 Omega网络实现上述配对。画出网络拓扑，标出各开关状态。
（2）判断是否存在阻塞。
（3）如果用Benes网络，请说明优势。

<details><summary>答案与解析</summary>

（1）8×8 Omega网络 = 3级，每级4个开关。

配对分析（输入→输出）：
- 5(101) → 0(000)：使用目的地址000路由
  - S0(第0级第2个开关,上端口为5,下端口为4)：最高位=0→走上出→S1上端口
  - S1：次高位=0→走上出→S2上端口
  - S2：最低位=0→走上出→输出0
- 1(001) → 4(100)：路由检查
  - S0(第0级第0个开关,上端口为0,下端口为1)：最高位=1→走下出
  - S1：次高位=0→走上出
  - S2：最低位=0→走上出→输出4
- 6(110) → 3(011)
- 2(010) → 7(111)

检查阻塞：如果5→0和1→4在S0经过不同的物理路径 → 不阻塞。

## 模拟大题 6：循环展开 + 指令调度（10分）

**题目**：对于循环 `for(i=100; i>0; i--) x[i] = x[i] + s;`

MIPS代码（单次迭代）：
```asm
L.D    F0, 0(R1)       ; Load x[i]
ADD.D  F4, F0, F2      ; x[i] + s (F2 = s)
S.D    F4, 0(R1)       ; Store
DADDIU R1, R1, #-8     ; i--
BNE    R1, R2, Loop    ; if i>0 goto Loop
```

延迟表：FP计算 → 另一FP计算：3周期；FP计算 → Store：2周期；Load → FP计算：1周期。

（1）不调度时单次迭代需要多少个周期？
（2）展开2次（处理2个元素），调度后需要多少周期/元素？

<details><summary>答案与解析</summary>

（1）不调度：
```
L.D F0, 0(R1)     ; 周期1, IF
                  ; 周期2, ID
                  ; 周期3, EX
                  ; 周期4, MEM ← F0可用
ADD.D F4, F0, F2  ; 周期5-6, EX(2周期) + 等待F0: stall 1
                  ; 周期5 stall (Load-use)
                  ; 周期6-7 EX
                  ; 周期8 MEM+WB
S.D F4, 0(R1)     ; 周期9-10 + stall: 等待F4 2周期
```
约需：L.D(1) + stall(1) + ADD.D(2) + stall(2) + S.D(1) = 约10周期/元素

（2）展开2次 + 调度：

展开后重命名寄存器（迭代1用F0,F4; 迭代2用F6,F8）：

```asm
L.D    F0, 0(R1)       ; iter1 load
L.D    F6, -8(R1)      ; iter2 load  
ADD.D  F4, F0, F2      ; iter1 add (F0已就绪)
ADD.D  F8, F6, F2      ; iter2 add
DADDIU R1, R1, #-16    ; 更新指针
S.D    F4, 16(R1)      ; iter1 store（偏移量调整）
BNE    R1, R2, Loop
S.D    F8, 8(R1)       ; iter2 store（在延迟槽中）
```

周期分析（2个元素共约12周期）：
→ 6周期/元素

相比展开前的10周期/元素 → 显著改善。

</details>
