### 1.3 学习顺序说明

**为什么MD的章节顺序不是1→2→3→4→5？**

因为第1章和第2章相对独立，而第3章（流水线）是第4章（Tomasulo/ILP）的前置知识。第5章（Cache）也有大量计算需要先理解流水线的概念。

**推荐学习顺序**：
```
第1章(基础概念) → 第3章(流水线) → 第4章(Tomasulo/ILP) → 第5章(Cache) → 第2章(指令集设计) → 第6+8+9章(概念) → 互联网络+综合大题
```

理由：
- 第1章是最基础的概念，Amdahl和CPU公式整门课都要用
- 第3章流水线是第4章的前置，必须先学
- 第4章和第5章是最难的，在精力最充沛的时候攻克
- 第2章相对独立，放后面也不影响
- 第6+8+9章是纯概念记忆，放考前突击即可
- 互联网络设计需要前几章的基础

### 4.2 刷题方法

**选择题/填空题**：
1. 先做一遍
2. 对答案（`multiple_choice.md`有答案）
3. 错题 → 回到MD找到对应知识点 → 读一遍 → 用自己的话解释一遍为什么错了

**大题**：
1. 先看题目，尝试自己做
2. 做不出来没关系，看答案理解思路
3. **盖上答案，自己从头写一遍完整过程**
4. 隔一天后再做一遍（检验是否真会了）

**真题**：
1. 严格限时2小时
2. 不会的题也尽量写（考试时不能空着）
3. 做完再对答案
4. 错题整理成笔记

## 五、关键公式快速默写清单

> 考前必须能**不看书默写**这些公式。建议每天早上花10分钟默写一遍。

```
□ Amdahl定律: Speedup = 1 / [(1-Fe) + Fe/Se]
□ CPU时间: CPU_Time = IC × CPI × Tc
□ CPI加权: CPI = Σ(CPIi × ICi) / IC
□ 流水线吞吐率: TP = n / [(k+n-1)Δt]
□ 流水线加速比: S = nk / (k+n-1)
□ 流水线效率: E = n / (k+n-1)
□ AMAT: HitTime + MissRate × MissPenalty
□ 两级Cache: T1 + MR1 × (T2 + MR2 × MP2)
□ 带Cache的CPI: CPIexec + MemAcc/Inst × MR × MP
□ 全局不命中率 = MR1 × MR2
□ 可用性: MTTF / (MTTF + MTTR)
□ 向量时间: Tstartup + N
□ 两级存储: TA = T1 + F × TM
□ 比较对触发器数: C(n,2) = n(n-1)/2
```

## 七、常见踩坑提醒

1. **不要只看不写**：Tomasulo的状态表、时空图、Cache地址格式、Omega网络路由——这些都**必须动手写**，光看是学不会的
2. **不要忽视概念章**：第1章和第6/8/9章虽然只考概念，但加起来也有20+分
3. **AMAT别忘了乘不命中率**：很多同学直接加不命中开销
4. **两级Cache注意局部vs全局**：L2的局部不命中率是对L1不命中的访问而言的
5. **CPU时间公式注意单位**：频率是GHz、时间是ns，别忘了转换
6. **写出步骤就有分**：大题即使算不出最终答案，写出公式和代入过程也能拿大部分分
