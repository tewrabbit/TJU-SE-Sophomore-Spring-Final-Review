## 5.1 存储系统层次结构

### 5.1.1 设计目标

存储系统的三个相互矛盾的要求：
1. **容量大** → 但成本随容量增加
2. **速度快** → 但高速存储成本高
3. **成本低** → 但低成本存储速度慢

**解决方案**：利用程序局部性原理建立多级存储层次。

### 5.1.2 两级存储层次

设 $M_1$ 和 $M_2$ 构成两级存储系统（$M_1$ 靠近CPU）：

| 参数 | 含义 |
|------|------|
| $T_1$ | $M_1$ 的访问时间 |
| $T_2$ | $M_2$ 的访问时间 |
| $S_1, S_2$ | $M_1, M_2$ 的容量（$S_1 \ll S_2$） |
| $C_1, C_2$ | $M_1, M_2$ 的每位价格 |
| $H$ | **命中率** |
| $F = 1-H$ | **不命中率（失效率）** |

#### 对外特性

$$S = S_2 \quad\text{（总容量接近M2）}$$

$$C = \frac{C_1 S_1 + C_2 S_2}{S_1 + S_2} \approx C_2 \quad\text{（每位价格接近M2）}$$

#### 平均访问时间（最核心公式）

$$T_A = H \cdot T_1 + (1-H) \cdot (T_1 + T_M) = T_1 + F \cdot T_M$$

其中：
- $T_M$ = 不命中开销 = $T_2 + T_B$（访问M2的时间 + 块传送时间）
- $T_B$：传送一个块所需的额外时间

### 5.1.3 Cache-主存 vs 主存-辅存

| 特征 | Cache - 主存 | 主存 - 辅存 |
|------|-------------|-------------|
| **目的** | 弥补主存**速度**不足 | 弥补主存**容量**不足 |
| **实现方式** | 主要靠**硬件** | 主要靠**软件**（操作系统） |
| **速度/容量差** | 几倍 | 数万倍 |
| **块大小** | 几十字节 | 几百~几千字节 |
| **不命中时CPU** | 不切换（等待） | 切换到其他进程 |

### 5.1.4 存储层次要解决的四个问题

| 问题 | 对应策略 |
|------|---------|
| 1. **调入策略**（块放在Cache的哪里？） | **映像规则** |
| 2. **查找策略**（如何找到块？） | **查找算法** |
| 3. **替换策略**（Cache满了换出哪个块？） | **替换算法** |
| 4. **写策略**（写操作如何处理？） | **写策略** |

## 5.2 Cache基本知识

### 5.2.1 Cache和主存的块划分

- Cache和主存均匀划分为大小相等的**块**（Block）
- CPU以**字/字节**为单位访问，Cache和主存之间以**块**为单位交换数据
- **块地址**（Tag）：用于在Cache中标识主存的哪个块
- **块内偏移**（Block Offset）：用于确定块内的具体数据位置

### 5.2.2 三种映像规则

#### 1. 全相联映像（Fully Associative）

- 主存中任意一块可以放入Cache中的**任意**位置
- 优点：空间利用率最高，冲突不命中最少
- 缺点：查找需要与所有块比较，硬件成本最高

地址格式：**Tag + Block Offset**（无Index字段）

#### 2. 直接映像（Direct Mapped）

- 主存第 $i$ 块只能放入Cache的第 $j$ 块：$j = i \bmod M$（$M$ = Cache块数）
- 优点：硬件最简单，查找只比较一个块
- 缺点：冲突不命中最多

地址格式：**Tag + Index + Block Offset**

#### 3. 组相联映像（Set Associative）

- 主存第 $i$ 块只能放入Cache的第 $j$ 组：$j = i \bmod G$（$G$ = Cache组数）
- 在一组之内可以放到任意位置
- 每组中有 $n$ 个块 → **n路组相联**（$n = M / G$）

地址格式：**Tag + Set Index + Block Offset**

| 映像方式 | Index位数 | 并行比较次数 | 适合规模 |
|---------|----------|-------------|---------|
| 全相联 | 0 | 所有块 | 小TLB |
| 直接映像 | $\log_2 M$ | 1 | 大L1 Cache |
| n路组相联 | $\log_2 G$ | n | L1/L2 Cache |

### 5.2.3 相联度（Associativity）

- **相联度** = 每组中块的数量
- 直接映像：相联度 = 1
- 全相联：相联度 = Cache总块数
- n路组相联：相联度 = n

**2:1 Cache经验规则**：容量为N的直接映像Cache的不命中率 ≈ 容量为N/2的两路组相联Cache的不命中率

### 5.2.4 地址格式计算例题

**例**：Cache容量8KB，块大小32B，主存4GB（32位地址）

**直接映像**：
- 块数 = 8KB / 32B = 256块
- Index = $\log_2(256) = 8$ 位
- Block Offset = $\log_2(32) = 5$ 位
- Tag = 32 - 8 - 5 = 19位

地址格式：`Tag(19) | Index(8) | Offset(5)`

**2路组相联**：
- 组数 = 256/2 = 128组
- Set Index = $\log_2(128) = 7$ 位
- Block Offset = 5位
- Tag = 32 - 7 - 5 = 20位

地址格式：`Tag(20) | Set Index(7) | Offset(5)`

---

**例2（来自历年真题）**：主存容量4M字，Cache容量128K字，组相联映像。Cache共32组，块大小64字。

- 主存块数 = 4M / 64 = 65536块
- Cache块数 = 128K / 64 = 2048块
- 每组块数 = 2048 / 32 = 64 → **64路组相联**
- Set Index = $\log_2(32) = 5$ 位
- Block Offset = $\log_2(64) = 6$ 位
- Tag = $\log_2(65536) - 5 = 16 - 5 = 11$ 位（另一种算法：$\log_2(\text{主存块数}/\text{组数})$）

地址格式：`Tag(11) | Set Index(5) | Block Offset(6)`

### 5.2.5 查找算法

- **目录表**：每个Cache块对应一个目录项（Tag + 有效位 + 脏位）
- 直接映像：只比较1个Tag
- 组相联：比较n个Tag（n路并行）
- 全相联：比较所有块的Tag

### 5.2.6 替换算法

| 算法 | 原则 | 特点 |
|------|------|------|
| **随机法** | 随机选择被替换块 | 简单，未利用局部性 |
| **FIFO** | 替换**最早装入**的块 | 简单，可能有Belady异常 |
| **LRU** | 替换**最久未被使用**的块 | 命中率最高，硬件复杂 |

> 考纲说明：LRU算法的硬件实现细节不考。了解概念即可。

### 5.2.7 写策略

#### 写命中（Write Hit）

| 策略 | 做法 | 优点 | 缺点 |
|------|------|------|------|
| **写直达法**（Write-Through） | 同时写入Cache和下一级存储器 | 一致性好，实现简单 | 写操作慢，占用带宽 |
| **写回法**（Write-Back） | 仅写入Cache，被替换时才写回 | 速度快，带宽需求低 | 一致性复杂，需脏位 |

#### 写不命中（Write Miss）

| 策略 | 做法 | 通常搭配 |
|------|------|---------|
| **按写分配法**（Write-Allocate） | 先将块调入Cache，再写 | **写回法** |
| **不按写分配法**（No-Write-Allocate） | 直接写入下一级，不调入Cache | **写直达法** |

## 5.3 Cache性能分析（考纲大题#4 — AMAT核心）

### 5.3.1 平均访存时间（AMAT）

$$AMAT = Hit\ Time + Miss\ Rate \times Miss\ Penalty$$

- $Hit\ Time$：命中时的访问时间（包含Cache访问和TLB转换）
- $Miss\ Rate$：不命中率（失效率）
- $Miss\ Penalty$：不命中开销（从下一级取块到送进CPU的全部时间）

### 5.3.2 CPU时间与Cache

$$CPU\ Time = (CPU_{exec\_cycles} + MemoryStallCycles) \times Clock\ Cycle\ Time$$

$$MemoryStallCycles = IC \times \frac{MemoryAccess}{Instruction} \times MissRate \times MissPenalty$$

$$CPU\ Time = IC \times \left(CPI_{exec} + \frac{MemAccess}{Instr} \times MissRate \times MissPenalty\right) \times ClockCycle$$

### 5.3.3 经典AMAT计算例题

**例1**：基础AMAT计算

Cache命中时间=1ns，不命中率=5%，不命中开销=50ns。求平均访存时间。

$$AMAT = 1 + 0.05 \times 50 = 1 + 2.5 = 3.5\text{ns}$$

---

**例2**：带CPI的CPU时间计算（经常考！）

某计算机CPI(exec)=2.0（假设全部Cache命中），Load/Store指令占40%，每条指令平均访存1.3次，数据Cache不命中率=2%，不命中开销=50周期。时钟频率2GHz，程序执行10^8条指令。

计算：
1. 包含Cache不命中的实际CPU时间
2. 与全部命中的情况相比，差多少倍

**解**：

实际CPI = $2.0 + 1.3 \times 0.02 \times 50 = 2.0 + 1.3 = 3.3$

全命中CPI = 2.0

$$\frac{CPU\ Time_{实际}}{CPU\ Time_{全命中}} = \frac{3.3}{2.0} = 1.65\text{倍}$$

$$CPU\ Time_{实际} = 10^8 \times 3.3 \times \frac{1}{2\times10^9} = 0.165\text{s}$$

---

**例3**：比较两种Cache设计（设计比较题）

两种方案：
- 方案A：16KB直接映像，命中时间=1ns，不命中率=3.0%
- 方案B：32KB两路组相联，命中时间=1.2ns，不命中率=2.0%

不命中开销均为50ns。问哪个方案更好？

**解**：

$AMAT_A = 1 + 0.03 \times 50 = 1 + 1.5 = 2.5\text{ns}$

$AMAT_B = 1.2 + 0.02 \times 50 = 1.2 + 1.0 = 2.2\text{ns}$

**方案B更好**（虽然命中时间多了0.2ns，但不命中率降低带来的收益更大）。

---

**例4**：两级Cache计算（局部不命中率 vs 全局不命中率）

某系统有L1和L2 Cache：
- L1命中时间=1周期，不命中率=4%
- L2命中时间=10周期，**局部不命中率**=50%
- L2不命中开销=100周期

求：（1）平均访存时间 （2）L2的全局不命中率

**解**：

(1) $$AMAT = 1 + 4\% \times (10 + 50\% \times 100) = 1 + 4\% \times 60 = 1 + 2.4 = 3.4\text{周期}$$

(2) L2全局不命中率 = L1不命中率 × L2局部不命中率 = $4\% \times 50\% = 2\%$

**关键区分**：
- **局部不命中率**：该级Cache的不命中次数 / 到达该级Cache的访问次数
- **全局不命中率**：该级Cache的不命中次数 / CPU发出的总访问次数

---

**例5**：要达到一定加速比需要什么条件（反推题）

Cache访问周期10ns，命中率0.95。要使Cache-主存系统的加速比大于10，主存的访问周期最大为多少？

**解**：

$$T_A = 10 + (1-0.95) \times T_M = 10 + 0.05T_M$$

无Cache时每次访存 $T_M$

$$Speedup = \frac{T_M}{10 + 0.05T_M} > 10$$

$$T_M > 100 + 0.5T_M$$

$$0.5T_M > 100$$

$$T_M > 200\text{ns}$$

主存访问周期至少要大于200ns。

## 5.4 降低Cache不命中率（3C模型 + 8种方法）

### 5.4.1 三种不命中（3C模型）

| 类型 | 英文 | 含义 | 如何降低 |
|------|------|------|---------|
| **强制性不命中** | Compulsory Miss | 首次访问某块，从未进入过Cache | 增加块大小、预取 |
| **容量不命中** | Capacity Miss | Cache太小，工作集放不下 | 增加Cache容量 |
| **冲突不命中** | Conflict Miss | 过多块映射到同一位置 | 提高相联度 |

**与各因素的关系**：
- 提高相联度 → 降低冲突不命中（不影响强制性/容量）
- 增加容量 → 降低容量不命中（不影响强制性）
- 增加块大小 → 降低强制性不命中（但可能增加冲突不命中）

### 5.4.2 增加块大小

- 降低了强制性不命中（利用了空间局部性）
- 但也减少了Cache中的块数 → **可能增加冲突不命中**
- 还会**增加不命中开销**（需要传更多数据）
- 效应曲线：先降后升

### 5.4.3 增加Cache容量

- 直接降低容量不命中
- 缺点：成本增加、可能增加命中时间

### 5.4.4 提高相联度

- 规则：相联度提高 → 不命中率降低
- **2:1经验规则**：N的直接映像 ≈ N/2的两路组相联的不命中率
- 超过8路后收益递减
- 缺点：增加命中时间

### 5.4.5 伪相联Cache

- 先按直接映像查找（快）
- 若未命中，取反最高位地址再查一次（稍慢但有用）
- 命中率接近组相联，命中时间接近直接映像

$$AMAT = HitTime + MissRate_{pseudo} \times MissPenalty_{pseudo}$$

### 5.4.6 硬件预取

- 在指令/数据被用到之前就取入Cache
- 指令预取：通常取下一个顺序块
- 数据预取：根据访问模式预测

### 5.4.7 编译器控制预取

- **寄存器预取** vs **Cache预取**（预取到寄存器还是Cache）
- **故障性预取** vs **非故障性预取**（地址非法时是否产生异常）
- 重点用于循环中数组的预取

### 5.4.8 编译优化

| 优化方法 | 说明 |
|---------|------|
| **数组合并** | 将多个相关数组合并为一个，提高空间局部性 |
| **内外循环交换** | 让内层循环按存储顺序访问数组 |
| **循环融合** | 将访问相同数据的多个循环合并 |
| **分块（Blocking）** | 将大数组分成小块处理，使数据留在Cache中 |

### 5.4.9 牺牲Cache（Victim Cache）

- 在Cache和下一级存储器之间设置一个小容量的全相联Cache
- 存放被替换出去的块
- 4个表项的Victim Cache可降低4KB直接映像Cache的冲突不命中20~90%

## 5.5 减少Cache不命中开销（5种方法）

### 5.5.1 两级Cache

L1：小而快（命中时间=1~2周期）
L2：大而慢（命中时间=10~20周期）

$$AMAT = Hit_{L1} + MissRate_{L1} \times (Hit_{L2} + MissRate_{L2} \times MissPenalty_{L2})$$

**L2 Cache设计考虑**：
- L2的速度不影响CPU时钟频率（L2不在关键路径上）
- L2可以使用更大容量、更高相联度、更大块

### 5.5.2 让读不命中优先于写

- 写缓冲器可能包含读不命中所需的最新数据
- 检查写缓冲器：如果命中则取用写缓冲器中的数据

### 5.5.3 写缓冲合并

- 如果写缓冲器中已有相同地址的待写数据，就合并写入
- 减少写缓冲器占用和写操作次数

### 5.5.4 请求字优先

- **尽早重启动**：请求字一到就恢复CPU执行
- **请求字优先**：先传请求字，再传块中其余字

### 5.5.5 非阻塞Cache

- 允许在Cache不命中期间继续处理其他访问
- **命中下不命中**（Hit Under Miss）：不命中期间允许继续命中
- **多重不命中下不命中**（Multiple Miss Under Miss）：允许多个不命中同时处理

## 5.8 Cache替换策略硬件实现——比较对法（考纲大题#5）

### 5.8.1 基本思想

对于n路组相联Cache，每组中有n个块。比较对法在每两个块之间设置一个**比较对触发器**，跟踪两个块的访问先后关系。

### 5.8.2 工作原理

以4路组相联为例（块A, B, C, D）：

需要的比较对 = $\binom{4}{2} = 6$ 个触发器：
- T_AB：A更近被访问则=1，B更近则=0
- T_AC：A更近被访问则=1，C更近则=0
- T_AD：A更近被访问则=1，D更近则=0
- T_BC：B更近被访问则=1，C更近则=0
- T_BD：B更近被访问则=1，D更近则=0
- T_CD：C更近被访问则=1，D更近则=0

当访问某块（如B）时：
- 所有和B相关的比较对都设置B=1：T_AB=0, T_BC=1, T_BD=1
- 其他比较对不变

**要找到LRU块**：找比较对中全为0的那个块 → 在所有比较对中都是"另一方更近"

### 5.8.3 成本分析

n路组相联的比较对触发器总数：
- 每组：$\binom{n}{2} = \frac{n(n-1)}{2}$ 个触发器
- 总共：组数 × $\frac{n(n-1)}{2}$

**成本比较**：

| 相联度 | 每组的触发器数 | 相对成本 |
|--------|------------|---------|
| 2路 | $\binom{2}{2}=1$ | 1 |
| 4路 | $\binom{4}{2}=6$ | 6× |
| 8路 | $\binom{8}{2}=28$ | 28× |
| 16路 | $\binom{16}{2}=120$ | 120× |

由此可见高相联度的LRU硬件实现代价非常高。

### 5.8.4 降低成本的方法

可以采用**近似LRU**：
- 只记录部分比较关系
- 或用二叉树变体（约简LRU）
- 牺牲一点命中率，大幅降低硬件成本

## 5.9 并行主存系统

### 5.9.1 增加存储器带宽的方法

1. **单体多字存储器**：每个存储周期读出多个字
   - 优点：简单
   - 缺点：有些字可能没用；写操作复杂

2. **多体交叉存储器**：多个独立的存储体轮流工作

### 5.9.2 高位交叉 vs 低位交叉编址

设有m个存储体，每个体有n个存储单元：

**高位交叉编址**（用于容量扩展）：
$$A = j \times n + i \quad (\text{j=体号, i=体内地址})$$
- 地址高位为体号，低位为体内地址
- 连续地址在同一个体中
- 适合容量扩展，带宽增加有限

**低位交叉编址**（用于带宽扩展）：
$$A = i \times m + j \quad (\text{i=体内地址, j=体号})$$
- 地址低位为体号，高位为体内地址
- 连续地址在不同的体中
- 可以流水访问：启动间隔 $\tau = T_M / m$

> **考试重点**：高位交叉常用于容量扩展，低位交叉常用于带宽扩展

### 5.9.3 体冲突

相邻访问落到同一存储体 → 产生**体冲突**（Bank Conflict）

**例子**：128个存储体，访问步长为128 → 所有访问都是同一个体 → 带宽退化为单体的1/128

**解决方法**：
- 软件：循环交换、数组填充
- 硬件：使用质数个存储体

## 5.10 课后思考题参考答案要点

1. **存储层次结构依据**：局部性原理；目标：高性价比
2. **两层存储对外特性**：容量≈M2，价格≈M2，速度≈M1（高命中率时）
3. **Cache-主存解决速度问题，主存-辅存解决容量问题**
4. **映像规则是存储层次的基础——决定了查找方式**
5. **三种映像比较**：全相联（利用率最高、最贵）、直接映像（最简单、冲突最多）、组相联（折中）
6. **写直达 vs 写回**：写直达一致性高、速度慢；写回速度快、一致性复杂
7. **平均访存时间**是评价Cache性能的核心指标
8. **提高Cache性能**：3个方面17种方法
9. **增加带宽**：单体多字和多体交叉（低位交叉编址）
10. **比较对法**：两两比较，LRU为全0的块
