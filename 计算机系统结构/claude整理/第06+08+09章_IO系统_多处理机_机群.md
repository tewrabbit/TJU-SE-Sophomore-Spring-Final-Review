# 第一部分：第6章 · 输入输出系统

## 6.1 I/O系统性能

### 6.1.1 I/O是系统性能瓶颈

Amdahl定律也适用于I/O系统：

**例**：I/O占响应时间的10%，CPU加速10倍 → 总加速比仅约5倍。CPU加速100倍 → 总加速比仅约10倍（CPU 90%时间浪费在等I/O）。

> **结论**：忽略I/O的CPU加速是无效的。

## 6.2 可靠性、可用性

### 6.2.1 核心指标

| 指标 | 英文 | 含义 |
|------|------|------|
| **MTTF** | Mean Time To Failure | 平均失效前时间 |
| **MTTR** | Mean Time To Repair | 平均修复时间 |
| **MTBF** | Mean Time Between Failures | 平均故障间隔时间（MTBF = MTTF + MTTR） |
| **失效率** | Failure Rate | $\lambda = 1/MTTF$ |
| **可用性** | Availability | $\frac{MTTF}{MTTF + MTTR}$ |

**系统失效率**：对于串联可靠性模型，系统失效率 = 各部件失效率之和。

$$FailureRate_{system} = \sum FailureRate_i$$

**例**：某磁盘子系统有10个硬盘（MTTF=1,000,000小时），1个SCSI控制器（MTTF=500,000h），1个UPS（200,000h），1个风扇（200,000h），1根电缆（1,000,000h）。

系统失效率 = $10/1M + 1/500K + 1/200K + 1/200K + 1/1M$
$= 10/1M + 2/1M + 5/1M + 5/1M + 1/1M = 23/1M = 0.000023$/小时

MTTF = $1/0.000023 \approx 43,478$ 小时 ≈ 5年

## 6.3 RAID分级

### 6.3.1 RAID 0-6总览

| 级别 | 名称 | 最少盘数 | 空间开销 | 容错能力 | 特点 |
|------|------|---------|---------|---------|------|
| **RAID 0** | 条带化 | 2 | 0 | 无 | 高性能，无冗余 |
| **RAID 1** | 镜像 | 2 | 100% | 1块盘故障 | 简单可靠，代价高 |
| **RAID 2** | Hamming码 | 3 | 对数级 | 1块盘故障 | 未商用化 |
| **RAID 3** | 位交叉奇偶校验 | 3 | 1块盘 | 1块盘故障 | 大块读写带宽高 |
| **RAID 4** | 块交叉奇偶校验 | 3 | 1块盘 | 1块盘故障 | 小写校验盘是瓶颈 |
| **RAID 5** | 块交叉分布奇偶校验 | 3 | 1块盘 | 1块盘故障 | **最常用**，分布式校验 |
| **RAID 6** | P+Q双校验 | 4 | 2块盘 | 2块盘故障 | 高可靠，写开销大 |

### 6.3.2 RAID 10 vs RAID 01

| RAID 10 (RAID 1+0) | RAID 01 (RAID 0+1) |
|---------------------|---------------------|
| 先镜像，后条带 | 先条带，后镜像 |
| 可靠性更高 | 可靠性较低 |

### 6.3.3 RAID实现方式

1. **软件实现**：由操作系统完成（成本低、占用CPU）
2. **子系统实现**：专用RAID控制器
3. **阵列卡实现**：插入式RAID卡

> 选择题陷阱：RAID0不是"RAID"（因为R=Redundant，RAID0没有冗余）。

## 6.4 通道处理机

### 6.4.1 通道的概念

**通道**：专门负责I/O的**专用处理器**，执行通道程序来控制数据传输。

- 通道有自己的指令系统（通道指令）
- CPU通过**访管指令**（陷入指令）调用通道
- 通道完成传输后向CPU发送I/O中断请求

### 6.4.2 通道分类

| 类型 | 适用设备 | 工作方式 | 特点 |
|------|---------|---------|------|
| **字节多路通道** | 低速设备（打印机、键盘） | 每次传送1字节，分时服务多个设备 | 子通道概念 |
| **选择通道** | 高速设备（磁盘） | 一次连接一个设备，传完所有数据 | 独占通道 |
| **数组多路通道** | 高速设备 | 一次传送一个数据块，交替服务多个设备 | 综合前两种的优点 |

### 6.4.3 通道流量

**通道流量**：通道在单位时间内能传送的数据量。

**选择通道最大流量** = $\frac{1}{T_s/n + T_D}$

其中 $T_s$ = 设备选择时间，$n$ = 一次传送的字节数，$T_D$ = 传送一个字节的时间

# 第二部分：第8章 · 多处理机

## 8.1 SMP与DSM

### 8.1.1 两种多处理机架构

| 特征 | **SMP**（集中式共享多处理机） | **DSM**（分布式共享多处理机） |
|------|---------------------------|----------------------------|
| 全称 | Symmetric MultiProcessor | Distributed Shared Memory |
| 别名 | UMA（均匀存储器访问） | NUMA（非均匀存储器访问） |
| 处理器数 | 少（≤32） | 多（数十到数百） |
| 存储器 | 一个集中的物理存储器 | 存储器分布在各结点 |
| 访问延迟 | 均匀 | 本地快、远程慢 |
| 互联 | 总线/交叉开关 | 高带宽定制网络 |

### 8.1.2 两种地址空间模型

| | 共享地址空间 | 独立地址空间 |
|---|---|---|
| **寻址** | 统一的逻辑地址空间 | 每个结点独立编址 |
| **通信方式** | 共享存储器（load/store） | 消息传递（send/receive） |
| **编程** | 类似单机编程 | 需要显式消息传递 |
| **同步** | 需要专门的同步机制 | 同步与消息关联 |

### 8.1.3 消息传递的优点

1. 通信是**显式**的 → 时间开销清楚
2. 硬件**更简单**
3. 同步**自然地**与发送消息相关联
4. 使程序员更关注通信开销

### 8.1.4 Amdahl定律在多处理机中的应用

**例**：100个处理器，加速比80倍。求串行部分占比。

$$80 = \frac{100}{1 + (100-1) \times f}$$

其中 $f$ 为串行部分比例（注意：这里用的是多处理机加速比的另一种形式）

解得：$f \approx 0.25\%$，并行部分99.75%

> 即使串行部分仅占0.25%，100个处理器也只获得80倍加速。

## 8.2 Cache一致性

### 8.2.1 问题

多个处理器的Cache中可能有同一块数据的副本，当某个处理器修改了该块 → **数据不一致**。

### 8.2.2 Cache一致性的三个条件

1. 处理器P对X写后再读X，中间无其他处理器写X → P读到的是自己写的值
2. P写X后，Q读X（中间无其他写）→ Q读到的是P写的值
3. 对同一位置的写被**串行化**（所有处理器看到相同的写顺序）

### 8.2.3 两种Cache一致性协议

| | **监听式协议** (Snooping) | **目录式协议** (Directory) |
|---|---|---|
| **状态信息位置** | 随Cache块一起 | 集中在目录中 |
| **通信方式** | 通过总线广播 | 点对点消息 |
| **适用规模** | 小规模（SMP） | 大规模（DSM） |
| **实现复杂度** | 低 | 高 |

---

## 8.3 监听式协议

### 8.3.1 写作废 vs 写更新

| | 写作废 (Write-Invalidate) | 写更新 (Write-Update) |
|---|---|---|
| **做法** | 写之前将其他所有副本作废 | 写时将新值广播给所有副本 |
| **后续行为** | 写者独占 | 所有副本都更新 |
| **适用场景** | 多次写之间无读 → **最常用** | 写之后立即有多个读 |
| **带宽** | 一次作废 | 每次写都广播 |

> **写作废协议是最常用的**。

### 8.3.2 MSI状态

| 状态 | 含义 | 可读？ | 可写？ |
|------|------|--------|--------|
| **I** (Invalid) | 该块无效 | 否 | 否 |
| **S** (Shared) | 共享（干净，只读） | 是 | 否 |
| **M** (Modified) | 已修改（脏，独占） | 是 | 是 |

### 8.3.3 状态转换

```
CPU读命中(I→S)：RdMiss → 从总线读入 → 置S
CPU写命中(S→M)：WtMiss → 作废总线上的其他副本 → 置M
CPU写命中(M→M)：直接写
CPU写不命中(I→M)：WtMiss → 作废 → 读入 → 写 → 置M

总线RdMiss(被监听,S)：提供数据，自己保持在S
总线RdMiss(被监听,M)：提供数据，写回存储器，自己变为S
总线WtMiss(被监听,S)：置I
总线WtMiss(被监听,M)：写回存储器，置I
```

## 8.4 目录式协议

### 8.4.1 目录

**目录**：每个存储块对应一个目录项，记录：
- 该块的状态
- 哪些处理器的Cache中有该块的副本（**共享集** / 位向量）

### 8.4.2 三种结点

| 结点类型 | 含义 |
|---------|------|
| **本地结点** (Local) | 发出访问请求的结点 |
| **宿主结点** (Home) | 包含所访问存储块及其目录项的结点 |
| **远程结点** (Remote) | 拥有副本的结点 |

### 8.4.3 目录块状态

| 状态 | 含义 |
|------|------|
| **U** (Uncached) | 块不在任何Cache中 |
| **S** (Shared) | 一个或多个Cache拥有只读副本（干净，与存储器一致） |
| **E** (Exclusive) | 仅一个Cache拥有并已修改（脏） |

### 8.4.4 消息类型

| 消息 | 方向 | 含义 |
|------|------|------|
| **RdMiss(P, K)** | 本地→宿主 | 结点P请求读块K |
| **WtMiss(P, K)** | 本地→宿主 | 结点P请求写块K |
| **Invalidate(K)** | 宿主→远程 | 作废块K的所有副本 |
| **DReply(D)** | 宿主→本地 | 回送数据D |
| **WtBack(K, D)** | 远程→宿主 | 写回修改后的块K |

### 8.4.5 目录结构类型

| 类型 | 实现方式 | 特点 |
|------|---------|------|
| **全映像目录** | N位位向量（N=处理器数） | 简单，但空间开销大（O(N²)） |
| **有限映像目录** | 固定数量指针（如2个） | 可扩展，但需要处理溢出 |
| **链式目录** | 用链表链接共享者 | 无长度限制，实现复杂 |

|------|------|

## 8.6 同时多线程（SMT）

> 考纲说明：8.5.2 SMT设计、8.5.3 SMT性能不考。了解概念即可。

### 8.6.1 三种多线程方式

| 方式 | 线程切换时机 | 优点 | 缺点 |
|------|------------|------|------|
| **细粒度多线程** | 每周期切换 | 隐藏长短停顿 | 降低单线程速度 |
| **粗粒度多线程** | 长停滞时切换 | 单线程降速少 | 隐藏停顿能力有限 |
| **同时多线程（SMT）** | 同周期多线程发射 | 同时开发TLP和ILP | 硬件复杂 |

### 8.6.2 SMT核心思想

在支持动态调度的超标量处理器上，同周期可发射多条来自不同线程的指令到不同功能部件。

# 第三部分：第9章 · 机群系统

## 9.1 机群的基本概念

**机群（Cluster）**：多台独立计算机（PC/工作站/SMP）通过高性能网络连接，协同完成并行计算任务。

### 9.1.1 机群结构

- 每个结点：完整计算机（有本地磁盘、OS，可独立使用）
- 连接方式：商用网络（松耦合，通过I/O总线连接）
- NIC（网络接口卡）接在PCI总线上

### 9.1.2 单一系统映像（SSI）

**SSI** = Single System Image。机群中间件提供的4个特性：

1. **单一系统**：从单一逻辑点获得系统服务
2. **单一控制**：管理员从一处管理所有资源
3. **对称性**：服务可从任意结点获得
4. **位置透明**：用户不知道物理资源位置

### 9.1.3 最小SSI服务

- **单一登录点**：一次登录即可使用所有结点
- **单一文件系统**：所有结点共享一套可执行文件
- **单一作业管理**：可从任意结点提交作业

## 9.2 机群的分类

| 类型 | 用途 | 特点 |
|------|------|------|
| **高可用性机群** (HA Cluster) | 故障时服务转移 | 冗余机制，任务接管 |
| **负载均衡机群** (Load Balancing) | 分发请求 | 监控结点，负载分发 |
| **高性能机群** (High Performance) | 并行计算 | 高速网络，性价比高 |

## 9.3 机群的优缺点

**优点**：
1. 开发周期短（使用商用组件）
2. 可靠性高（结点独立）
3. 可扩展性好
4. 配置灵活
5. 性价比高
6. 编程方便

**缺点**：
- 维护成本高（管理多个独立计算机）
