## 1.1 计算机系统的多级层次结构

### 1.1.1 六级层次

从计算机语言的角度，计算机系统可以分为6个层次：

```
┌──────────────────────────────────────┐
│  L6: 应用语言虚拟机                      │  ← 最高层（软件）
│       (Application Language VM)      │
├──────────────────────────────────────┤
│  L5: 高级语言虚拟机                      │
│       (High-Level Language VM)       │     翻译实现
├──────────────────────────────────────┤
│  L4: 汇编语言虚拟机                      │
│       (Assembly Language VM)         │
├──────────────────────────────────────┤
│  L3: 操作系统虚拟机                      │
│       (Operating System VM)          │
├──────────────────────────────────────┤     解释实现
│  L2: 机器语言级（传统机器级）              │
│       (Machine Language Level)       │
├──────────────────────────────────────┤
│  L1: 微程序机器级                        │  ← 最低层（硬件）
│       (Microprogram Level)           │
└──────────────────────────────────────┘
```

- **L1 ~ L3**：通过**解释**（Interpretation）实现
- **L4 ~ L6**：通过**翻译**（Translation）实现
- **L1 ~ L2**：属于**硬件/固件**
- **L3 ~ L6**：属于**软件**
- **本课程重点**：机器语言级（L2）

### 1.1.2 翻译 与 解释

| | 翻译 (Translation) | 解释 (Interpretation) |
|---|---|---|
| **方式** | 先将整个L+1级程序转换成L级程序，再执行转换后的L级程序 | 每译码一条L+1指令，就立即执行一串等效的L级指令 |
| **速度** | 快 | 慢 |
| **存储** | 占用空间大 | 占用空间小 |
| **类比** | 编译型语言（C、C++） | 解释型语言（Python早期、BASIC） |

**虚拟机 (Virtual Machine)**：通过软件实现的机器。L1-L6每一层都是其上一层的虚拟机。

> **考点提示**：选择题常考"哪一层通过翻译/解释实现"，填空题常考"虚拟机是指通过**软件**实现的机器"。

## 1.2 计算机系统结构、组成与实现

### 1.2.1 三个概念的关系

```
系统结构 (Architecture) ──逻辑实现──▶ 组成 (Organization) ──物理实现──▶ 实现 (Implementation)
     │                                    │                               │
  "是什么"                              "怎么做"                         "用什么做"
  软硬件交界面                          数据流/控制流                     物理器件/工艺
```

### 1.2.2 定义

**计算机系统结构**（经典定义，Amdahl 1964）：传统机器语言程序员所看到的计算机属性——即**概念性结构**和**功能特性**。

对于通用寄存器型机器，系统结构包括：
- 指令系统（类型、格式、排序、控制方式）
- 数据表示（硬件能直接识别和处理的数据类型）
- 寻址规则（最小寻址单位、寻址方式种类）
- 寄存器定义（类型、数量、使用约定）
- 中断系统（中断类型、硬件响应功能）
- 机器工作状态定义和切换（管态/用户态）
- 存储系统（主存容量、最大可寻址空间）
- 信息保护（保护方式、硬件支持）
- I/O结构（连接方式、数据传输方式、I/O操作状态）

**系统结构的本质**：确定**软硬件交界面**——之上是软件，之下是硬件/固件。

**计算机组成**：系统结构的逻辑实现（数据流、控制流、事件的排序方式）

**计算机实现**：组成的物理实现（处理器/主存物理结构、器件集成度、信号传输等）

### 1.2.3 透明性

**透明性（Transparency）**：本来存在的事物或属性，从某种角度看又好像不存在。

| 对谁透明 | 透明的事物举例 | 不透明的事物举例 |
|----------|---------------|-----------------|
| 机器语言程序员 | 乘法器（硬件实现） | 乘法指令、数据表示、寻址方式 |
| 汇编语言程序员 | 指令寄存器 | 程序计数器(PC) |
| 高级语言程序员 | 寄存器分配 | — |

> **考点提示**："对机器语言程序员透明"是选择题高频考点。记住：**硬件实现细节**是透明的，**编程接口**是不透明的。

## 1.3 Flynn分类法

1966年Flynn提出的分类方法，根据**指令流**和**数据流**的**多倍性**对计算机系统进行分类。

| 类型 | 全称 | 说明 | 实例 |
|------|------|------|------|
| **SISD** | Single Instruction, Single Data | 传统单处理器 | 早期PC |
| **SIMD** | Single Instruction, Multiple Data | 一条指令处理多个数据 | 向量处理机、阵列处理机 |
| **MISD** | Multiple Instruction, Single Data | 多条指令处理同一数据 | 无实际产品（流水线有时被归入此类） |
| **MIMD** | Multiple Instruction, Multiple Data | 多指令流多数据流 | 多处理机、多核处理器 |

- **指令流**（Instruction Stream）：机器执行的指令序列
- **数据流**（Data Stream）：指令流调用的数据序列
- **多倍性**（Multiplicity）：在系统最受限的部件上，同一执行阶段的最大指令/数据数

**流水线与Flynn分类的关系**：
- 标量流水线 → SISD
- 向量流水线 → SIMD

## 1.4 Amdahl定律（大题考点#1）

### 1.4.1 基本公式

$$Speedup = \frac{T_{改进前}}{T_{改进后}} = \frac{1}{(1-F_e) + \frac{F_e}{S_e}}$$

其中：
- $F_e$：**可改进比例**——可改进部分在执行时间中所占的比例（0~1）
- $S_e$：**部件加速比**——改进部分的加速比（≥1）

### 1.4.2 关键理解

1. 加速比取决于**两个因素**：$F_e$（有多大比例可以改进）和 $S_e$（能改进多少倍）
2. **即使把某一部分加速到无限快，总加速比也有上限**：$Speedup_{max} = \frac{1}{1-F_e}$
3. **Amdahl定律的实质**：改进效果受限于可改进部分的占比

### 1.4.3 四道经典例题

**例1：基本计算**

将系统中某一功能的处理速度加快到原来的20倍，但该功能的处理时间仅占整个系统运行时间的40%。求系统加速比。

$$F_e = 0.4, \quad S_e = 20$$

$$Speedup = \frac{1}{(1-0.4) + \frac{0.4}{20}} = \frac{1}{0.6 + 0.02} = \frac{1}{0.62} \approx 1.61$$

**结论**：即使单部件快了20倍，整体仅加速1.61倍。这就是Amdahl定律的"收益递减"效应。

---

**例2：反推 $F_e$**

将浮点运算速度提高到原来的25倍，系统整体加速比为4。问浮点运算原占总执行时间的比例是多少？

$$4 = \frac{1}{(1-F_e) + \frac{F_e}{25}}$$

$$1 - F_e + \frac{F_e}{25} = 0.25$$

$$1 - \frac{24}{25}F_e = 0.25$$

$$\frac{24}{25}F_e = 0.75$$

$$F_e = 0.75 \times \frac{25}{24} = 0.78125 \approx 78.1\%$$

---

**例3：比较两种改进方案（CPU公式结合）**

某计算机的FP指令占25%，CPI=4.0；其它指令CPI=1.33；FPSQR指令占2%，CPI=20。

方案一：将FPSQR的CPI降为2
方案二：将所有FP指令的CPI降为2.5

问：哪个方案更好？

**解**：

原CPI = 0.25 × 4.0 + 0.75 × 1.33 = 1.0 + 0.9975 = 2.0（约）

方案一：CPI1 = 2.0 - (20-2) × 0.02 = 2.0 - 0.36 = **1.64**

方案二：CPI2 = 2.0 - (4.0-2.5) × 0.25 = 2.0 - 0.375 = **1.625**

方案二CPI更小，**方案二更好**。这说明"对所有FP指令都加速"比"仅加速FPSQR"效果好，尽管加速倍数更小——因为覆盖范围广。

---

**例4：加速比上限**

某程序80%可以并行化，20%必须串行执行。问：
1. 使用10个处理器加速比是多少？
2. 最多能达到多少加速比？

**解**：

$F_e = 0.8, \quad S_e = 10$

加速比 = $\frac{1}{(1-0.8) + \frac{0.8}{10}} = \frac{1}{0.2 + 0.08} = \frac{1}{0.28} \approx 3.57$

最大加速比 = $\frac{1}{1-0.8} = \frac{1}{0.2} = 5$（即使使用无限多处理器）

## 1.5 CPU性能公式（大题考点#1）

### 1.5.1 基本公式

$$CPU\ Time（CPU执行时间）= IC \times CPI \times T_c$$

三个参数的**影响因素**：

| 参数 | 含义 | 主要影响因素 |
|------|------|-------------|
| **IC**（指令数） | 程序执行的总指令条数 | 指令集结构 + 编译器技术 |
| **CPI**（每指令周期数）| 平均每条指令的时钟周期数 | 计算机组成 + 指令集结构 |
| **$T_c$**（时钟周期）| 时钟周期时间 = 1/频率 | 硬件工艺 + 计算机组成 |

### 1.5.2 CPI的加权计算

$$CPI = \frac{总时钟周期数}{IC} = \sum_{i=1}^{n}\left(CPI_i \times \frac{IC_i}{IC}\right)$$

其中 $\frac{IC_i}{IC}$ 是第 $i$ 类指令的比例。

### 1.5.3 经典例题

**例1：计算CPI和CPU时间**

某计算机有三类指令，频率和CPI如下。时钟频率2GHz，程序执行10^8条指令。

| 指令类型 | 占比 | CPI |
|----------|------|-----|
| A（整数运算） | 30% | 1 |
| B（数据传输） | 50% | 2 |
| C（浮点运算） | 20% | 3 |

**解**：

CPI = 0.3×1 + 0.5×2 + 0.2×3 = 0.3 + 1.0 + 0.6 = **1.9**

$T_c = \frac{1}{2 \times 10^9} = 0.5ns$

CPU Time = $10^8 \times 1.9 \times 0.5 \times 10^{-9} = 0.095s$

---

**例2：比较两种设计方案（重点题型）**

某程序指令混合比例为：整数45%（1周期），数据传输32%（2周期），浮点15%（2周期），控制转移8%（2周期）。处理器频率40MHz。

方案A：保持原设计
方案B：优化数据传输指令，将其CPI降为1，但时钟周期增加10%

问：哪个方案更快？计算加速比。

**解**：

方案A：
CPI_A = 0.45×1 + 0.32×2 + 0.15×2 + 0.08×2 = 0.45 + 0.64 + 0.30 + 0.16 = 1.55
$T_{cA} = 1/40\text{MHz} = 25\text{ns}$
CPU Time_A = IC × 1.55 × 25 = **38.75 IC ns**

方案B：
CPI_B = 0.45×1 + 0.32×1 + 0.15×2 + 0.08×2 = 0.45 + 0.32 + 0.30 + 0.16 = 1.23
$T_{cB} = 25 \times 1.10 = 27.5\text{ns}$
CPU Time_B = IC × 1.23 × 27.5 = **33.825 IC ns**

$Speedup = \frac{38.75}{33.825} \approx 1.15$

方案B更快，加速1.15倍。虽然时钟变慢了，但CPI降低更多，总体来说更优。

## 1.6 计算机系统设计的定量原理

### 1.6.1 四大定量原理

| 原理 | 内容 | 指导意义 |
|------|------|---------|
| **以经常性事件为重点** | 优化频繁发生的情况 | 把资源用在刀刃上 |
| **Amdahl定律** | 加速比受限于可改进部分占比 | 不要无限追求局部优化 |
| **CPU性能公式** | CPU时间 = IC × CPI × T | 三个维度综合优化 |
| **程序局部性原理** | 90%时间执行10%代码 | 设计Cache、流水线等的基础 |

### 1.6.2 程序的局部性原理

- **时间局部性**（Temporal Locality）：即将要用到的信息，很可能就是现在正在使用的信息
- **空间局部性**（Spatial Locality）：即将要用到的信息，很可能与现在正在使用的信息在存储空间上相邻

**考试常见问法**：
- "Cache利用了程序的什么特性？" → 局部性原理
- "预取技术利用了什么原理？" → 空间局部性
- "循环中的变量复用利用了什么？" → 时间局部性

## 1.7 计算机系统结构的发展

### 1.7.1 冯诺依曼结构

**六大特点**：

1. 以**运算器**为中心
2. 指令和数据同等对待（程序可被修改）
3. 存储器按地址访问、线性编址、固定位数
4. 指令顺序执行（通过PC控制）
5. 指令由操作码和地址码组成
6. 指令和数据以二进制表示

**核心思想**：**存储程序**概念（Stored-Program）——程序和数据存放在同一存储器中

**改进**：
- I/O方式改进（直接控制 → 通道 → I/O处理机）
- 存储器组织改进（相联存储器CAM、寄存器堆、Cache）
- 并行处理技术的应用
- 指令集发展（CISC和RISC）

### 1.7.2 哈弗结构 vs 冯诺依曼结构

| 冯诺依曼结构 | 哈弗结构 |
|-------------|---------|
| 指令和数据共享同一存储器、同一条总线 | 指令和数据分开存储、分开总线 |
| 结构简单 | 可以同时取指和取数 |
| 可能出现"冯诺依曼瓶颈" | 现代DSP常用 |

### 1.7.3 软件兼容性

**软件可移植性**的实现方法：
1. **系列机**（同一厂家、相同系统结构、不同组成实现）
2. **模拟与仿真**
3. **统一高级语言**

**兼容性分类**：

| 方向 | 含义 |
|------|------|
| 向上兼容 | 低档机程序可在高档机运行 |
| 向下兼容 | 高档机程序可在低档机运行 |
| 向前兼容 | 早期型号程序可在后续型号运行 |
| 向后兼容 | **← 这是系列机的根本特征！** 后续型号程序可在早期型号运行 |

**模拟 (Simulation)**：用软件（解释方式）实现另一台机器的指令集，速度慢

**仿真 (Emulation)**：用微程序（固件）实现另一台机器的指令集，速度快于模拟但仅限于相似体系结构

### 1.7.4 摩尔定律

集成电路上可容纳的晶体管数目约每**18个月**翻一番。

## 1.8 并行性

### 1.8.1 并行性与并发性

- **并行性**（Parallelism）：两个或多个事件在**同一时刻**发生
- **并发性**（Concurrency）：两个或多个事件在**同一时间间隔内**发生

### 1.8.2 并行性等级

**从数据处理角度**（由低到高）：

| 等级 | 名称 | 说明 |
|------|------|------|
| 1 | 字串位串 | 无并行（最原始的串行处理） |
| 2 | 字串位并 | 开始出现并行（传统单处理器） |
| 3 | 字并位串 | 位片处理，并行度较高 |
| 4 | 全并行 | 同时处理多个字的所有位（阵列处理机） |

**从程序执行角度**（由低到高）：
1. 指令内部并行（微指令级）
2. **指令级并行（ILP）** ← 第3/4章重点
3. 线程级并行（TLP）
4. 任务/进程级并行
5. 作业/程序级并行

### 1.8.3 提高并行性的三种技术途径

| 途径 | 含义 | 单机系统举例 | 多机系统举例 |
|------|------|------------|------------|
| **时间重叠** | 让多个过程在时间上错开，重叠使用同一套硬件 | 流水线技术 | 异构多处理机 |
| **资源重复** | 重复设置硬件资源 | 多模块存储器、多个功能部件 | 同构多处理机 |
| **资源共享** | 多个任务分时使用同一资源 | 分时系统 | 分布式系统 |

### 1.8.4 耦合度

| 类型 | 连接方式 | 共享资源 | 举例 |
|------|---------|---------|------|
| **紧耦合** | 总线/高速开关 | 共享主存 | SMP多处理机 |
| **松耦合** | 通道/通信线路 | 共享外存 | 机群系统 |

### 1.8.5 同构与异构多处理机

- **异构多处理机**：各处理机功能不同，体现**时间重叠**
- **同构多处理机**：各处理机功能相同，体现**资源重复**

## 1.9 关键概念辨析

### 1.9.1 数据表示 vs 数据结构

| 数据表示 | 数据结构 |
|---------|---------|
| 硬件能直接识别和处理的数据类型 | 软件处理的数据类型 |
| 例：定点数、浮点数 | 例：链表、树、图 |
| 由系统结构决定 | 由算法和编程决定 |

### 1.9.2 系列机 vs 兼容机

| 系列机 | 兼容机 |
|--------|--------|
| 同一厂家生产 | 不同厂家生产 |
| 相同系统结构、不同组成/实现 | 相同系统结构 |
| 例：Intel Core i3/i5/i7 | 例：AMD处理器兼容x86 |

## 1.11 课后思考题参考答案要点

1. **翻译和解释的特点**：翻译一次性将全部L+1程序转为L级程序再执行；解释逐条译码立即执行
2. **透明性的好处**：简化上层编程模型，使程序员不必关心底层实现细节
3. **系统结构/组成/实现的关系**：系统结构是软硬件接口（概念），组成是逻辑实现，实现是物理实现。例如乘法指令属于系统结构，用加法器和移位器循环实现乘法属于组成，用哪种工艺的门电路属于实现
4. **Flynn分类依据**：指令流和数据流的多倍性
5. **Amdahl定律指导意义**：系统整体加速受限于可改进部分的比例，提示我们要优化频繁发生的情况
6. **CPU时间公式各因素关联**：IC受编译器和指令集影响，CPI受组成和指令集影响，时钟周期受工艺和组成影响；三者相互制约非独立
7. **局部性原理的应用**：Cache利用时间+空间局部性，预取利用空间局部性，循环优化利用时间局部性
