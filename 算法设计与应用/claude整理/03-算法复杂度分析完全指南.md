## 一、渐近符号的严格定义与使用

### 1.1 五个符号的定义 🔴

设 $t(n)$ 和 $g(n)$ 是定义在非负整数集上的函数，$g(n) > 0$。

#### O(g(n)) —— 上界（"最多是"）

$$t(n) \in O(g(n)) \iff \exists c > 0, \exists n_0 \geq 0,\ \forall n \geq n_0:\ t(n) \leq c \cdot g(n)$$

**含义**：$t(n)$ 的增长**不超过** $g(n)$（最多同阶）

**例子**：
- $10n^2 \in O(n^2)$ ✅（取c=10, n₀=1）
- $10n \in O(n^2)$ ✅（取c=1, n₀=10）
- $n^3 \notin O(n^2)$ ❌

#### Ω(g(n)) —— 下界（"至少是"）

$$t(n) \in \Omega(g(n)) \iff \exists c > 0, \exists n_0 \geq 0,\ \forall n \geq n_0:\ t(n) \geq c \cdot g(n)$$

**含义**：$t(n)$ 的增长**不低于** $g(n)$（至少同阶）

**例子**：
- $10n^2 \in \Omega(n^2)$ ✅
- $10n^3 \in \Omega(n^2)$ ✅
- $n \notin \Omega(n^2)$ ❌

#### Θ(g(n)) —— 精确界（"就是"）

$$t(n) \in \Theta(g(n)) \iff \exists c_1, c_2 > 0, \exists n_0 \geq 0,\ \forall n \geq n_0:\ c_1 g(n) \leq t(n) \leq c_2 g(n)$$

**含义**：$t(n)$ 的增长**恰好等于** $g(n)$（同阶）

**例子**：
- $10n^2 \in \Theta(n^2)$ ✅（$c_1=1, c_2=10$）
- $10n^2 + 5n \in \Theta(n^2)$ ✅（低阶项被吸收）
- $10n \notin \Theta(n^2)$ ❌

#### o(g(n)) —— 严格上界（"真正小于"）

$$t(n) \in o(g(n)) \iff \forall c > 0, \exists n_0 \geq 0,\ \forall n \geq n_0:\ t(n) < c \cdot g(n)$$

也可以用极限：$\lim_{n \to \infty} \frac{t(n)}{g(n)} = 0$

**例子**：$n \in o(n^2)$，$n^2 \notin o(n^2)$

#### ω(g(n)) —— 严格下界（"真正大于"）

$$t(n) \in \omega(g(n)) \iff \forall c > 0, \exists n_0 \geq 0,\ \forall n \geq n_0:\ t(n) > c \cdot g(n)$$

也可以用极限：$\lim_{n \to \infty} \frac{t(n)}{g(n)} = \infty$

**例子**：$n^2 \in \omega(n)$，$n \notin \omega(n)$

### 1.2 符号记忆法 🔴

| 符号 | 类比 | 严格的？ | 记忆 |
|------|------|----------|------|
| O | ≤ | 否（可以等于） | Oh, 最多 |
| Ω | ≥ | 否（可以等于） | Omega, 至少 |
| Θ | = | 否（在这个集合里） | Theta, 就是 |
| o | < | **是**（严格小于） | 小o, 真的少 |
| ω | > | **是**（严格大于） | 小omega, 真的多 |

### 1.3 渐近符号的性质（O符号为例，Ω和Θ类似）

1. **反身性**：$f(n) \in O(f(n))$
2. **传递性**：$f(n) \in O(g(n)), g(n) \in O(h(n)) \Rightarrow f(n) \in O(h(n))$
3. **互对称性**：$f(n) \in O(g(n)) \iff g(n) \in \Omega(f(n))$
4. **对称性**：$f(n) \in \Theta(g(n)) \iff g(n) \in \Theta(f(n))$
5. **加法**：$O(g_1(n)) + O(g_2(n)) = O(g_1(n) + g_2(n))$
6. **乘法**：$O(g_1(n)) \cdot O(g_2(n)) = O(g_1(n) \cdot g_2(n))$
7. **常数可忽略**：$O(c \cdot f(n)) = O(f(n))$
8. **吸收律**：若 $g(n) = O(f(n))$，则 $O(f(n)) + O(g(n)) = O(f(n))$

### 1.4 常见用法和考试陷阱

**正确用法**：
- $T(n) = O(n^2)$ ✅（表示上界）
- 合并排序的最差时间是 $\Theta(n \log n)$ ✅（精确界）
- 对所有算法，$T(n) = \Omega(1)$ ✅（平凡下界）

**易错点**：
- 不能说"该算法的复杂度是 $O(n^2)$"，而应该说"该算法的**最差/平均/最佳**复杂度是 $O(n^2)$"
- $O(n^2) = O(n^3)$ 是错的！应该是 $O(n^2) \subseteq O(n^3)$，或者 $n^2 \in O(n^3)$
- 不能写成 $T(n) \leq O(n^2)$（O已经是上界了，不需要再加≤）

### 1.5 判断符号关系的方法

判断 $t(n)$ 与 $g(n)$ 的关系：

1. 计算极限 $\lim_{n \to \infty} \frac{t(n)}{g(n)}$
2. 根据结果判断：
   - $= 0$：$t(n) \in O(g(n))$ 且 $t(n) \in o(g(n))$
   - $= c > 0$：$t(n) \in \Theta(g(n))$（同时也 $\in O$ 和 $\in \Omega$）
   - $= \infty$：$t(n) \in \Omega(g(n))$ 且 $t(n) \in \omega(g(n))$

## 二、增长阶数比较方法

### 2.1 标准增长阶数序列（从小到大）⭐

$$1 < \log n < \log^2 n < \sqrt{n} < n < n\log n < n^2 < n^3 < 2^n < 3^n < n! < n^n$$

### 2.2 极限法判断 🔴

核心公式：
$$\lim_{n \to \infty} \frac{t(n)}{g(n)} = \begin{cases} 0 & \Rightarrow t(n) \text{ 增长比 } g(n) \text{ 慢} \\ c > 0 & \Rightarrow \text{两者同阶} \\ \infty & \Rightarrow t(n) \text{ 增长比 } g(n) \text{ 快} \end{cases}$$

### 2.3 必须记住的极限

1. $\lim_{n \to \infty} \frac{\log n}{n^\alpha} = 0$（$\alpha > 0$）：对数慢于任何多项式
2. $\lim_{n \to \infty} \frac{n^\alpha}{a^n} = 0$（$a > 1$）：多项式慢于指数
3. $\lim_{n \to \infty} \frac{a^n}{n!} = 0$：指数慢于阶乘
4. $\lim_{n \to \infty} \frac{n!}{n^n} = 0$：阶乘慢于n^n

**记忆**：对数 < 多项式 < 指数 < 阶乘 < n^n

### 2.4 典型判断例题

**例1**：判断 $n!$ 与 $n^n$ 的关系

$$\lim_{n \to \infty} \frac{n!}{n^n} = \lim_{n \to \infty} \frac{1 \cdot 2 \cdot 3 \cdots n}{n \cdot n \cdot n \cdots n} = \lim_{n \to \infty} \frac{1}{n} \cdot \frac{2}{n} \cdot \ldots \cdot \frac{n}{n} = 0$$

所以 $n! \in o(n^n)$，即 $n! < n^n$（增长意义上）

**例2**：比较 $\log_2 n$ 和 $\sqrt{n}$

$$\lim_{n \to \infty} \frac{\log_2 n}{\sqrt{n}} = 0$$

所以 $\log_2 n \in o(\sqrt{n})$

**例3**：判断以下哪些是 $O(n^2)$？
- $100n^2$ → ✅ ∈ $\Theta(n^2)$ → ∈ $O(n^2)$
- $n^3$ → ❌ ∈ $\omega(n^2)$ → ∉ $O(n^2)$
- $n \log n$ → ✅ ∈ $o(n^2)$ → ∈ $O(n^2)$
- $2^n$ → ❌ ∈ $\omega(n^2)$ → ∉ $O(n^2)$

### 2.5 利用斯特林公式

斯特林近似（判断阶乘相关极限时很有用）：
$$n! \approx \sqrt{2\pi n}\left(\frac{n}{e}\right)^n$$

由此可得 $\log(n!) \approx n \log n - n \in \Theta(n \log n)$

## 三、非递归算法分析（5步法）

### 3.1 标准五步 🔴

```
步骤1：确定表示输入规模的参数n
步骤2：找出算法的基本操作
步骤3：检查是否只依赖于输入规模（否则分别研究最差/平均/最佳）
步骤4：建立求和表达式
步骤5：利用求和公式化简，确定增长阶数
```

### 3.2 经典例题

**例题1：找最大元素**

```
Algorithm MaxElement(A[0..n-1])
    maxval ← A[0]
    for i ← 1 to n-1 do
        if A[i] > maxval then
            maxval ← A[i]
    return maxval
```

**分析**：
- 步骤1：输入规模 = n（数组长度）
- 步骤2：基本操作 = 比较 `A[i] > maxval`（每次循环执行）
- 步骤3：对于任何规模n的数组，比较次数相同
- 步骤4：$C(n) = \sum_{i=1}^{n-1} 1 = n - 1$
- 步骤5：$C(n) \in \Theta(n)$

**例题2：元素唯一性**

```
Algorithm UniqueElements(A[0..n-1])
    for i ← 0 to n-2 do
        for j ← i+1 to n-1 do
            if A[i] = A[j] return false
    return true
```

**分析**：
- 步骤1：输入规模 = n
- 步骤2：基本操作 = 比较 `A[i] = A[j]`
- 步骤3：最差情况（无重复） → 全部遍历
- 步骤4：$C_{worst}(n) = \sum_{i=0}^{n-2} \sum_{j=i+1}^{n-1} 1$
- 步骤5：
  $$\sum_{i=0}^{n-2} (n-1-i) = (n-1) + (n-2) + \ldots + 1 = \frac{n(n-1)}{2} \in \Theta(n^2)$$

**例题3：矩阵乘法（蛮力法）**

```
Algorithm MatrixMultiplication(A[0..n-1, 0..n-1], B[0..n-1, 0..n-1])
    for i ← 0 to n-1 do
        for j ← 0 to n-1 do
            C[i,j] ← 0
            for k ← 0 to n-1 do
                C[i,j] ← C[i,j] + A[i,k] * B[k,j]
    return C
```

**分析**：
- 步骤1：输入规模 = n（矩阵维度）
- 步骤2：基本操作 = 乘法 $A[i,k] \cdot B[k,j]$（最内层循环体）
- 步骤3：无论输入值如何，执行次数相同
- 步骤4：$M(n) = \sum_{i=0}^{n-1} \sum_{j=0}^{n-1} \sum_{k=0}^{n-1} 1 = n^3$
- 步骤5：$M(n) \in \Theta(n^3)$

**例题4：二进制位数**

```
Algorithm Binary(n)
    count ← 1
    while n > 1 do
        count ← count + 1
        n ← ⌊n/2⌋
    return count
```

**分析**：
- 步骤1：输入规模 = n（数值大小）
- 步骤2：基本操作 = 比较 `n > 1`
- 步骤3：仅取决于n
- 步骤4：循环每次将n减半，大约执行 $\lfloor \log_2 n \rfloor + 1$ 次
- 步骤5：$C(n) \in \Theta(\log n)$

## 四、递归算法分析（5步法）

### 4.1 标准五步 🔴

```
步骤1：确定表示输入规模的参数
步骤2：找出算法的基本操作
步骤3：检查相同规模不同输入的影响（分别研究最差/平均/最佳）
步骤4：建立递推关系式 + 初始条件
步骤5：解递推关系式（反向替换法/主定理/特征方程法）
```

### 4.2 经典例题

**例题1：阶乘递归**

```
Algorithm F(n)
    if n = 0 return 1
    else return F(n-1) * n
```

**分析**：
- 步骤1：输入规模 = n
- 步骤2：基本操作 = 乘法
- 步骤3：每次都相同
- 步骤4：$M(n) = M(n-1) + 1$，$M(0) = 0$
- 步骤5（反向替换）：
  $$M(n) = M(n-1) + 1 = M(n-2) + 2 = \ldots = M(0) + n = n \in \Theta(n)$$

**例题2：汉诺塔** ⭐

```
Algorithm Hanoi(n, from, to, aux)
    if n = 1 then
        move disk from from to to
    else
        Hanoi(n-1, from, aux, to)
        move disk from from to to
        Hanoi(n-1, aux, to, from)
```

**分析**：
- 步骤1：输入规模 = n（盘子数）
- 步骤2：基本操作 = 移动盘子
- 步骤4：$M(n) = M(n-1) + 1 + M(n-1) = 2M(n-1) + 1$，$M(1) = 1$
- 步骤5（反向替换）：
  $$M(n) = 2M(n-1) + 1$$
  $$= 2(2M(n-2) + 1) + 1 = 4M(n-2) + 3$$
  $$= \ldots = 2^{n-1}M(1) + \sum_{i=0}^{n-2}2^i$$
  $$= 2^{n-1} + (2^{n-1} - 1) = 2^n - 1 \in \Theta(2^n)$$

**例题3：斐波那契（递归版）** ⭐

递推关系：$F(n) = F(n-1) + F(n-2)$，$F(0) = 0$，$F(1) = 1$

递归算法（低效版）的调用次数：
- 基本操作：加法
- $A(n) = A(n-1) + A(n-2) + 1$，$A(0) = 0$，$A(1) = 0$
- $A(n) \in \Theta(\phi^n)$，其中 $\phi = \frac{1+\sqrt{5}}{2} \approx 1.618$

**为什么递推计算斐波那契这么低效？**
因为存在**大量重叠子问题**。例如，F(5)需要计算F(4)和F(3)，而F(4)又需要F(3)和F(2)。F(3)被重复计算了两次！
用DP可以将复杂度降至O(n)。

**例题4：折半查找（递归版）**

```
Algorithm BinarySearchRec(A[l..r], K)
    if l > r return -1
    m ← ⌊(l+r)/2⌋
    if K = A[m] return m
    else if K < A[m]
        return BinarySearchRec(A[l..m-1], K)
    else
        return BinarySearchRec(A[m+1..r], K)
```

**分析（最差情况——查找失败）**：
- 递推关系：$C_{worst}(n) = C_{worst}(\lceil n/2 \rceil) + 1$，$C_{worst}(1) = 1$
- 设 $n = 2^k$：
  $$C_{worst}(2^k) = C_{worst}(2^{k-1}) + 1 = \ldots = C_{worst}(1) + k = k + 1 = \log_2 n + 1$$
- $C_{worst}(n) \in \Theta(\log n)$

## 五、主定理 (Master Theorem) 🔴

### 5.1 主定理公式

对于递推关系 $T(n) = aT(n/b) + f(n)$，其中 $a \geq 1, b > 1$，$f(n)$ 是渐近正函数。

若 $f(n) \in \Theta(n^d)$，$d \geq 0$，则：

$$T(n) = \begin{cases} \Theta(n^d) & \text{if } a < b^d \quad \text{(合并时间占主导)} \\ \Theta(n^d \log n) & \text{if } a = b^d \quad \text{(平摊)} \\ \Theta(n^{\log_b a}) & \text{if } a > b^d \quad \text{(递归占主导)} \end{cases}$$

### 5.2 使用步骤

```
第1步：从递推关系中提取 a（子问题个数）、b（规模除数）、d（f(n)中n的指数）
第2步：计算 b^d
第3步：比较 a 和 b^d
第4步：根据比较结果，套用对应情况
```

### 5.3 记忆技巧 🔴

- a = 分支数（有多少个子问题）
- b = 缩小的倍数（原规模除以几得到子问题规模）
- d = 合并工作的增长阶数指数，即 f(n) = Θ(n^d)

**直觉**：
- 如果递推树**底部**工作量大（a > b^d，很多子问题）→ 复杂度由递推树叶子层决定
- 如果**合并**工作量大（a < b^d）→ 复杂度由顶层合并决定
- 如果两者相当（a = b^d）→ 每层工作量相同，多一个log n因子

### 5.4 经典例题

**题1**：合并排序 $T(n) = 2T(n/2) + \Theta(n)$
- a=2, b=2, d=1
- $b^d = 2^1 = 2 = a$
- 情况2：$\Theta(n^1 \log n) = \Theta(n \log n)$ ✅

**题2**：折半查找 $T(n) = T(n/2) + \Theta(1)$
- a=1, b=2, d=0
- $b^d = 2^0 = 1 = a$
- 情况2：$\Theta(n^0 \log n) = \Theta(\log n)$ ✅

**题3**：Karatsuba $T(n) = 3T(n/2) + \Theta(n)$
- a=3, b=2, d=1
- $b^d = 2^1 = 2 < 3 = a$
- 情况3：$\Theta(n^{\log_2 3}) \approx \Theta(n^{1.585})$ ✅

**题4**：Strassen $T(n) = 7T(n/2) + \Theta(n^2)$
- a=7, b=2, d=2
- $b^d = 2^2 = 4 < 7 = a$
- 情况3：$\Theta(n^{\log_2 7}) \approx \Theta(n^{2.807})$ ✅

**题5**：$T(n) = 4T(n/2) + \Theta(n)$
- a=4, b=2, d=1
- $b^d = 2^1 = 2 < 4 = a$
- 情况3：$\Theta(n^{\log_2 4}) = \Theta(n^2)$

**题6**：$T(n) = 2T(n/2) + \Theta(n^2)$
- a=2, b=2, d=2
- $b^d = 2^2 = 4 > 2 = a$
- 情况1：$\Theta(n^2)$

**题7**：$T(n) = nT(n/2) + \Theta(n^2)$
- a=n（不是常数！），**主定理不适用**

### 5.5 主定理的适用范围和限制

**适用条件**：
- a和b必须是常数（不依赖于n）
- f(n)必须渐近正函数
- f(n)必须是多项式类型的（$\Theta(n^d)$）

**不适用的情况**：
- a不是常数（如 $a=n$）
- f(n)不是多项式形式（如 $f(n) = n \log n$，$f(n) = 2^n$）
- 递推关系不是 $T(n) = aT(n/b) + f(n)$ 的形式（如 $T(n) = T(n-1) + n$）

## 六、常用递推关系求解方法

### 6.1 反向替换法（Backward Substitution）

**适用于**：递推形式简单、有明确初始条件的情况。

**步骤**：
1. 不断将递推关系代入自身
2. 观察模式
3. 推导到初始条件
4. 化简得到闭合公式

### 6.2 特征方程法

**适用于**：常系数线性齐次/非齐次递推关系

例：斐波那契 $F(n) = F(n-1) + F(n-2)$

特征方程：$r^2 = r + 1$
解得：$r = \frac{1 \pm \sqrt{5}}{2}$
$F(n) = A \cdot r_1^n + B \cdot r_2^n$

利用初始条件 $F(0)=0, F(1)=1$ 确定A、B，得到Binet公式。

### 6.3 递推树法（Recursion Tree）

画出递推树，统计每层的工作量，然后求和。

**例**：$T(n) = 2T(n/2) + n$

```
            n               ← 第0层: n
         /     \
       n/2     n/2          ← 第1层: n
      /  \     /  \
    n/4 n/4 n/4 n/4         ← 第2层: n
    ...
```

每层工作量都是n，共有log₂n层，所以 $T(n) = n \log_2 n$

## 七、全部经典算法复杂度速查

### 7.1 按算法类型分类

**排序算法**：

| 算法 | 最佳 | 平均 | 最差 | 额外空间 | 稳定性 | 在位 |
|------|------|------|------|----------|--------|------|

## 八、考试题型与解题策略

### 8.1 题型一：判断渐近关系

**典型问法**："判断以下函数的O、Ω、Θ关系"，"$f(n) = O(g(n))$ 是否成立？"

**解题策略**：
1. 写极限 $\lim_{n \to \infty} \frac{f(n)}{g(n)}$
2. 利用已知极限结论
3. 得出结论

### 8.2 题型二：非递归算法复杂度分析

**典型问法**："分析以下算法的效率（确定基本操作、建立求和、求闭合式）"

**解题策略**：
1. 严格遵循5步法
2. 基本操作选**最内层循环**的操作
3. 检查是否需要区分最差/最佳/平均
4. 求和化简要仔细

### 8.3 题型三：递归算法复杂度分析

**典型问法**："建立递推关系并求解"

**解题策略**：
1. 严格遵循5步法
2. 递推关系 = 递归调用的复杂度 + 非递归部分的复杂度
3. 优先尝试主定理（如果适用）
4. 否则用反向替换法

### 8.4 题型四：主定理应用

**典型问法**："用主定理求以下递推关系的增长阶数"

**解题策略**：
1. 确认递推关系是 $T(n) = aT(n/b) + f(n)$ 的形式
2. 提取a, b, d，计算 $b^d$
3. 比较 a 和 $b^d$
4. 给出结论（注意是 $\Theta$ 不是 O）

### 8.5 常考递推关系总结

| 公式 | 解 | 对应算法 |
|------|-----|----------|
| $T(n)=T(n-1)+1$ | $\Theta(n)$ | 阶乘递归 |
| $T(n)=T(n-1)+n$ | $\Theta(n^2)$ | 插入排序最差 |
| $T(n)=2T(n-1)+1$ | $\Theta(2^n)$ | 汉诺塔 |
| $T(n)=T(n/2)+1$ | $\Theta(\log n)$ | 折半查找 |
| $T(n)=2T(n/2)+n$ | $\Theta(n\log n)$ | 合并排序 |
| $T(n)=2T(n/2)+n^2$ | $\Theta(n^2)$ | |
| $T(n)=3T(n/2)+n$ | $\Theta(n^{\log_2 3})$ | Karatsuba |
| $T(n)=7T(n/2)+n^2$ | $\Theta(n^{\log_2 7})$ | Strassen |
| $T(n)=T(n-1)+T(n-2)$ | $\Theta(\phi^n)$ | 斐波那契递归 |
| $T(n)=8T(n/2)+n^2$ | $\Theta(n^3)$ | 普通矩阵乘法分治 |
