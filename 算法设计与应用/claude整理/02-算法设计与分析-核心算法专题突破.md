## 一、蛮力法 (Brute Force)

### 1.1 什么时候用？

- 输入规模**非常小**
- 问题没有已知的高效算法
- 作为验证其他算法正确性的**基准**
- 某些更高效算法的基础（如DFS、BFS）

### 1.2 经典问题清单

| 问题 | 复杂度 | 关键点 |
|------|--------|--------|
| 选择排序 | $\Theta(n^2)$ | 无论输入如何，比较次数固定 |
| 冒泡排序 | $\Theta(n^2)$ | 可提前终止优化 |
| 顺序查找 | $O(n)$ | 最基础的查找 |
| 字符串匹配 | $O(nm)$ | 简单但效率低 |
| TSP穷举 | $O((n-1)!)$ | NP难问题 |
| 0-1背包穷举 | $\Omega(2^n)$ | NP难问题 |
| 分配问题 | $O(n!)$ | NP难问题 |

### 1.3 解题模板

```
// 穷举查找通用模板
生成所有可能的候选解
for each 候选解 do
    if 候选解满足约束条件 then
        if 候选解比当前最优更好 then
            更新当前最优
返回当前最优
```

### 1.4 关键理解

蛮力法的价值不在于高效，而在于：
1. 它是**最简单直接**的方法
2. 为更高效算法提供**比较基准**（至少要比蛮力强）
3. 对于小规模问题，蛮力法可能因**无额外开销**而比复杂算法更快

## 二、减治法 (Decrease and Conquer)

### 2.1 核心思想

$$f(n) \rightarrow f(n-k) \rightarrow \ldots \rightarrow f(1)$$

**每次将问题规模减小，解决减小后的问题，再扩展得到原问题的解。**

### 2.2 三类变体及解题识别

| 变体 | 减小方式 | 问题特征 | 典型算法 |
|------|----------|----------|----------|
| **减常数** (k=1) | n → n-1 | 已知第n-1个的解能方便推出第n个 | 插入排序、拓扑排序(源删除) |
| **减常因子** (k=n/b) | n → n/b | 问题可以不断折半 | 折半查找、假币问题、俄式乘法 |
| **减可变规模** | 每次减的量不同 | 每次减小的规模依赖于当前实例 | 欧几里得算法、选择第k小 |

### 2.3 递推关系分析模板

**减常数**：
$$T(n) = T(n-1) + f(n)$$
$$T(n) = T(0) + \sum_{j=1}^{n} f(j)$$

**减常因子**（设n = b^k）：
$$T(n) = T(n/b) + f(n)$$
$$T(b^k) = T(1) + \sum_{j=1}^{k} f(b^j)$$

### 2.4 拓扑排序的两种方法对比

| 特征 | DFS法 | 源删除法 |
|------|-------|----------|
| 策略 | DFS后逆序出栈 | 反复删除入度为0的顶点 |
| 数据来源 | 出栈顺序的逆序 | 入度为0的顶点 |
| 设计技术 | 图遍历 | 减治法（减一法） |
| 复杂度 | O(\|V\|+\|E\|) | O(\|V\|+\|E\|) |
| 判断无解 | 发现回边(有环) | 但还有顶点未删除(有环) |

### 2.5 插入排序深度理解

**为什么平均情况下比最差快约一倍？**
- 最差情况（逆序）：每次插入需要比较1+2+...+(n-1) = n(n-1)/2次
- 平均情况：一个随机元素插入到已排序的i元素数组中，平均比较次数约i/2
- 总和约n^2/4 ≈ 最差的一半

**为什么对几乎有序数组性能极好？**
- 内层while循环很快退出，接近线性时间

**为什么快速排序在小数组改用插入排序？**
- 小数组时，插入排序的常数因子小，且没有递归开销

## 三、分治法 (Divide and Conquer)

### 3.1 核心思想

$$T(n) = \underbrace{a \cdot T(n/b)}_{\text{递归求解子问题}} + \underbrace{f(n)}_{\text{分解+合并}}$$

### 3.2 分治法三步走 🔴

```
Divide:  将原问题划分成a个规模为n/b的子问题
Conquer: 递归求解每个子问题
Combine: 将子问题的解合并成原问题的解
```

### 3.3 主定理应用速查

| 算法 | a | b | d | $b^d$ | 比较 | 复杂度 | 情况 |
|------|---|---|---|------|------|--------|------|
| 合并排序 | 2 | 2 | 1 | 2 | a = b^d | $\Theta(n\log n)$ | 情况2 |
| 折半查找 | 1 | 2 | 0 | 1 | a = b^d | $\Theta(\log n)$ | 情况2 |
| 快速排序(最佳) | 2 | 2 | 1 | 2 | a = b^d | $\Theta(n\log n)$ | 情况2 |
| Karatsuba乘法 | 3 | 2 | 0 | 1 | a > b^d | $\Theta(n^{1.585})$ | 情况3 |
| Strassen乘法 | 7 | 2 | 0 | 1 | a > b^d | $\Theta(n^{2.807})$ | 情况3 |
| 快速排序(最差) | 2 | 2 | 1 | — | — | $\Theta(n^2)$ | 退化(T(n-1)+n) |

### 3.4 分治法经典问题深度解析

#### 合并排序 —— 分治的教科书案例

**为什么总能保证$\Theta(n \log n)$？**
因为合并排序**按位置**划分，子问题规模严格相等（或差1），不受数据分布影响。

**合并过程**需要多少个比较？
- 最差（两个子数组交替取出）：p+q-1次
- 最佳（一个数组元素全小于另一个）：min(p,q)次

**平滑法则**：如果n不是b的幂，可以将n"平滑"到下一个b的幂来分析复杂度。

#### 快速排序 —— 分治的另一种范式

**关键区别**：合并排序的划分是简单的（位置对半），主要工作在合并；快速排序的划分是复杂的（按值分区），但不需要合并。

**为什么平均$O(n \log n)$但最差$O(n^2)$？**
- 划分的是否均衡决定了递归树的深度
- 均衡划分 → 树深≈log n → $O(n \log n)$
- 极端不均衡 → 树深≈n → $O(n^2)$

**为什么实践中快速排序通常比合并排序快？**
1. 内层循环极简（只有比较和指针移动）
2. 在位排序，缓存友好
3. 平均情况下常数因子小（约1.39n log n vs 合并的~2n log n）

#### 大整数乘法 —— 减少乘法次数

**核心洞察**：
$$(A+B)(C+D) = AC + (AD+BC) + BD$$
所以 $AD+BC = (A+B)(C+D) - AC - BD$

从4次乘法减少到3次：AC, BD, (A+B)(C+D)

当m远小于n时（u有m位，v有n位，m << n）：
- 将v分成n/m段，每段m位
- 对每段进行大整数乘法
- 时间复杂度：$O(\frac{n}{m} \cdot m^{\log_2 3}) = O(n \cdot m^{\log_2 1.5})$

#### Strassen矩阵乘法 —— 减少子问题数量

将8次乘法减少到7次，通过精心设计的加减组合。理解Strassen的精神即可（减少子问题个数可以降低复杂度），不要求记住全部7个M_i的公式但需要知道复杂度分析。

### 3.5 分治法与减治法的区别 🔴

| 特征 | 分治法 | 减治法 |
|------|--------|--------|
| 子问题数量 | 多个（通常 ≥ 2） | 一个 |
| 是否需要合并 | ✅ 需要合并子问题的解 | ❌ 不需要（扩展即可） |
| 递推关系 | $T(n) = aT(n/b) + f(n), a \geq 2$ | $T(n) = T(n/b) + f(n)$ |
| 典型例子 | 合并排序、快速排序 | 折半查找、插入排序 |
| 交集 | 折半查找既是减治法也是分治法（a=1的分治） | |

## 四、变治法 (Transform and Conquer)

### 4.1 核心思想

**不是直接求解，而是先把问题变换成更容易的形式。**

### 4.2 三种变换策略对比

```
实例化简：  换个简单的输入  →  同一问题的不同实例
            例: [3,1,2] → [1,2,3]（预排序）

改变表现：  换个表达方式    →  同一实例的不同表示
            例: 数组 → 堆 / 多项式 → 嵌套式

问题化简：  换个问题        →  完全不同的已知问题
            例: 过河问题 → 图的状态空间搜索
```

### 4.3 堆 —— 最优雅的数据结构

**为什么自底向上构造堆是O(n)？**

数学证明：设堆高为h（h=⌊log₂n⌋）
- 第i层有$2^i$个节点，每个节点向下筛选最多h-i步
- 总操作量：$\sum_{i=0}^{h} 2^i \cdot (h-i) = 2^{h+1} - h - 2 = O(n)$

直观理解：底层节点多（约占一半）但几乎不需要筛选；顶层节点少但需要多步筛选。

**堆排序为什么稳定在Θ(n log n)？**
- 构造堆：O(n)
- 每次删除+筛选：O(log n)，重复n-1次
- 总计：$O(n) + (n-1) \cdot O(\log n) = \Theta(n \log n)$

### 4.4 高斯消去法的三个应用

1. **解线性方程组**：前向消去 + 回代
2. **LU分解**：高斯消去的过程等价于对矩阵做LU分解
3. **求行列式**：上三角矩阵的行列式 = 对角线元素之积

### 4.5 连续背包的线性规划模型

**连续背包**（物品可分割）：
- $\max \sum_{i=1}^{n} v_i x_i$
- $\text{s.t. } \sum_{i=1}^{n} w_i x_i \leq W$
- $0 \leq x_i \leq 1$

**0-1背包**（物品不可分割）：
- 同上，但约束变为 $x_i \in \{0, 1\}$
- 这使问题从P类变成了NP-hard类！

## 五、回溯法 (Backtracking)

### 5.1 核心思想

> **逐步构造解，一旦发现当前路径不可能导出解，立即回溯。**

### 5.2 何时用回溯法？

- 问题需要**搜索整个解空间**
- 解空间呈**指数级或阶乘级**增长
- 可以用**剪枝**显著减少搜索量
- 问题规模不太大（但可以用剪枝处理较大的n）

### 5.3 回溯法解题流程 🔴

```
第1步：确定解向量形式 → (x₁, x₂, ..., xₙ)
第2步：确定xᵢ的取值范围（显约束）
第3步：确定分量间的约束条件（隐约束）
第4步：选择解空间树类型（子集树 or 排列树）
第5步：设计约束函数和限界函数（剪枝）
第6步：编码实现（选择对应的框架模板）
```

### 5.4 子集树 vs 排列树 —— 如何选择？

**判断标准**：看解的结构。

- 如果解是**选或不选**每个元素 → **子集树**
  - 例：0-1背包（选/不选每个物品）
  - 例：子集和问题（选哪些数）
  
- 如果解是元素的**排列顺序** → **排列树**
  - 例：TSP（访问城市的顺序）
  - 例：n皇后（每行皇后放在第几列）
  - 例：圆排列问题（圆的排列顺序）

### 5.5 剪枝函数设计技巧

**约束函数**（Constraint）：
- 直接检查当前部分解是否违反约束条件
- 例如：n皇后中检查是否与已有皇后冲突

**限界函数**（Bound）：
- 估计从当前节点出发能获得的最优值
- 如果估计值不如已知最优解，则剪枝
- 例如：0-1背包中计算剩余物品的上界价值

**好的限界函数设计原则**：
1. 计算要快（否则得不偿失）
2. 尽可能紧（越接近真实最优值越好）
3. 不能过于乐观（否则可能剪掉真正的最优解）

### 5.6 回溯法效率分析

取决于以下因素：
- 生成下一个分量x[t]的时间
- 满足显约束的x[t]取值个数
- 约束函数的计算时间
- 限界函数的计算时间
- 满足约束和限界条件的x[t]的个数

**关键**：好的约束函数能**显著减少搜索节点数**，但如果计算代价太高，需要权衡。

## 六、分支限界法 (Branch and Bound)

### 6.1 与回溯法的本质区别

回溯法是在**深度优先**地找**所有解**。
分支限界法是在**广度优先/最佳优先**地找**一个最优解**。

**最关键的差异**：

| 方面 | 回溯法 | 分支限界法 |
|------|--------|------------|
| 活结点扩展 | 反复成为扩展节点 | **一次性**产生所有儿子后就不再扩展 |
| 节点选择 | DFS自然顺序 | 从活结点表中选择"最好"的 |
| 剪枝依据 | 约束函数 + 限界函数 | **计算函数值**（LB/UB），选最优方向 |
| 终止条件 | 搜索完整个解空间 | 活结点表为空或找到解 |

### 6.2 FIFO vs 优先队列分支限界

**FIFO**（队列式）：
- 按到达顺序处理
- 相当于BFS + 限界
- 较简单实现

**优先队列式**（LC搜索）：
- 按节点的**优先值**（如下界UB/上界LB）排序
- 始终先扩展"最有希望"的节点
- 通常**更快找到最优解**
- 需要维护优先队列

### 6.3 TSP问题分支限界法 —— 经典难点

**上界的计算（贪心法）**：
- 每次去最近的未访问城市
- 最后回到起点
- 得到一个可行解的长度（任意可行解都可作为上界）

**下界的计算（矩阵化简法）**：
- 每行减去该行最小值，每列减去该列最小值
- 所有减去的值之和就是下界
- 每确定一条边，矩阵缩小，重新计算下界

**剪枝条件**：LB ≥ 当前最优（上界）

### 6.4 0-1背包问题分支限界法

**上界计算**（用于优先队列的排序）：
```
remaining_value = 当前已选价值
remaining_weight = 容量 - 当前已选重量
for 每个剩余物品（按价值密度降序）:
    if 物品重量 ≤ remaining_weight:
        remaining_value += 物品价值
        remaining_weight -= 物品重量
    else:
        remaining_value += 物品价值/物品重量 * remaining_weight
        break
return remaining_value
```

**关键**：必须先按价值密度排序！

## 七、动态规划 (Dynamic Programming)

> 详细专题见 `05-动态规划与贪心算法专题.md`，此处仅列出核心框架。

### 7.1 DP适用性判断

一个问题能用DP需要同时满足：
1. **最优子结构**：最优解包含子问题的最优解
2. **重叠子问题**：递归树中存在大量重复子问题

### 7.2 DP四大解题步骤

```
步骤1: 定义状态（最优解的结构特征）
步骤2: 写出状态转移方程（递推关系 + 初始条件）
步骤3: 确定计算顺序（自底向上 or 自顶向下）
步骤4: 记录决策信息，构造最优解（如果需要）
```

### 7.3 常见状态定义模式

| 问题类型 | 典型状态定义 | 状态转移 |
|----------|-------------|----------|
| **序列对比** (LCS) | dp[i][j] = X前i个和Y前j个的LCS长度 | 取决于最后字符是否匹配 |
| **区间DP** (矩阵链乘) | dp[i][j] = 区间[i,j]的最优值 | 枚举分割点k |
| **背包型** | dp[i][j] = 前i个物品、容量j的最优值 | 选或不选第i个 |
| **路径型** (Floyd) | dp[k][i][j] = 用前k个节点作中间点的最短路 | 经过k或不经过k |
| **子段型** (最大子段和) | dp[i] = 以第i个元素结尾的最优值 | 接上前面的或自己独立 |
| **股票买卖型** | dp[i][k][0/1] = 第i天、交易k次、持有/未持有的最优值 | 买入/卖出/不操作 |

## 八、贪心法 (Greedy Algorithm)

> 详细专题见 `05-动态规划与贪心算法专题.md`。

### 8.1 贪心适用性判断

问题能用贪心需要：
1. **贪心选择性质**：每次局部最优选择能导出全局最优
2. **最优子结构**：做出贪心选择后剩下的子问题的最优解与原问题的最优解一致

### 8.2 贪心证明模板

**反证法**：
1. 假设问题存在一个不以贪心选择开始的最优解S
2. 将S中的第一个选择替换为贪心选择，得到S'
3. 证明S'也是可行解且不比S差
4. 因此贪心选择一定包含在某个最优解中

### 8.3 经典贪心算法的贪心选择

| 算法 | 贪心选择 | 证明关键 |
|------|----------|----------|
| Prim | 选择与当前树相连的最小权边 | 交换法：用更小边替换环中的大边 |
| Kruskal | 选择不构成回路的最小权边 | 交换法 |
| Dijkstra | 选择当前dist最小的未确定顶点 | dist最小的顶点不可能再被其他路径更新 |
| Huffman | 每次合并频率最小的两个节点 | 交换引理：最优树中最深层的两个节点频率最小 |
| 分数背包 | 按价值密度降序取物品 | 反证：如果最优解没有取满密度最高的物品，交换可更优 |

### 8.4 DP vs 贪心的核心区别（深层理解）

DP：**"我先看看子问题怎么解，再决定当前怎么做"**
贪心：**"我先做当前最好的选择，剩下的慢慢来"**

这导致了：
- DP必须**自底向上**（从子问题向大问题）
- 贪心可以**自顶向下**（当前选完，只要处理子问题）
- DP需要**存储所有子问题的解**（因为不确定哪个会被用到）
- 贪心只需要**一次选择**（因为已有确定的贪心准则）

## 九、算法设计技术全景对比

### 9.1 核心问题：一个给定问题该用什么技术？

| 问题特征 | 适用技术 | 理由 |
|----------|----------|------|
| 问题规模小，没已知高效算法 | 蛮力法 | 简单直接 |
| 子问题与原问题同类型，规模减小 | 减治法 | 利用规模间的关系 |
| 可以分解为多个独立同类子问题 | 分治法 | 分而治之，合并结果 |
| 通过变换可以简化 | 变治法 | 换个角度看问题 |
| 解空间指数级，需要搜索 | 回溯法/分支限界 | 剪枝减少搜索量 |
| 存在重叠子问题和最优子结构 | 动态规划 | 自底向上填表 |
| 局部最优能导出全局最优 | 贪心法 | 快速、简单但需正确性证明 |

### 9.2 时间复杂度层级图

```
O(1)        → 常数时间（如直接访问数组元素）
O(log n)    → 对数时间（如折半查找）
O(n)        → 线性时间（如顺序查找、霍纳法则）
O(n log n)  → 线性对数时间（如合并排序、快速排序平均、堆排序）
O(n²)       → 平方时间（如选择排序、插入排序最差）
O(n³)       → 立方时间（如Floyd算法、矩阵链乘DP、高斯消去）
O(2ⁿ)       → 指数时间（如汉诺塔、回溯法子集树）
O(n!)       → 阶乘时间（如TSP穷举、回溯法排列树）
```

### 9.3 考试中的技术判断题

**Q：以下问题用什么技术最合适？**

| 问题 | 最合适技术 | 为什么 |
|------|-----------|--------|
| 找数组最大元素 | 蛮力法（扫描） | 必须检查每个元素 |
| 在有序数组查找 | 折半查找（减常因子） | 每次排除一半 |
| 对随机数组排序（要快） | 快速排序（分治） | 平均最优 |
| 保证O(n log n)排序 | 合并排序/堆排序（分治/变治） | 最差也O(n log n) |
| 最小生成树 | Prim/Kruskal（贪心） | 贪心选择保证最优 |
| 0-1背包求最优解 | 动态规划 | 有最优子结构和重叠子问题 |
| 数独求解 | 回溯法 | 逐步填数字，冲突则回退 |
| 最优编码 | Huffman（贪心） | 贪心选择正确 |
| LCS | 动态规划 | 序列对比的标准方法 |
| 所有点对最短路径 | Floyd（DP） | DP自然适用 |
| 单源最短路径(无负权) | Dijkstra（贪心） | 贪心选择正确且高效 |
