## 第一章：数据库系统介绍

### 为什么需要DBMS？早期文件系统的缺陷

| 问题 | 解释 |
|------|------|
| 数据冗余和不一致 | 相同数据多份存储，更新困难 |
| 数据访问困难 | 每个新需求都要写新程序 |
| 数据孤立 | 多种文件格式无法统一访问 |
| 完整性问题 | 约束条件埋在程序代码里 |
| 原子性问题 | 操作失败导致数据库处于中间状态（如转账只扣不入） |
| 并发访问问题 | 多人同时修改导致不一致 |
| 安全性问题 | 难以精细控制访问权限 |

### 数据抽象三层（重要！）

视图层（View level）     ← 用户看到的（可隐藏敏感数据如工资）
逻辑层（Logical level）  ← 数据结构和关系（DBA关心）
物理层（Physical level） ← 数据如何实际存储在磁盘上

物理数据独立性（Physical Data Independence）：修改物理层不需要改逻辑层。

### 实例与模式

| 术语 | 类比 | 含义 |
|------|------|------|
| 逻辑模式（Logical Schema） | 变量类型 | 数据库的整体逻辑结构 |
| 物理模式（Physical Schema） | 存储方式 | 数据在磁盘上的结构 |
| 实例（Instance） | 变量的值 | 某时刻数据库的实际内容 |

### 数据库语言
- DDL（数据定义语言）：CREATE TABLE, DROP TABLE, ALTER TABLE
- DML（数据操纵语言）：SELECT, INSERT, UPDATE, DELETE

## 第二章：关系模型

### 核心术语（背这些英文！）

| 英文 | 中文 | 解释 |
|------|------|------|
| Relation | 关系/表 | 一张表 |
| Tuple | 元组 | 表中的一行 |
| Attribute | 属性 | 表中的一列 |
| Domain | 域 | 某属性允许的值的集合 |
| Relation Schema | 关系模式 | 表的结构，如 instructor(ID, name, dept_name, salary) |
| Relation Instance | 关系实例 | 某时刻表中的实际数据 |
| Null | 空值 | 表示"未知"，属于每个域 |

### 键（Keys）

| 类型 | 英文 | 含义 |
|------|------|------|
| 超码 | Superkey | 能唯一标识记录的属性集（可冗余），如{ID}和{ID,name}都是 |
| 候选码 | Candidate Key | 最小超码（去掉任何一个属性就不再是超码），如{ID} |
| 主码 | Primary Key | 从候选码中选一个作为主要标识 |
| 外码 | Foreign Key | 引用另一个关系主码的属性 |

### 关系代数六大基本操作

| 操作 | 符号 | 对应SQL | 说明 |
|------|------|---------|------|
| 选择（Select） | σ | WHERE | 筛选满足条件的行 |
| 投影（Project） | π | SELECT 列 | 选择某几列（自动去重） |
| 并（Union） | ∪ | UNION | 两个关系的所有元组（去重）|
| 差（Set Difference） | − | EXCEPT | 在A中但不在B中 |
| 笛卡尔积（Cartesian Product） | × | FROM A,B | 所有行的组合 |
| 重命名（Rename） | ρ | AS | 给关系或属性改名 |

派生操作：
- 连接（Join）= 笛卡尔积 + 选择
- 交（Intersection）= r ∩ s

注意：并、差、交要求两个关系相容（列数相同，对应属性域相同）

## 第三章：SQL基础

### SQL数据类型

| 类型 | 含义 |
|------|------|
| char(n) | 定长字符串 |
| varchar(n) | 变长字符串，最大n |
| int | 整数 |
| numeric(p,d) | 定点小数，总p位，小数点后d位 |
| real, double precision | 浮点数 |

### 建表（DDL）

```sql
CREATE TABLE instructor (
    ID        char(5),
    name      varchar(20) NOT NULL,
    dept_name varchar(20),
    salary    numeric(8,2),
    PRIMARY KEY (ID),
    FOREIGN KEY (dept_name) REFERENCES department
);

DROP TABLE r;                        -- 删除表（表和数据都没了）
DELETE FROM r;                       -- 删除所有数据但保留表结构
ALTER TABLE r ADD A varchar(20);     -- 添加列
ALTER TABLE r DROP A;                -- 删除列
```

### 基本查询

```sql
SELECT A1, A2, ...    -- 投影（SELECT DISTINCT 去重）
FROM   r1, r2, ...    -- 笛卡尔积
WHERE  P;             -- 选择

-- 重命名
SELECT salary / 12 AS monthly_salary FROM instructor;

-- 字符串匹配（LIKE）：% 匹配任意子串，_ 匹配任意单字符
SELECT name FROM instructor WHERE name LIKE '%dar%';

-- 排序：先按salary降序，相同时按name升序
ORDER BY salary DESC, name ASC;
```

### 集合操作

| 操作 | 说明 |
|------|------|
| UNION | 并，自动去重 |
| UNION ALL | 并，保留重复 |
| INTERSECT | 交 |
| EXCEPT | 差 |

### NULL值处理（重要！）

- 包含NULL的算术运算结果都是NULL
- 比较NULL必须用 IS NULL 或 IS NOT NULL，不能用 = NULL
- 三值逻辑：true > unknown > false；NOT unknown = unknown
- 聚集函数（除COUNT(*)外）忽略NULL值

### 聚集函数

```sql
SELECT dept_name, AVG(salary), COUNT(*), SUM(salary), MIN(salary), MAX(salary)
FROM instructor
GROUP BY dept_name
HAVING AVG(salary) > 42000;
```

WHERE vs HAVING：
- WHERE：GROUP BY 之前过滤（针对每行）
- HAVING：GROUP BY 之后过滤（针对每组）
- SELECT和HAVING里只能出现GROUP BY的属性 + 聚集函数

### 嵌套子查询

```sql
-- IN / NOT IN
WHERE dept_name IN ('Physics', 'Comp. Sci.')

-- SOME：至少一个满足
WHERE salary > SOME (SELECT salary FROM instructor WHERE dept_name = 'Biology')

-- ALL：所有都满足
WHERE salary > ALL (SELECT salary FROM instructor WHERE dept_name = 'Comp. Sci.')

-- EXISTS：子查询非空则为真
WHERE EXISTS (SELECT * FROM section AS T WHERE S.course_id = T.course_id ...)

-- WITH：定义临时关系
WITH max_budget(value) AS (SELECT MAX(budget) FROM department)
SELECT dept_name FROM department, max_budget WHERE budget = max_budget.value;
```

标量子查询：返回单一值，可出现在任何需要单值的地方。

### 数据修改

```sql
INSERT INTO instructor VALUES ('10211', 'Smith', 'Biology', 66000);
DELETE FROM instructor WHERE dept_name = 'Finance';
UPDATE instructor SET salary = salary * 1.05 WHERE salary < 70000;

-- CASE更新
UPDATE instructor SET salary = CASE
    WHEN salary <= 100000 THEN salary * 1.05
    ELSE salary * 1.03
END;
```

## 第四章：中级SQL

### 连接表达式

Natural Join（自然连接）：自动对同名属性等值连接，结果中只保留一份同名属性。
危险！同名属性可能语义不同，导致错误结果。

```sql
-- 自然连接
SELECT name, course_id FROM student NATURAL JOIN takes;

-- USING（指定连接属性，避免同名异义）
SELECT name, title FROM (student NATURAL JOIN takes) JOIN course USING (course_id);

-- ON（最灵活）
SELECT * FROM student JOIN takes ON student.ID = takes.ID;
```

### Outer Join（外连接）—— 重点！

| 类型 | 效果 |
|------|------|
| LEFT OUTER JOIN | 保留左表所有行，右边无匹配的填NULL |
| RIGHT OUTER JOIN | 保留右表所有行，左边无匹配的填NULL |
| FULL OUTER JOIN | 两边都保留，无匹配的填NULL |
| INNER JOIN | 只保留匹配行（默认，JOIN = INNER JOIN）|

ON vs WHERE 在外连接中的区别（易错！）：
- ON：非匹配元组保留，用NULL填充
- WHERE：把NULL行过滤掉，等同于内连接

```sql
-- 保留所有课程（没有前置课的也保留，prereq属性为NULL）
SELECT * FROM course LEFT OUTER JOIN prereq ON course.course_id = prereq.course_id;
```

### 视图（Views）

```sql
-- 创建视图（虚关系，不真实存储）
CREATE VIEW faculty AS SELECT ID, name, dept_name FROM instructor;

-- 使用
SELECT name FROM faculty WHERE dept_name = 'Biology';

-- 带自定义列名
CREATE VIEW dept_total_salary(dept_name, total_salary) AS
    SELECT dept_name, SUM(salary) FROM instructor GROUP BY dept_name;
```

物化视图（Materialized View）：物理存储，底层更新时需要维护同步。

视图可更新的条件（同时满足）：
- FROM中只有一个基本表
- SELECT中只有属性名（无表达式、无聚集、无DISTINCT）
- 没有GROUP BY或HAVING

### 事务（Transactions）

COMMIT WORK;    -- 提交，更改永久生效
ROLLBACK WORK;  -- 回滚，撤销所有更改

原子性：要么全做，要么全不做（如转账）。

### 完整性约束

```sql
NOT NULL
PRIMARY KEY (A, B)
UNIQUE (A, B)          -- 候选码，允许NULL（主码不允许NULL）
CHECK (salary >= 0)
CHECK (semester IN ('Fall', 'Winter', 'Spring', 'Summer'))

-- 外码参照完整性
FOREIGN KEY (dept_name) REFERENCES department
    ON DELETE CASCADE       -- 父表删除时级联删除
    ON UPDATE CASCADE       -- 父表更新时级联更新
    -- 其他选项：SET NULL, SET DEFAULT

-- 断言（全局约束）
CREATE ASSERTION <name> CHECK (<predicate>);
```

### SQL扩展数据类型

date '2005-7-27'
time '09:00:30'
timestamp '2005-7-27 09:00:30.75'
interval '1' day

A clob(10KB)    -- 字符大对象
A blob(1GB)     -- 二进制大对象（图片视频等），查询返回指针而不是对象本身

-- 自定义类型
CREATE TYPE Dollars AS numeric (12,2) FINAL;

-- 自定义域（可设约束，区别于类型）
CREATE DOMAIN degree_level VARCHAR(10)
    CONSTRAINT test CHECK (value IN ('Bachelors', 'Masters', 'Doctorate'));

类型 vs 域：域可以设约束和默认值；相同基本类型的域可以互相赋值，不同类型不行。

### 索引

CREATE INDEX studentID_index ON student(ID);  -- 快速查找，避免全表扫描

### 授权

```sql
-- 授予权限
GRANT SELECT ON department TO Amit, Satoshi;
GRANT SELECT ON department TO PUBLIC;                        -- 所有用户
GRANT SELECT ON department TO Amit WITH GRANT OPTION;        -- 允许再授权

-- 收回权限
REVOKE SELECT ON student FROM U1, U2, U3;
REVOKE SELECT ON department FROM Amit CASCADE;               -- 级联收权
REVOKE SELECT ON department FROM Amit RESTRICT;              -- 不级联
REVOKE GRANT OPTION FOR SELECT ON department FROM Amit;      -- 只收回再授权的权力

-- 角色
CREATE ROLE instructor;
GRANT SELECT ON takes TO instructor;
GRANT instructor TO Amit;     -- 把角色授予用户
GRANT instructor TO dean;     -- 角色可以继承

-- 视图授权（行级安全）
CREATE VIEW geo_instructor AS (SELECT * FROM instructor WHERE dept_name = 'Geology');
GRANT SELECT ON geo_instructor TO geo_staff;
```

## 第五章：高级SQL

### 函数和过程

```sql
-- 函数（有返回值）
CREATE FUNCTION dept_count(dept_name VARCHAR(20))
RETURNS INTEGER
BEGIN
    DECLARE d_count INTEGER;
    SELECT COUNT(*) INTO d_count FROM instructor
    WHERE instructor.dept_name = dept_count.dept_name;
    RETURN d_count;
END;

-- 表函数（返回一张表）
CREATE FUNCTION instructor_of(dept_name CHAR(20))
RETURNS TABLE (ID VARCHAR(5), name VARCHAR(20), dept_name VARCHAR(20), salary NUMERIC(8,2))
RETURN TABLE (SELECT ID, name, dept_name, salary FROM instructor
              WHERE instructor.dept_name = instructor_of.dept_name);
-- 调用：SELECT * FROM TABLE(instructor_of('Music'));

-- 过程（无返回值，用IN/OUT参数）
CREATE PROCEDURE dept_count_proc(IN dept_name VARCHAR(20), OUT d_count INTEGER)
BEGIN
    SELECT COUNT(*) INTO d_count FROM instructor
    WHERE instructor.dept_name = dept_count_proc.dept_name;
END;
-- 调用：CALL dept_count_proc('Physics', d_count);
```

控制结构：
```sql
WHILE 条件 DO 语句; END WHILE;
REPEAT 语句; UNTIL 条件 END REPEAT;
FOR r AS (SELECT ...) DO 语句; END FOR;

IF 条件 THEN 语句 ELSEIF 条件 THEN 语句 ELSE 语句 END IF;

-- 异常
DECLARE 异常名 CONDITION;
DECLARE EXIT HANDLER FOR 异常名 BEGIN ... END;
SIGNAL 异常名;   -- 触发异常
```

### 触发器（Triggers）

结构：触发时机（BEFORE/AFTER）+ 触发事件（INSERT/UPDATE/DELETE）+ 作用范围（FOR EACH ROW/STATEMENT）

```sql
-- BEFORE触发器：把空白成绩转为NULL（可修改新行）
CREATE TRIGGER setnull_trigger BEFORE UPDATE OF takes
REFERENCING NEW ROW AS nrow
FOR EACH ROW
WHEN (nrow.grade = ' ')
BEGIN ATOMIC
    SET nrow.grade = NULL;
END;

-- AFTER触发器：维护已修学分（维护其他表）
CREATE TRIGGER credits_earned AFTER UPDATE OF takes ON (grade)
REFERENCING NEW ROW AS nrow REFERENCING OLD ROW AS orow
FOR EACH ROW
WHEN nrow.grade <> 'F' AND nrow.grade IS NOT NULL
     AND (orow.grade = 'F' OR orow.grade IS NULL)
BEGIN ATOMIC
    UPDATE student SET tot_cred = tot_cred +
        (SELECT credits FROM course WHERE course.course_id = nrow.course_id)
    WHERE student.id = nrow.id;
END;
```

触发器陷阱：可能意外触发（备份恢复时）、级联触发、现代数据库有更好替代品。

### 递归查询

```sql
-- 找所有课程的传递先修课（transitive closure）
WITH RECURSIVE rec_prereq(course_id, prereq_id) AS (
    SELECT course_id, prereq_id FROM prereq           -- 基础情况
    UNION
    SELECT rec_prereq.course_id, prereq.prereq_id     -- 递归步骤
    FROM rec_prereq, prereq
    WHERE rec_prereq.prereq_id = prereq.course_id
)
SELECT * FROM rec_prereq;
```
结果叫不动点（Fixed Point），递归视图必须是单调的。

### 排名函数

```sql
-- RANK：有并列有空缺（1,1,3）
SELECT ID, RANK() OVER (ORDER BY GPA DESC) AS s_rank FROM student_grades;

-- DENSE_RANK：有并列无空缺（1,1,2）
SELECT ID, DENSE_RANK() OVER (ORDER BY GPA DESC) AS s_rank FROM student_grades;

-- 按系内排名（PARTITION BY）
SELECT ID, dept_name, RANK() OVER (PARTITION BY dept_name ORDER BY GPA DESC) AS dept_rank
FROM student_grades;

RANK() OVER (ORDER BY GPA DESC NULLS LAST)   -- NULL排最后
NTILE(4) OVER (ORDER BY GPA DESC)             -- 分4桶（四分位）
ROW_NUMBER() OVER (ORDER BY GPA DESC)         -- 唯一行号
```

### 窗口函数

```sql
-- 三日移动平均
SELECT date, SUM(value) OVER (ORDER BY date ROWS BETWEEN 1 PRECEDING AND 1 FOLLOWING)
FROM sales;

-- 累积余额
SELECT account_number, date_time,
    SUM(value) OVER (PARTITION BY account_number ORDER BY date_time ROWS UNBOUNDED PRECEDING) AS balance
FROM transaction;
```

### OLAP

- 上卷（Rollup）：细→粗（日→月→年）
- 下钻（Drill down）：粗→细
- 切片（Slice）：固定某一维的值
- 旋转（Pivot）：交换行/列维度

```sql
-- CUBE：对指定属性的所有子集GROUP BY（2^n种分组）
SELECT item_name, color, size, SUM(number)
FROM sales GROUP BY CUBE(item_name, color, size);

-- ROLLUP：只对前缀GROUP BY（n+1种分组）
SELECT item_name, color, size, SUM(number)
FROM sales GROUP BY ROLLUP(item_name, color, size);

-- grouping()：判断NULL是否是CUBE/ROLLUP产生的"all"（返回1表示是）
SELECT item_name, color, SUM(number), GROUPING(item_name), GROUPING(color)
FROM sales GROUP BY CUBE(item_name, color);
```

## 常见易错点

1. NULL不能用= NULL比较，要用IS NULL
2. WHERE vs HAVING：WHERE在GROUP BY之前，HAVING在GROUP BY之后
3. 外连接的ON vs WHERE：ON保留非匹配行填NULL，WHERE过滤掉NULL行（等同内连接）
4. RANK vs DENSE_RANK：RANK有间隔(1,1,3)，DENSE_RANK没有(1,1,2)
5. CUBE生成2^n种分组，ROLLUP生成n+1种分组
6. 视图更新限制：有聚集/DISTINCT/多表/GROUP BY的视图不能更新
7. 自然连接危险：同名属性不一定语义相同，建议用USING或ON
8. UNION去重，UNION ALL保留重复
9. 事务必须显式COMMIT才持久化
10. 触发器BEFORE可修改新行数据，AFTER用于维护其他表
