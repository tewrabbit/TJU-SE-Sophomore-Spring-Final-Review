## 六、考试常见陷阱（各章易错点）

### Ch6 易错点
1. **忘记弱实体的主码构成**：弱实体主码 = 标识实体主码 + 分辨符，不是自己的属性就够了
2. **多值属性忘记单独建表**：phone_number是多值 → 必须另外建 `inst_phone(ID, phone_number)` 表
3. **对"一"侧的全部参与无法用外键约束**：M:N的全部参与无法用SQL FK强制
4. **混淆total和disjoint**：total是关于"是否必须属于某个低层集"，disjoint是关于"能否属于多个"

### Ch7 易错点
1. **对分解后的子关系测试BCNF时只用F而不是F⁺**：必须用属性闭包方法检查子关系
2. **计算属性闭包时遗漏传递依赖**：闭包算法是迭代的，必须循环到不变
3. **同时删除多个无关属性**：每次只能删一个！删一个后另一个可能不再无关
4. **3NF分解时忘记加候选码表**：如果所有Rⱼ都不含候选码，必须额外加一个
5. **混淆无损和依赖保持**：无损条件(R₁∩R₂→R₁) ≠ 依赖保持条件(F₁∪F₂)⁺=F⁺
6. **正则覆盖使用F测试而非Fc**：测试无关属性必须用当前的Fc

### Ch8 易错点
1. **忘了SPARQL中三元组有"."分隔符**
2. **混淆JSON对象({})和数组([])**
3. **TF-IDF忘记取对数**
4. **PostGIS函数名记错**: ST_Contains不是Contains，ST_Distance不是Distance
