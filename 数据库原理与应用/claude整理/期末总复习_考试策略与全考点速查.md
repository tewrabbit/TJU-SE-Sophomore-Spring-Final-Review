## 二、全章节"一句话考点"速查

### Ch1: 数据库系统导论
- 文件系统的7个缺陷（冗余、访问难、孤立、完整性、原子性、并发、安全）
- 数据抽象三层：物理层→逻辑层→视图层
- 物理数据独立性 & 逻辑数据独立性
- 实例(Instance) vs 模式(Schema)：类比变量和类型
- DDL vs DML

### Ch2: 关系模型
- 关键术语：Relation, Tuple, Attribute, Domain, Schema, Instance, Null
- 键的层次：Superkey ⊃ Candidate Key ⊃ Primary Key，Foreign Key
- **关系代数六大操作**：σ, π, ∪, -, ×, ρ
- 关系代数去重（投影和并自动去重），SQL留重

### Ch3: SQL基础
- CREATE TABLE + 约束 (PK, FK, NOT NULL, CHECK, UNIQUE)
- SELECT-FROM-WHERE → 理解执行的逻辑顺序
- GROUP BY + HAVING + 聚集函数
- 嵌套子查询：IN, EXISTS, SOME, ALL, WITH
- NULL三值逻辑：IS NULL, IS NOT NULL
- 集合操作：UNION/INTERSECT/EXCEPT (自动去重) vs ALL版本

### Ch4: 中级SQL
- 自然连接的危害（同名异义）
- 外连接：LEFT/RIGHT/FULL（ON保留NULL行，WHERE过滤NULL行）
- 视图：可更新条件；物化视图
- 完整性约束：CASCADE, SET NULL, SET DEFAULT
- 授权：GRANT/REVOKE（CASCADE/RESTRICT），角色(ROLE)

### Ch5: 高级SQL
- 函数 vs 过程（返回值 vs CALL）
- 触发器：BEFORE可改数据，AFTER维护其他表
- RANK vs DENSE_RANK：有空缺 vs 无空缺
- CUBE(2^n) vs ROLLUP(n+1)
- 递归查询 WITH RECURSIVE → 不动点

### Ch6: E-R模型
- 实体集、联系集、属性
- 基数约束：有箭头=one，无箭头=many；双线=total
- 弱实体：双线矩形+双线菱形，主码=标识实体主码+分辨符
- ER转关系模式：多值属性单独建表，弱实体获得强实体主码
- 特化/泛化：Total/Partial, Disjoint/Overlapping

### Ch7: 关系数据库设计(规范化)
- 属性闭包α⁺ → 候选码 → 超码
- Armstrong公理：自反率、增补率、传递率
- 正则覆盖Fc：去除无关属性
- BCNF判定：每个FD的左边必须是超码
- BCNF分解：找违反的FD，按α→β分解(α∪β单独一个表)
- 3NF分解：用Fc建表+加候选码表
- 无损连接：R₁∩R₂→R₁或R₁∩R₂→R₂
- 依赖保持：(F₁∪F₂)⁺=F⁺
- MVD → 4NF

### Ch8: 复杂数据类型
- JSON对象`{}` vs 数组`[]`
- XML自描述标签
- RDF三元组：(主语, 谓语, 宾语)
- SPARQL：谓词可以是变量
- TF-IDF = TF × IDF
- PostGIS: ST_Contains, ST_Distance

### Ch9: 应用开发
- SQL注入 → PreparedStatement防御
- MVC架构：Model-View-Controller
- 三层架构：表示层→业务逻辑层→数据层
- 对称加密 vs 公钥加密
- 双因素认证

### Ch10: 大数据
- 3V: Volume, Velocity, Variety
- HDFS: NameNode(元数据) + DataNode(数据块，64MB，3副本)
- CAP定理：C和A只能选一个配合P
- MapReduce: map → shuffle → reduce
- NoSQL类型：K-V存储, 宽表, 文档存储

### Ch11: 数据分析
- OLAP操作：上卷/下钻/切片/旋转
- CUBE(2^n种分组) vs ROLLUP(n+1种分组)
- 关联规则：Support = P(A∪B), Confidence = P(B|A)
- 决策树：Gini = 1-Σp_i², Entropy = -Σp_i·log₂(p_i)
- 分类 vs 回归 vs 聚类

### Ch12: 物理存储
- 存储层次：Cache→主存→Flash→磁盘→磁带（速度遞减）
- 磁盘访问时间 = 寻道时间(4-10ms) + 旋转延迟(4-11ms)
- RAID级别：0(分条), 1(镜像), 4(专用校验), 5(分散校验), 6(双校验)
- SSD：页读，块擦除，FTL+Wear Leveling
- 电梯算法

### Ch13: 存储结构
- Slotted Page：槽数组+从尾向头增长的记录
- 四种组织：堆/顺序/多表聚集/分区
- 缓冲替换：LRU(不适合顺序扫描!), MRU, Toss-Immediate
- **行存储(OLTP) vs 列存储(OLAP)** 
- 数据字典(元数据存储)

### Ch14: 索引
- 聚集索引(最多1个) vs 辅助索引(可多个)
- 稠密索引 vs 稀疏索引（辅助索引必须是稠密！）
- **B+树操作：插入(分裂)和删除(合并/借位)**
- 散列：等值查询好，范围查询差
- B树 vs B+树：B+树扇出更大，有叶链表
- LSM树：顺序写代替随机写
- 位图索引：不同值少时用

### Ch15: 查询处理
- 代价公式：b_r×t_T + S×t_S
- 五种Join算法：嵌套循环→块嵌套→索引嵌套→归并连接→散列连接
- 外部排序：创建归并段+多路归并
- **物化 vs 流水线**
- 阻塞操作：排序、散列连接、聚集

### Ch17: 事务
- **ACID**(必背)：Atomicity, Consistency, Isolation, Durability
- 五个状态：Active→Partially Committed/→Failed→Aborted→Committed
- 冲突可串行化：优先图无环
- 四种隔离级别：Serializable > Repeatable Read > Read Committed > Read Uncommitted
- 死锁 vs 饥饿
- 级联回滚

## 三、跨章节高频对比考点

### 3.1 容易混淆的概念对

| 概念A | 概念B | 怎么区分 |
|------|------|---------|
| 聚集索引(Clustering) | 稠密索引(Dense) | 聚集=文件和索引同序；稠密=每个值都有索引项。两个维度！ |
| 主索引(Primary Index) | 主码(Primary Key) | 主索引=索引类型；主码=表约束。中文都叫"主" |
| 视图(View) | 物化视图(Materialized View) | 视图=存查询语句；物化视图=存结果数据 |
| WHERE | HAVING | WHERE在分组前；HAVING在分组后。WHERE不能用聚集函数 |
| UNION | UNION ALL | UNION去重；UNION ALL保留重复 |
| RANK | DENSE_RANK | RANK有间隔(1,1,3)；DENSE_RANK无间隔(1,1,2) |
| CUBE | ROLLUP | CUBE=所有子集(2^n种)；ROLLUP=前缀子集(n+1种) |
| 函数(Function) | 过程(Procedure) | 函数有返回值在表达式中调用；过程用CALL显式调用 |
| 冲突可串行化 | 视图可串行化 | 冲突⊂视图；视图包含盲写场景 |
| 死锁(Deadlock) | 饥饿(Starvation) | 死锁=互相等；饥饿=一直等不到 |
| B+树 | B树 | B+树数据指针只在叶节点(支持范围查询)；B树非叶节点也有 |
| 物化 | 流水线 | 物化=中间结果存盘；流水线=直接传递 |
| RAID 4 | RAID 5 | RAID 4专用校验盘(瓶颈)；RAID 5分散校验 |
| 逻辑数据独立性 | 物理数据独立性 | 逻辑=改逻辑模式不影响外模式；物理=改物理模式不影响逻辑模式 |
| 自然连接(Natural Join) | 等值连接(Inner Join ON) | 自然连接=自动用同名属性连接(危险)；ON=明确指定 |
| 选择σ | 投影π | 选择=选行(过滤)；投影=选列 |
| DDL | DML | DDL=定义结构(CREATE TABLE)；DML=操纵数据(SELECT/INSERT) |
| OLTP | OLAP | OLTP=事务处理(行存储)；OLAP=分析查询(列存储) |

### 3.2 关键数字记忆

| 知识点 | 数字 |
|------|------|
| B+树阶数n | 最大孩子数 |
| 叶节点最小值 | ⌈(n-1)/2⌉ |
| 非叶非根最小值 | ⌈n/2⌉ 个孩子 |
| B+树高度 | ≤ ⌈log_⌈n/2⌉(K)⌉ |
| CUBE分组数 | 2^n |
| ROLLUP分组数 | n+1 |
| HDFS块大小 | 64 MB(典型) |
| HDFS副本数 | 3(默认) |
| 磁盘扇区 | 512 字节 |
| SSD擦除次数 | 10万-100万次 |
| RAID 0最少盘数 | 2 |
| RAID 1最少盘数 | 2 |
| RAID 5最少盘数 | 3 |
| RAID 6最少盘数 | 4 |

## 五、答题技巧

### 英文简答题模板

**概念解释型**：
```
[Term] refers to [definition]. 
For example, [concrete example]. 
It is important because [reason].
```

**对比型**：
```
[A] and [B] differ in the following aspects:
1. [Aspect 1]: A is ... while B is ...
2. [Aspect 2]: A does ... whereas B does ...
Therefore, A is better for [scenario X], B is better for [scenario Y].
```

**判断可串行化型**：
```
Step 1: Identify conflicting operations on each data item.
Step 2: Draw precedence graph edges.
Step 3: The graph is acyclic / has a cycle ...
Step 4: Therefore, the schedule is / is not conflict serializable.
[If serializable] A possible serial order is T_a, T_b, T_c.
```

### 选择题技巧

1. **排立法**：先排除明显错误的选项（如"绝对""一定""只"等绝对性描述，多数是错的）
2. **概念对应**：每个选项逐一回忆课本对应知识点
3. **SQL阅读**：先在草稿纸上模拟执行SQL，注意NULL处理和去重
