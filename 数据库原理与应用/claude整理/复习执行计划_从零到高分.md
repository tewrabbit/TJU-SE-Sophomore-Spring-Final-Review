## 二、总体复习路线

```
Phase 1: SQL能力（最重要，约12小时）
   ↓
Phase 2: 数据库设计 — ER + 规范化（约10小时）
   ↓
Phase 3: 索引与存储（约8小时）
   ↓
Phase 4: 事务与查询处理（约6小时）
   ↓
Phase 5: 大数据与分析 + 应用开发（约4小时）
   ↓
Phase 6: 综合冲刺刷题（约6小时）
```

### 为什么这个顺序？
- SQL是"必考"且贯穿所有题目（连范式题都可能要求写SQL验证），所以第一优先
- ER+范式是"重点"，分值重且需要动手练习，放第二
- 索引也是"重点"，但概念比范式少，放第三
- 事务和查询处理是"会考"，有操作题（可串行化判断），放第四
- 大数据和分析"少但有"，概念记忆为主，放最后

**材料**: `6-8章复习/1-核心复习资料/Chapter6_ER_Model_Complete_Review.md`

### Step 2.2: ER图动手练习（1.5小时）

### Step 2.3: 看PPT对照（0.5小时）

**材料**: `PPT【U盘】/ch6.pptx`
快速浏览，看PPT里的例题。`课件扩展/第6章/` 里有补充材料。

### Step 2.4: 规范化精读（3小时）

**材料**: `6-8章复习/1-核心复习资料/Chapter7_Normalization_Complete_Review.md`

**按照这个顺序学，不要跳步**:
1. 先搞懂什么是函数依赖（FD）
2. 学 Armstrong 公理（自反、增补、传递——三条！）
3. **学属性闭包 α⁺ 算法**（这是所有后续操作的基础！）
4. 学候选码求解（用闭包方法）
5. 学正则覆盖 Fc（无关属性检测）
6. 学范式判定（BCNF: 每个FD左边是超码；3NF: 或属性在候选码中）
7. **学BCNF分解算法**（找违反FD→拆表→对子表重复）
8. **学3NF分解算法**（求Fc→建表→加候选码→去冗余）
9. 学无损连接判定（R₁∩R₂ → R₁ 或 R2）
10. 学MVD和4NF（了解即可）

**每学完一个算法立即动手算一遍，光看没用！**

### Step 2.5: 规范化动手练习（1.5小时）

**材料**:
1. `6-8章复习/2-原始资料/教材答案/Ch07_Relational_Database_Design/`

**必做题目**:
- 7.6 (求候选码)
- 7.30 (综合题)
- 7.33 (复杂3NF分解)

**检验标准**: 给一个R和F，能独立：
- [ ] 计算任意α的属性闭包
- [ ] 找出所有候选码
- [ ] 求正则覆盖Fc
- [ ] 判断R属于第几范式
- [ ] 做BCNF分解（证明无损）
- [ ] 做3NF分解（证明无损+依赖保持）

## 五、Phase 3: 索引与存储（约8小时）⭐⭐⭐⭐

> **目标**: 能解释索引类型差异；能独立完成B+树插入和删除

### Step 3.1: 索引精读（3小时）

**材料**: `复习指南_第14章_索引.md`

**重点**:
- 聚集/辅助索引区别（为什么辅助索引顺序扫描效率低？——2006A-Q7原题！）
- 稠密/稀疏索引区别（为什么辅助索引必须是稠密的？——2006B-Q7原题！）
- **B+树结构属性**（n是什么，叶节点min=⌈(n-1)/2⌉，非叶min=⌈n/2⌉个孩子）
- **B+树插入**（分裂——先⌈n/2⌉在原节点，其余去新节点）
- **B+树删除**（合并优先，借位其次）
- 散列vs顺序索引（等值→散列，范围→B+树）
- B树vs B+树

### Step 3.2: B+树操作练习（2小时）【必做！】

**这是整个考试中最需要动手练的操作题！**

1. 拿纸笔，设定一个n=4的B+树
2. 从空树开始，依次插入: Adams, Brandt, Califieri, Darwin, Einstein, Faraday
3. 然后依次删除: Faraday, Einstein, Darwin
4. 每一步画出树的结构

**然后再做一轮n=3的**（更难，因为分裂点不同）

**自测标准**: 给一个B+树（n=4），插入一个值后你能立刻画出结果；删除同理。

### Step 4.3: 查询处理（2小时）

**材料**: `复习指南_第15章_查询处理.md`

**重点**（这些是选择题高频考点）:
1. **五种Join算法**的代价对比（嵌套循环→块嵌套→索引嵌套→归并连接→散列连接）
2. 归并连接和散列连接的前提条件（都需要等值连接；归并要求已排序）

**记忆口诀**: 
- "嵌套循环最慢，散列归并最快"
- "等值要高效，归并或散列；不等值查索引，没有索引块嵌套"
- "流水不阻塞，排序必阻塞"

## 十一、关于PPT的使用策略

**不是每个PPT都要通读！**

| PPT | 什么时候看 |
|-----|-----------|
| ch1.pptx-ch5.pptx | **不用通读**。复习指南1-5已经覆盖了重点。只在某个概念不清楚时查找 |
| ch6.pptx | 学完ER核心复习后快速浏览（0.5h），看PPT例题 |
| ch7.pptx | 学完规范化核心复习后快速浏览（0.5h），确认知识点覆盖 |
| ch8.pptx | 快速浏览（0.5h），重点看JSON/XML/RDF部分 |
| ch9.pptx | 只看SQL注入和MVC部分，其余跳过 |
| ch10.pptx | 只看MapReduce、CAP、HDFS、NoSQL部分 |
| ch11.pptx | 只看OLAP操作、关联规则、决策树部分 |
| ch12.pptx | 只看RAID和磁盘性能部分 |
| ch13.pptx | 只看缓冲替换和列存储部分 |
| ch14.pptx | **重点看B+树图示部分**，PPT上的图比文字直观 |
| ch15.pptx | 只看Join算法对比 |
| ch17.pptx | 重点看ACID、状态图、可串行化部分 |

**总原则**: PPT是**字典**不是**课本**。复习指南看不懂时才查PPT，不要从PPT开始学。

## 十四、最后一句话的执行优先级

如果你时间极其有限，按这个顺序救命：

```
1. Phase 1 Step 1.2 (SQL综合精读, 3h) ← 必考，分值最大
2. Phase 2 Step 2.4 (规范化精读, 3h) ← 重点，分值第二
3. Phase 3 Step 3.1-3.2 (索引+B+树, 5h) ← 重点，操作题
4. Phase 6 做一套真题 (2h) ← 熟悉题型
5. 考前2小时背词汇+看易错点
```

这5步做完，至少能覆盖70%的分数。剩下的30%靠其他章节的概念题。
