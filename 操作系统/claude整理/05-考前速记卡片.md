## 一、必背定义（一句话版）

| 概念 | 一句话定义 |
|------|-----------|
| **操作系统** | 控制和管理计算机软硬件资源、合理组织工作流程、提供用户接口的最基本系统软件 |
| **进程** | 程序的一次执行，是资源分配和调度的独立单位 |
| **线程** | 进程内的执行体，CPU调度的基本单位，轻量级进程 |
| **PCB** | 进程控制块，描述进程情况和运行状态的数据结构，**进程存在的唯一标志** |
| **临界资源** | 一次只允许一个进程使用的资源 |
| **临界区** | 进程中访问临界资源的那段**代码** |
| **信号量** | 表示资源实体的、与队列相关的整型变量 |
| **死锁** | 多进程因竞争资源互相等待造成的僵局，需外力解决 |
| **虚拟存储器** | 具有请求调入和置换功能，逻辑上扩充内存容量的存储器系统 |
| **快表(TLB)** | 联想寄存器中存放的部分页表，加速地址转换 |
| **文件** | 一组带标识的、逻辑上有完整意义的信息项序列 |
| **FCB** | 文件控制块，存放文件所有属性信息，**文件存在的标志** |
| **SPOOLing** | 对脱机I/O的模拟，实现虚拟设备功能 |
| **RAID** | 独立冗余磁盘阵列，并行交叉存取技术 |

## 二、必记比较表格

### 进程 vs 线程

| 维度 | 进程 | 线程 |
|------|------|------|
| 资源拥有 | **拥有资源** | 基本不拥有，共享进程资源 |
| 调度单位 | 否（引入线程后） | **是** |
| 创建/切换开销 | **大** | 小 |
| 通信方式 | IPC机制 | 共享地址空间 |
| 独立性 | 独立地址空间 | 共享地址空间 |

### 并发 vs 并行

| 概念 | 时间维度 | 条件 |
|------|----------|------|
| **并发** | 同一**时间间隔内** | 单核即可（交替执行） |
| **并行** | 同一**时刻** | 需要多核 |

### 同步 vs 互斥

| 关系 | 制约类型 | 原因 |
|------|----------|------|
| **同步** | **直接**制约 | 合作关系，协调速度 |
| **互斥** | **间接**制约 | 竞争共享公共资源 |

### 死锁 vs 饥饿

| 概念 | 进程状态 | 循环等待 |
|------|----------|----------|
| **死锁** | 一定阻塞态 | 一定有 |
| **饥饿** | 可能阻塞态或就绪态 | 不一定 |

## 三、四种动态分区分配算法速记

| 算法 | 空闲区排列方式 | 一句话 |
|------|---------------|--------|
| **最先适配** | 按**地址递增** | 从头找第一个够用的 |
| **循环最先适配** | 按**地址递增** | 从上次的位置开始找 |
| **最佳适配** | 按**大小递增** | 选最小的够用的 |
| **最坏适配** | 按**大小递减** | 选最大的切一块 |

---

## 四、五种页面置换算法速记

| 算法 | 淘汰规则 | 特点 |
|------|----------|------|
| **FIFO** | 最早进入的 | 性能差，**Belady异常** |
| **OPT** | 未来最远不用的 | **理想算法**，不可实现 |
| **LRU** | 最久未**使用**的 | 性能接近最佳 |
| **LFU** | 访问**次数最少**的 | 需计数器 |
| **Clock/NRU** | 循环找use=0的 | LRU和FIFO的折衷 |

⚠️ **陷阱**: LRU淘汰的是"最久未被**访问**的"，不是"驻留时间最长的"！

## 五、四种磁盘调度算法速记

| 算法 | 策略 | 缺点 |
|------|------|------|
| **FCFS** | 按到达顺序 | 效率最低 |
| **SSTF** | 距当前磁头最近 | 可能**饥饿** |
| **SCAN(电梯)** | 一个方向扫描到底再反向 | 多走了不需要的远路 |
| **C-SCAN** | 单向扫描，快速返回 | 更公平 |

## 六、进程状态转换速记

```
        创建(New)
           ↓
        就绪(Ready) ←──────────┐
           ↓ 调度               │ 事件发生
        运行(Running) ──时间片完→┘
           ↓ 等事件
        阻塞(Blocked) ──事件发生→ 就绪
           ↓
        终止(Terminated)
```

⚠️ **不可能**: 阻塞→运行（必须经过就绪！）
⚠️ **唯一主动**: 运行→阻塞（进程自己等待事件）

## 七、死锁四个必要条件

| 条件 | 如何破坏 |
|------|----------|
| **互斥** | SPOOLing技术，改为可共享 |
| **持有并等待** | 一次性申请所有资源 |
| **非抢占** | 申请不到就释放已占资源 |
| **循环等待** | 资源编号，按序申请 |

⚠️ 前三个条件由资源分配机制决定，只有**第四个**是偶然发生可能改变的。

## 八、资源分配图判定死锁

```
图无环 → 一定无死锁
图有环 + 每类资源只有一个实例 → 必死锁
图有环 + 每类资源有多个实例 → 可能死锁，不一定
```

## 十一、常见判断题陷阱汇总 ⚠️

| 陷阱说法（都是**错的**） | 正确认知 |
|--------------------------|----------|
| "不同进程执行的代码一定不同" | 可执行相同代码（多用户同一程序） |
| "阻塞态可以直接变为运行态" | 必须经过**就绪态** |
| "LRU选择驻留时间最长的页面淘汰" | 选最久未被**使用**的，非驻留时间 |
| "线程是资源分配的基本单位" | 进程仍是资源分配基本单位 |
| "进程同步是间接制约，互斥是直接制约" | **反过来**：同步=直接，互斥=间接 |
| "最佳适配性能最好" | 最佳适配并非最优 |
| "树形目录可以节省存储空间" | 不能节省，目的不是节省空间 |
| "虚拟存储器大小没有限制" | 最大容量由CPU地址结构决定 |
| "段式没有碎片问题" | 段式有**外部碎片** |
| "文件存取方式仅与物理结构有关" | 也与逻辑结构等有关 |
| "链接结构适合随机存取" | 隐式链接**不适合**随机存取 |
| "单一CPU不需要同步互斥" | 并发执行仍需要 |
| "响应时间=时间片×用户数，时间片越小越好" | 太小会导致过多上下文切换 |
| "临界区是用于实现互斥的代码" | 临界区是**访问临界资源**的代码 |

## 十二、P/V操作速查

### 信号量初值设置

| 场景 | 信号量 | 初值 |
|------|--------|------|
| 互斥 | mutex | **1** |
| N个空位 | empty | **N** |
| 0个产品 | full | **0** |
| 同步前驱 | S | **0** |

### P/V伪代码

```c
P(S): S--; if S<0 then block()
V(S): S++; if S≤0 then wakeup()
```

### 黄金法则

1. **同步P在外，互斥P在内**
2. P顺序不能倒，V顺序无所谓
3. 互斥PV在同一进程，同步PV在不同进程
4. PV必须成对

## 十三、地址转换公式

```
页号 = 逻辑地址 / 页大小
页内偏移 = 逻辑地址 % 页大小
物理地址 = 块号 × 页大小 + 页内偏移
```

---

## 十四、文件索引最大文件公式

```
每块索引项数 K = 块大小 / 每个索引项大小
最大文件 = 直接项×块大小 + K×块大小 + K²×块大小 + K³×块大小
```

## 十五、磁盘访问时间

```
总访问时间 = 寻道时间 + 旋转延迟 + 传输时间
```

**优化重点**：减少**寻道时间**（占比最大，调度算法的优化目标）

## 临考前最后10分钟看这里

1. 进程状态图：阻塞→运行 **不可能**
2. 死锁四个条件：互斥、持有等待、非抢占、循环等待
3. 各种算法排序方式：最先→地址、最佳→大小递增、最坏→大小递减
4. LRU ≠ 驻留时间最长，LRU = 最久未使用
5. 并发≠并行：并发=时间间隔，并行=同一时刻
6. P/V: P申请(减)，V释放(加)；P在外同步，内互斥
7. 页式解决碎片，段式满足用户需求
8. 位示图→磁盘管理，TLB→页表缓存，缓冲→缓解速度差
9. UNIX: 流式文件 + inode混合索引 + 成组链接法
10. **银行家算法**：Need=Max-Allocation → 找Need≤Work的进程 → 释放Allocation → 全部Finish则安全
11. **段页式**：逻辑地址=段号+页号+偏移 → 查段表→查页表→拼物理地址（无TLB需3次访存）
12. **硬vs软链接**：硬=同inode不可跨FS，软=存路径可跨FS可链目录
13. **DMA三模式**：突发(全块)/周期窃取(逐字交替)/透明(CPU不用时)
14. 考试别慌，大题都有固定套路（现在有**六种**了），按训练步骤来
