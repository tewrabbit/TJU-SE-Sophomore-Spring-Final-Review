# 操作系统核心概念速查表

> **用途**: 考前15分钟快速浏览 | 查漏补缺 | 所有核心概念一表打尽

## 一、概述与体系结构

| 序号 | 概念 | 定义要点 | 考点提示 |
|------|------|----------|----------|
| 1 | **操作系统** | 计算机系统中最基本的系统软件，管理和控制软硬件资源，合理组织工作流程，提供用户接口 | 三种视角：资源分配器/控制程序/内核 |
| 2 | **并发** | 两个或以上事件在同一时间间隔内发生 | 与并行的区别（时间间隔 vs 同一时刻） |
| 3 | **并行** | 两个或以上事件在同一时刻同时发生 | 需要多核支持 |
| 4 | **共享** | 系统资源可被多个用户同时使用 | 两个最基本特征之一（与并发互为依存） |
| 5 | **虚拟化** | 物理上一个实体变成逻辑上多个对应物 | 时分复用/空分复用 |
| 6 | **异步性** | 多进程以不可预知速度向前推进 | 不确定性 |
| 7 | **批处理系统** | 作业成批送入计算机，由作业调度程序自动选择运行 | 1964 IBM360 OS/360——第一款通用OS |
| 8 | **分时系统** | CPU时间划分为时间片，多个终端用户轮流使用 | **交互性**是最核心特征 |
| 9 | **实时系统** | 对输入反应速度足以控制发出信号的对象 | 硬实时（严格时间约束）/软实时 |
| 10 | **微内核** | 以微内核为核心的小型内核，客户/服务器模式，面向对象 | 最有发展前途的OS结构 |
| 11 | **系统调用** | 用户程序在源程序级请求OS服务的手段 | 需要从用户态切换到内核态（陷入/访管指令） |
| 12 | **内核态（管态）** | CPU可执行特权指令的状态 | 也叫 supervisor mode |
| 13 | **用户态（目态）** | CPU只能执行非特权指令的状态 | 也叫 user mode |
| 14 | **中断** | 硬件/软件信号使CPU暂停当前任务，转向中断处理程序 | OS是中断驱动的 |
| 15 | **双重操作模式** | 用户态+内核态，硬件通过模式位支持 | 保护机制的基础 |
| 16 | **高速缓存** | 高速存储器，保存最近访问的数据 | 解决部件间速度不匹配 |
| 17 | **DMA** | 直接存储器访问，DMA控制器自动控制内存与设备间的批量数据传送 | 无需CPU干预，大幅提高效率 |
| 18 | **通道** | 独立于CPU的专用输入输出处理器，具有执行I/O指令集的能力 | CPU与通道可并行工作 |
| 19 | **基址寄存器** | 存放程序合法地址空间的最小物理地址 | 内存保护机制 |
| 20 | **界限寄存器** | 存放程序合法地址空间的长度 | 内存保护机制 |

## 二、进程管理

### 进程基础

| 序号 | 概念 | 定义要点 | 考点提示 |
|------|------|----------|----------|
| 21 | **进程** | 程序的一次执行，是资源分配和调度的独立单位 | 动态性是最本质特征 |
| 22 | **程序** | 静态的指令集合 | 进程=程序+数据+PCB的执行过程 |
| 23 | **进程控制块(PCB)** | 描述进程情况和运行状态的数据结构，存放进程全部管理信息 | **进程存在的唯一标志**，常驻内存 |
| 24 | **就绪态** | 除CPU外获得所有必要资源，等待CPU分配 | 排队等待调度 |
| 25 | **运行态** | 获得CPU，正在执行指令 | 单核系统同一时刻只有一个进程处于此状态 |
| 26 | **阻塞态（等待态）** | 等待某事件发生而暂停执行，即使给CPU也无法运行 | 只能由运行态主动进入 |
| 27 | **新建态** | 进程正在被创建，还未进入就绪队列 | — |
| 28 | **终止态** | 进程执行完成，等待资源回收 | — |
| 29 | **挂起态** | 进程被换出到外存 | 与中级调度相关 |
| 30 | **原语** | 内核中由若干指令构成的、执行时不可中断的过程 | 原子操作 |
| 31 | **进程创建(fork)** | 父进程创建子进程 | UNIX：fork()创建，execve()替换内存空间 |
| 32 | **进程终止(exit)** | 进程执行完最后语句，请求OS删除 | 子进程可被父进程终止(abort) |

### 临界区与同步

| 序号 | 概念 | 定义要点 | 考点提示 |
|------|------|----------|----------|
| 33 | **临界资源** | 一次只允许一个进程使用的共享资源 | 也叫互斥资源 |
| 34 | **临界区** | 进程中访问临界资源的那段**代码** | 不是"实现互斥的代码" |
| 35 | **竞争条件** | 多进程并发访问同一数据，结果取决于执行顺序 | 需要同步机制解决 |
| 36 | **互斥** | 一次最多允许一个进程进入临界区 | **间接**制约关系（竞争共享资源） |
| 37 | **同步** | 合作进程在关键执行点上相互协调 | **直接**制约关系（协作任务） |
| 38 | **信号量** | 表示资源实体的、与队列相关的整型变量 | 只能通过P/V原语访问 |
| 39 | **P操作(wait)** | S=S-1；若S<0则阻塞自己 | 申请资源，可能阻塞 |
| 40 | **V操作(signal)** | S=S+1；若S≤0则唤醒一个等待进程 | 释放资源，可能唤醒 |
| 41 | **互斥信号量** | 初值通常为1 | 用于实现临界区互斥 |
| 42 | **资源信号量** | 初值可为大于1的正整数 | 用于实现进程同步 |
| 43 | **管程** | 由编译器实现互斥特性的高级同步机制 | 资源管理模块 |
| 44 | **忙等待** | 进程反复检查条件而不释放CPU | 不能让权等待，浪费CPU |
| 45 | **生产者-消费者问题** | 共享有限缓冲区N的生产消费协调问题 | 信号量：mutex=1, empty=N, full=0 |
| 46 | **读者-写者问题** | 读者可同时读，写者互斥 | 第一类：读者优先→写者可能饥饿 |
| 47 | **哲学家进餐问题** | 5人5筷，每人需2根 | 可能死锁 |
| 48 | **有限等待** | 等待进程在有限时间内一定能进入临界区 | 同步机制的重要原则 |
| 49 | **让权等待** | 不能进入临界区时立即释放CPU | 软件算法一般做不到 |
| 50 | **Test-and-Set(TS)** | 原子地读-修改-写指令 | 硬件互斥方法 |

### 线程

| 序号 | 概念 | 定义要点 | 考点提示 |
|------|------|----------|----------|
| 51 | **线程** | 进程内的执行体，CPU调度的基本单位 | 轻量级进程(LWP) |
| 52 | **线程控制块(TCB)** | 线程的控制信息数据结构 | 保存线程上下文 |
| 53 | **用户级线程** | 线程库实现，切换不经过内核 | 一个线程阻塞→整个进程阻塞 |
| 54 | **内核级线程** | OS内核直接支持，阻塞在单个线程级别 | 多处理机上可真正并行 |
| 55 | **多对一模型** | 多个用户线程映射到一个内核线程 | 不能利用多核 |
| 56 | **一对一模型** | 每个用户线程对应一个内核线程 | Windows系列采用 |
| 57 | **多对多模型** | 多个用户线程映射到多个内核线程 | 折衷方案 |

### CPU调度

| 序号 | 概念 | 定义要点 | 考点提示 |
|------|------|----------|----------|
| 58 | **高级调度（作业调度）** | 选后备作业进入内存，创建进程 | 时间尺度：分钟/小时级 |
| 59 | **中级调度** | 进程在内外存间的交换 | 提高内存利用率 |
| 60 | **低级调度（进程调度）** | 从就绪队列选进程分配CPU | **毫秒级**，最频繁 |
| 61 | **分派程序** | 将CPU控制权交给选中进程的模块 | 完成上下文切换 |
| 62 | **上下文切换** | 保存旧进程状态，恢复新进程状态 | 纯开销，切换速度影响系统性能 |
| 63 | **非抢占式调度** | 进程运行直到自愿放弃CPU | 简单、开销小，实时性差 |
| 64 | **抢占式调度** | OS可强制剥夺CPU | 实时性好，基于优先级/时间片 |
| 65 | **先来先服务(FCFS)** | 按到达就绪队列的顺序分配CPU | 非抢占，简单；长作业有利 |
| 66 | **短作业优先(SJF)** | 从就绪队列选预计执行时间最短的 | 平均等待时间最小，但长作业可能饥饿 |
| 67 | **最短剩余时间优先(SRTF)** | SJF的抢占版本 | 理论最优 |
| 68 | **优先级调度** | 选优先级最高的进程（数字小=优先级高） | SJF是它的一种特例 |
| 69 | **老化** | 逐渐提高等待进程的优先级 | 解决优先级调度中的饥饿问题 |
| 70 | **时间片轮转(RR)** | 每个进程轮流获得固定的CPU时间片 | 适合分时系统；q太大→FCFS，q太小→过多切换 |
| 71 | **时间片** | RR算法中每个进程一次获得的CPU时间长度 | 通常10-100ms |
| 72 | **多级队列调度** | 进程永久分配到不同优先级队列 | — |
| 73 | **多级反馈队列** | 进程可在队列间移动，动态调整优先级和时间片 | **最综合、最优的调度算法** |
| 74 | **周转时间** | 进程从提交到完成的时间间隔 | 越小越好 |
| 75 | **等待时间** | 进程在就绪队列中等待的总时间 | 越小越好 |
| 76 | **响应时间** | 从提交请求到首次产生响应的时间 | 分时系统关键指标 |
| 77 | **吞吐量** | 单位时间内完成的进程数量 | 越大越好 |

### 死锁

| 序号 | 概念 | 定义要点 | 考点提示 |
|------|------|----------|----------|
| 78 | **死锁** | 多进程因竞争资源互相等待造成僵局 | 需要外力才能解除 |
| 79 | **互斥条件** | 资源一次只能被一个进程使用 | 四必要条件之一 |
| 80 | **持有并等待** | 进程占有资源的同时等待其他资源 | 四必要条件之一 |
| 81 | **非抢占条件** | 资源不能被强行剥夺 | 四必要条件之一 |
| 82 | **循环等待条件** | 进程间形成资源等待环路 | 四必要条件之一 |
| 83 | **死锁预防** | 破坏四个必要条件之一，静态策略 | 如资源排序、一次性分配 |
| 84 | **死锁避免** | 动态检查每次分配是否会导致死锁 | 银行家算法 |
| 85 | **死锁检测** | 允许死锁发生，通过检测算法发现 | 利用资源分配图 |
| 86 | **死锁恢复** | 终止进程或剥夺资源，代价大 | 终止策略：优先级低/已运行时间短/批处理优先终止 |
| 87 | **安全状态** | 存在某种执行序列使所有进程都能完成 | 安全→不产生死锁 |
| 88 | **不安全状态** | 不存在安全的执行序列 | 不安全≠一定死锁，只是可能死锁 |
| 89 | **银行家算法** | 检查每个资源请求后系统是否仍处于安全状态 | 死锁避免的经典算法 |
| 90 | **资源分配图** | 用有向图描述资源分配和请求关系 | 无环→不死锁；有环+单实例→死锁 |

### 进程通信

| 序号 | 概念 | 定义要点 | 考点提示 |
|------|------|----------|----------|
| 91 | **进程通信(IPC)** | 进程间信息交换机制 | 分低级（状态/整数值）和高级（任意数据） |
| 92 | **共享存储** | 进程共享内存区域进行通信 | 基于共享数据结构（低级）或共享存储区（高级） |
| 93 | **消息传递** | 通过send()/receive()原语交换消息 | 最广泛使用的通信机制 |
| 94 | **管道** | 基于共享文件的通信 | 匿名管道（父子进程）/命名管道（任意进程） |
| 95 | **套接字** | 网络通信的端点 | 跨机器通信 |
| 96 | **远程过程调用(RPC)** | 跨网络的函数调用 | 网络通信方式 |

## 三、内存管理

### 基本概念与连续分配

| 序号 | 概念 | 定义要点 | 考点提示 |
|------|------|----------|----------|
| 97 | **逻辑地址** | 程序生成的地址（CPU产生的地址） | 也叫相对地址/虚拟地址 |
| 98 | **物理地址** | 内存单元的硬件地址 | 也叫绝对地址 |
| 99 | **地址重定位** | 将逻辑地址转换为物理地址的过程 | OS的装入程序完成 |
| 100 | **静态重定位** | 装入时一次性完成地址转换 | 执行期间地址不再改变 |
| 101 | **动态重定位** | 每次访问内存时实时转换地址 | 需要硬件支持，程序可移动 |
| 102 | **内碎片** | 已分配给进程但未被使用的内部空间 | 固定分区有，动态分区无 |
| 103 | **外碎片** | 未分配但因太小无法使用的空闲区 | 动态分区有，分页无 |
| 104 | **紧凑** | 移动已分配区将外碎片合并为大空闲区 | 解决外碎片的方法之一 |
| 105 | **最先适配** | 分配第一个满足要求的空闲分区 | 空闲区按**地址递增**排列 |
| 106 | **最佳适配** | 分配最小的能满足要求的空闲分区 | 空闲区按**大小递增**排列 |
| 107 | **最坏适配** | 分配最大的空闲分区 | 空闲区按**大小递减**排列 |
| 108 | **覆盖** | 程序员手动划分模块，分时装入同一内存区 | 同一作业内部，以时间换空间 |
| 109 | **交换** | 进程在内存与外存之间整体换入换出 | 进程之间，引入中级调度 |

### 分页与分段

| 序号 | 概念 | 定义要点 | 考点提示 |
|------|------|----------|----------|
| 110 | **页** | 逻辑地址空间被划分成的固定大小块 | 大小为2的整数次幂（如4KB） |
| 111 | **页框（帧）** | 物理内存被划分成的与页等大小的块 | 也叫物理页面/实页 |
| 112 | **页表** | 记录逻辑页号到物理块号的映射关系 | 每个进程一张，放在内存中 |
| 113 | **快表(TLB)** | 存放在联想寄存器中的部分页表 | 内容按页号查找，加速地址转换 |
| 114 | **段** | 按逻辑关系划分的可变大小内存块 | 用户可见，可独立编址 |
| 115 | **段表** | 记录段号到段基址和段长度的映射 | 每个进程一张 |
| 116 | **段页式** | 进程按段划分，内存按页管理 | 结合两者优点 |
| 117 | **多级页表** | 页表本身也分页存储 | 减少页表所需的连续内存空间 |

### 虚拟存储器

| 序号 | 概念 | 定义要点 | 考点提示 |
|------|------|----------|----------|
| 118 | **虚拟存储器** | 请求调入+置换，逻辑上扩充内存 | 核心三特征：不连续、部分交换、大空间 |
| 119 | **局部性原理** | 程序执行时访问的地址局限在一定区域 | 虚拟存储器的理论基础 |
| 120 | **时间局部性** | 刚访问的内容近期可能再次被访问 | 如循环中的指令和数据 |
| 121 | **空间局部性** | 刚访问内容附近的地址可能被访问 | 如顺序执行的代码 |
| 122 | **缺页中断** | 访问的页不在内存时由硬件触发的中断 | OS调入缺页，可能需要置换 |
| 123 | **请求调页** | 只在需要时才调入页面 | 减少I/O和内存占用 |
| 124 | **预调页** | 调入所需页面时同时调入相邻页面 | 初始加载时使用，依赖预测 |
| 125 | **先进先出(FIFO)** | 选择最早进入内存的页面淘汰 | Belady异常（页框增加但缺页率反而上升） |
| 126 | **最佳置换(OPT)** | 选择未来最久不会使用的页面淘汰 | 理想算法，不可实现，仅用作评价基准 |
| 127 | **最近最久未使用(LRU)** | 选择最久未被访问的页面淘汰 | 性能接近OPT，局部性原理的合理近似 |
| 128 | **最不常用(LFU)** | 选择被访问次数最少的页面淘汰 | 需要访问计数器 |
| 129 | **轮转算法(Clock/NRU)** | 循环检查访问位use=0的页淘汰 | LRU和FIFO的折衷，实际系统常用 |
| 130 | **Belady异常** | 分配更多物理页框反而缺页率上升 | 只有FIFO算法有此现象 |
| 131 | **工作集** | 某时间窗口内进程实际访问的页面集合 | 用于防止抖动 |
| 132 | **抖动** | 频繁的页面换入换出导致CPU利用率急剧下降 | 多道程序度过高引起 |
| 133 | **常驻集** | 操作系统分配给进程的物理页面数量 | 固定分配 / 可变分配 |
| 134 | **缺页率** | 缺页次数 ÷ 总访问次数 | 越低越好 |

## 四、文件管理

### 文件与文件系统

| 序号 | 概念 | 定义要点 | 考点提示 |
|------|------|----------|----------|
| 135 | **文件** | 一组带标识的、逻辑上有完整意义的信息项序列 | 由文件名+后缀名+路径组成 |
| 136 | **文件系统** | OS中统一管理信息资源的软件 | 管理存储/检索/更新，提供共享和保护 |
| 137 | **流式文件** | 无结构，按字符序列组织的文件 | UNIX采用，目前最常用的逻辑结构 |
| 138 | **记录式文件** | 有结构，由若干记录组成的文件 | 每记录有键(Key)，可按键查找 |
| 139 | **顺序（连续）结构** | 文件存储在连续物理块中 | 顺序存取最快，支持随机存取；不能动态增长 |
| 140 | **链接结构** | 文件块通过指针链接 | 无外部碎片，便于动态扩充；**不适合随机存取** |
| 141 | **索引结构** | 系统维护索引表记录每块物理地址 | 支持顺序和随机存取，但索引表有开销 |
| 142 | **i节点(inode)** | UNIX文件的索引节点 | 存储文件元数据和数据块指针 |
| 143 | **直接索引** | 索引项直接指向文件数据块 | UNIX中前10项为直接索引 |
| 144 | **一级间接索引** | 索引项指向一级索引块，块中每项指向一个数据块 | UNIX第11项 |
| 145 | **二级间接索引** | 索引项指向一级索引块→二级索引块→数据块 | UNIX第12项 |
| 146 | **三级间接索引** | 经过三层间接指向数据块 | UNIX第13项 |

### 目录与空间管理

## 七、补充概念（深度补充）

| 序号 | 概念 | 定义要点 | 考点提示 |
|------|------|----------|----------|
| 181 | **银行家算法** | 死锁避免算法。维护Available/Max/Allocation/Need，每次分配前执行安全性检查，只有安全才分配 | 大题计算，安全序列+资源请求判断 |
| 182 | **安全状态** | 存在一个执行序列使所有进程都能顺利完成 | 安全→不死锁，不安全→可能死锁 |
| 183 | **管程** | 由编译器实现互斥的高级同步机制，包含共享数据结构+操作过程+条件变量 | 与信号量对比：更安全但灵活性低 |
| 184 | **条件变量** | 管程内用于同步的机制，提供wait和signal操作 | 信号量的管程替代方案 |
| 185 | **硬链接** | 多个目录项指向同一inode，共享link count | 不可跨文件系统，删除原文件不影响 |
| 186 | **软链接** | 存储目标文件路径的特殊独立文件 | 可跨文件系统，原文件删除后悬空 |
| 187 | **低级格式化** | 磁盘出厂时将盘面划分为扇区（含头部/数据区/ECC） | 也叫物理格式化 |
| 188 | **高级格式化** | 创建文件系统：写入引导块、超级块、空闲空间管理结构、根目录 | 也叫逻辑格式化，OS安装时完成 |
| 189 | **MBR** | 主引导记录，位于0扇区，含引导代码+分区表（最多4主分区） | GPT是新一代分区表 |
| 190 | **二类读者写者** | 写者优先：第一个写者堵住后续读者，最后一个写者放行 | 用额外信号量R实现优先 |
| 191 | **DMA突发模式** | 一次传输整个数据块，期间CPU无法使用总线 | 效率最高但CPU停等最久 |
| 192 | **DMA周期窃取** | 每次传一个字节/字就释放总线，与CPU交替 | 折衷方案，CPU性能下降少 |
| 193 | **DMA透明模式** | 只在CPU不使用总线时传输 | 对CPU完全透明，实现最复杂 |
| 194 | **SPOOLing工作流** | 用户print→数据入输出井→加入队列→立即返回→守护进程按序输出到打印机 | 独占设备→共享设备的关键实现 |
