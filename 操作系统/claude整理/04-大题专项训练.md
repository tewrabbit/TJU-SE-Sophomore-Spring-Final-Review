# 题型一：P/V操作（信号量同步）🔥🔥🔥

## 解题套路

### 步骤一：分析进程数量和角色
- 有几个进程？每个进程做什么？
- 谁是生产者？谁是消费者？
- 有无互斥资源（共享缓冲区等）？

### 步骤二：确定信号量及初值

| 信号量类型 | 用途 | 初值如何确定 |
|-----------|------|-------------|
| 互斥信号量 mutex | 保护共享资源（缓冲区等） | **初值=1** |
| 资源信号量 empty | 表示空位数 | **初值=缓冲区大小N** |
| 资源信号量 full | 表示满位数 | **初值=0** |
| 同步信号量 S | 前驱关系 | **初值=0** |

### 步骤三：写出每个进程的伪代码

**黄金法则**：
1. **同步在外，互斥在内**：先P资源信号量，再P互斥信号量
2. **P操作顺序至关重要**，V操作顺序无所谓
3. **PV必须成对出现**，互斥PV在同一进程，同步PV在不同进程
4. 互斥信号量的P和V之间是**临界区**，要尽量短

## 例题1：生产者-消费者（基础必会）

**题目**：一组生产者向一组消费者提供产品，共享一个有N个缓冲块的缓冲池。生产者放产品，消费者取产品。用P/V操作实现同步。

**解答**：

```c
semaphore mutex = 1;   // 互斥访问缓冲池
semaphore empty = N;   // 空缓冲块数
semaphore full = 0;    // 满缓冲块数

// 生产者进程
Producer() {
    while (1) {
        生产一个产品;
        P(empty);      // ① 先申请空位（同步P）
        P(mutex);      // ② 再申请互斥（互斥P）
        将产品放入缓冲池;
        V(mutex);      // ③ 释放互斥
        V(full);       // ④ 增加满位数
    }
}

// 消费者进程
Consumer() {
    while (1) {
        P(full);       // ① 先申请产品（同步P）
        P(mutex);      // ② 再申请互斥
        从缓冲池取出产品;
        V(mutex);      // ③ 释放互斥
        V(empty);      // ④ 增加空位数
        消费产品;
    }
}
```

**易错点**：P(empty)和P(mutex)的顺序不能颠倒！如果先P(mutex)再P(empty)，当缓冲满时生产者持锁等空位，消费者因拿不到锁无法取产品→**死锁**。

## 例题2：三进程协作（期末真题原型）

**题目**：三个进程PA、PB、PC合作解决文件打印问题。PA将文件记录从磁盘读入内存缓冲区1（每次一个记录）；PB将缓冲区1内容复制到缓冲区2（每次一个记录）；PC将缓冲区2内容打印（每次一个记录）。缓冲区大小等于一个记录大小。请用P/V操作保证正确打印。

**解答**：

```c
semaphore empty1 = 1;  // 缓冲区1空
semaphore full1 = 0;   // 缓冲区1满
semaphore empty2 = 1;  // 缓冲区2空
semaphore full2 = 0;   // 缓冲区2满

// PA：从磁盘读入buf1
PA() {
    while (1) {
        从磁盘读一个记录;
        P(empty1);          // 等buf1空
        将记录放入buf1;
        V(full1);           // 通知buf1满
    }
}

// PB：从buf1复制到buf2
PB() {
    while (1) {
        P(full1);           // 等buf1有数据
        从buf1取出记录;
        V(empty1);          // 通知buf1已空
        P(empty2);          // 等buf2空
        将记录放入buf2;
        V(full2);           // 通知buf2满
    }
}

// PC：从buf2取出打印
PC() {
    while (1) {
        P(full2);           // 等buf2有数据
        从buf2取出记录;
        V(empty2);          // 通知buf2已空
        打印记录;
    }
}
```

## 例题3：独木桥问题

**题目**：一座独木桥，同一方向的行人可以连续过桥。当某一方向有人过桥时，另一方向的人必须等待。用P/V操作实现。

**解答**：

```c
int countA = 0, countB = 0;   // 两个方向上的人数
semaphore mutex = 1;          // 桥的使用权
semaphore SA = 1;             // 保护countA
semaphore SB = 1;             // 保护countB

// A方向行人
A_direction() {
    P(SA);
    if (countA == 0)
        P(mutex);             // 第一个A行人占桥
    countA++;
    V(SA);
    
    过桥;
    
    P(SA);
    countA--;
    if (countA == 0)
        V(mutex);             // 最后一个A行人放桥
    V(SA);
}

// B方向行人（对称）
B_direction() {
    P(SB);
    if (countB == 0)
        P(mutex);
    countB++;
    V(SB);
    
    过桥;
    
    P(SB);
    countB--;
    if (countB == 0)
        V(mutex);
    V(SB);
}
```

**关键点**：第一个同向的人占桥(P(mutex))，最后一个同向的人放桥(V(mutex))。

## 例题4：读者-写者问题（读者优先）

**题目**：读者可同时读；写者与任何人不共存；读者优先（新读者来了，即使有写者在等，读者也能进）。

**解答**：

```c
int readcount = 0;           // 当前读者数
semaphore rmutex = 1;        // 保护readcount
semaphore wmutex = 1;        // 写互斥（读写互斥）

// 读者
Reader() {
    while (1) {
        P(rmutex);
        if (readcount == 0)
            P(wmutex);       // 第一个读者锁住写者
        readcount++;
        V(rmutex);
        
        读操作;
        
        P(rmutex);
        readcount--;
        if (readcount == 0)
            V(wmutex);       // 最后一个读者释放写者
        V(rmutex);
    }
}

// 写者
Writer() {
    while (1) {
        P(wmutex);           // 写者必须独占
        写操作;
        V(wmutex);
    }
}
```

# 题型二：进程调度计算 🔥🔥

## 常用调度算法及计算方法

### 基本概念

| 时间概念 | 定义 |
|----------|------|
| **到达时间** (Arrival Time) | 进程到达就绪队列的时刻 |
| **服务时间** (Burst Time) | 进程需要的CPU执行时间 |
| **完成时间** (Completion Time) | 进程执行结束的时刻 |
| **周转时间** = 完成时间 - 到达时间 | |
| **带权周转时间** = 周转时间 / 服务时间 | |
| **等待时间** = 周转时间 - 服务时间 | |

### 算法要点

| 算法 | 调度策略 | 抢占？ |
|------|---------|--------|
| **FCFS** | 按到达顺序 | 非抢占 |
| **SJF** | 选服务时间最短的 | 非抢占 |
| **SRTF** | 选剩余时间最短的 | 抢占（SJF抢占版） |
| **RR** | 时间片轮转，用完排到队尾 | 抢占 |
| **优先级** | 选优先级最高的 | 均可 |

---

## 例题：四种算法对比计算

**题目**：四个进程到达时间和服务时间如下，计算FCFS、SJF(非抢占)、SRTF(抢占)、RR(q=2)的平均等待时间。

| 进程 | 到达时间 | 服务时间 |
|------|----------|----------|
| P1 | 0 | 8 |
| P2 | 1 | 4 |
| P3 | 2 | 9 |
| P4 | 3 | 5 |

### FCFS（按到达顺序：P1→P2→P3→P4）

| 进程 | 到达 | 服务 | 开始 | 完成 | 周转 | 等待 |
|------|------|------|------|------|------|------|
| P1 | 0 | 8 | 0 | 8 | 8 | 0 |
| P2 | 1 | 4 | 8 | 12 | 11 | 7 |
| P3 | 2 | 9 | 12 | 21 | 19 | 10 |
| P4 | 3 | 5 | 21 | 26 | 23 | 18 |

**平均等待时间** = (0 + 7 + 10 + 18) / 4 = **8.75**

### SJF（非抢占，每次选最短）

**执行序列分析**：
- t=0: 只有P1，执行P1(8)
- t=8: P2(剩4), P3(剩9), P4(剩5)都在就绪队列。最短P2(4)，执行P2
- t=12: P3(剩9), P4(剩5)。最短P4(5)，执行P4
- t=17: 执行P3(9)
- t=26: 完成

| 进程 | 到达 | 服务 | 开始 | 完成 | 周转 | 等待 |
|------|------|------|------|------|------|------|
| P1 | 0 | 8 | 0 | 8 | 8 | 0 |
| P2 | 1 | 4 | 8 | 12 | 11 | 7 |
| P3 | 2 | 9 | 17 | 26 | 24 | 15 |
| P4 | 3 | 5 | 12 | 17 | 14 | 9 |

**平均等待时间** = (0 + 7 + 15 + 9) / 4 = **7.75**

### SRTF（抢占式最短剩余时间优先）

**执行序列**（逐步分析）：

```
t=0:  只有P1到达，P1执行
t=1:  P2到达，P1剩7 vs P2需4，P2更短→抢占P1，P2执行
t=2:  P3到达，P1剩7, P2剩3, P3需9。P2最短，继续P2
t=3:  P4到达，P1剩7, P2剩2, P3需9, P4需5。P2最短，继续
t=5:  P2完成！就绪: P1(7), P3(9), P4(5)。P4(5)最短，P4执行
t=10: P4完成！就绪: P1(7), P3(9)。P1(7)执行
t=17: P1完成！执行P3(9)
t=26: P3完成
```

| 进程 | 到达 | 服务 | 完成 | 周转 | 等待 |
|------|------|------|------|------|------|
| P1 | 0 | 8 | 17 | 17 | 9 |
| P2 | 1 | 4 | 5 | 4 | 0 |
| P3 | 2 | 9 | 26 | 24 | 15 |
| P4 | 3 | 5 | 10 | 7 | 2 |

**平均等待时间** = (9 + 0 + 15 + 2) / 4 = **6.5**

### RR（q=2，时间片轮转）

**就绪队列变化**（时间片=2）：

```
t=0:  队列[P1], P1执行2
t=2:  P2(t=1到达已在队), 队列[P2,P1(剩6)], P2执行
t=3:  P4到达, 队列[P2,P1(6),P4]
t=4:  P2用2完成, 队列[P1(6),P3,P4], P1执行2
t=6:  队列[P3,P4,P1(4)], P3执行2
t=8:  队列[P4,P1(4),P3(7)], P4执行2
t=10: 队列[P1(4),P3(7),P4(3)], P1执行2
t=12: 队列[P3(7),P4(3),P1(2)], P3执行2
t=14: 队列[P4(3),P1(2),P3(5)], P4执行2
t=16: 队列[P1(2),P3(5),P4(1)], P1执行2 → P1完成!
t=18: 队列[P3(5),P4(1)], P3执行2
t=20: 队列[P4(1),P3(3)], P4执行1 → P4完成!
t=21: 队列[P3(3)], P3执行3
t=24: P3完成
```

| 进程 | 到达 | 服务 | 完成 | 周转 | 等待 |
|------|------|------|------|------|------|
| P1 | 0 | 8 | 16 | 16 | 8 |
| P2 | 1 | 4 | 4 | 3 | -1→0 |
| P3 | 2 | 9 | 24 | 22 | 13 |
| P4 | 3 | 5 | 20 | 17 | 12 |

实际等待时间：P2=0（等0，执行到完成无等待）

**平均等待时间** = (8 + 0 + 13 + 12) / 4 = **8.25**

### 总结对比

| 算法 | 平均等待时间 |
|------|-------------|
| FCFS | 8.75 |
| SJF | 7.75 |
| **SRTF** | **6.5** ← 最短！ |
| RR(q=2) | 8.25 |

**SJF/SRTF平均等待时间最小**，但SRTF需抢占，开销大。

# 题型三：页式存储管理计算 🔥🔥

## 3.1 地址转换（逻辑地址→物理地址）

### 基本公式

```
逻辑地址 = 页号 × 页大小 + 页内偏移
页号 = 逻辑地址 / 页大小（整数除法）
页内偏移 = 逻辑地址 % 页大小（取余）

物理地址 = 物理块号 × 页大小 + 页内偏移
```

### 例题：十进制地址转换

**题目**：某虚拟存储器用户空间共32个页面，每页1KB，主存16KB。某时刻为第0、1、2、3页分配的物理块号为5、10、4、7。用户程序共6页。

(1) 写出逻辑地址和物理地址格式
(2) 将逻辑地址 0x05AC（十六进制）转换为物理地址

**解答**：

**(1) 地址格式**：
- 页大小 = 1KB = 2^10，页内偏移占10位
- 逻辑页面32个 = 2^5，页号占5位
- 逻辑地址 = 5位页号 + 10位页内偏移，共15位
- 物理块 = 16KB/1KB = 16块 = 2^4，块号占4位
- 物理地址 = 4位块号 + 10位块内偏移，共14位

**(2) 0x05AC转换**：
- 0x05AC = 0000 0101 1010 1100 (二进制15位)
- 页号 = 高5位 = 00000 = **0**（不对，让我重算）

更简单的方法：
- 页大小 = 1024
- 0x05AC = 5×256 + 10×16 + 12 = 1280 + 160 + 12 = **1452**（十进制）
- 页号 = 1452 / 1024 = **1**
- 页内偏移 = 1452 % 1024 = **428**
- 第1页对应物理块号 = 10
- 物理地址 = 10 × 1024 + 428 = 10240 + 428 = **10668** = 0x29AC

**验证**：10668 / 1024 = 10 余 428 ✓

---

### 例题进阶：缺页中断情况

**题目**：接上题，逻辑地址0x103C能否直接转换？如能，给出物理地址；如不能，描述过程。

**解答**：
- 0x103C = 1×4096 + 0×256 + 3×16 + 12 = 4096 + 48 + 12 = 4156（十进制）
- 或者：0x103C = 0001 0000 0011 1100（二进制）
  - 页号 = 高5位 = 00100 = **4**
  - 页内偏移 = 低10位 = 00 0011 1100 = 60

- 页号=4，但题目只给第0~3页分配了物理块，**第4页不在内存中**！
- **不能直接转换**！产生缺页中断(Page Fault)

**后续过程**：
1. OS检查第4页在外存的位置
2. 若有空闲物理块 → 分配一块，将第4页从外存调入，修改页表
3. 若无空闲 → 选择一个页面置换出去（若修改过则写回），再调入第4页
4. 重新执行触发缺页的指令，此时地址可以正常转换

## 3.2 页面置换算法计算

### 例题：FIFO vs OPT vs LRU

**题目**：页面走向为 4, 3, 2, 1, 4, 3, 5, 4, 3, 2, 1, 5，分配给进程的物理块数分别为3和4。计算FIFO和OPT的缺页次数（初始内存为空）。

### 物理块数 = 3

#### FIFO（先进先出）

| 访问 | 4 | 3 | 2 | 1 | 4 | 3 | 5 | 4 | 3 | 2 | 1 | 5 |
|------|---|---|---|---|---|---|---|---|---|---|---|---|
| 内存1 | 4 | 4 | 4 | 1 | 1 | 1 | 5 | 5 | 5 | 2 | 2 | 2 |
| 内存2 | - | 3 | 3 | 3 | 4 | 4 | 4 | 4 | 3 | 3 | 1 | 1 |
| 内存3 | - | - | 2 | 2 | 2 | 3 | 3 | 3 | 3 | 3 | 3 | 5 |
| **缺页?** | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | × | ✓ | ✓ | ✓ | ✓ |

**缺页次数 = 11**（共12次访问，只有访问4在第8次命中）

#### OPT（最佳置换）

| 访问 | 4 | 3 | 2 | 1 | 4 | 3 | 5 | 4 | 3 | 2 | 1 | 5 |
|------|---|---|---|---|---|---|---|---|---|---|---|---|
| 内存1 | 4 | 4 | 4 | 4 | 4 | 4 | 4 | 4 | 4 | 2 | 2 | 2 |
| 内存2 | - | 3 | 3 | 3 | 3 | 3 | 3 | 3 | 3 | 3 | 1 | 1 |
| 内存3 | - | - | 2 | 1 | 1 | 1 | 5 | 5 | 5 | 5 | 5 | 5 |
| **缺页?** | ✓ | ✓ | ✓ | ✓ | × | × | ✓ | × | × | ✓ | ✓ | × |

分析每次置换时的"未来"：
- 访问5时(第7次)：内存{4,3,1}，未来引用：4出现在第8, 3在第9, 1不再出现 → 淘汰1，换入5
- 访问2时(第10次)：内存{4,3,5}，未来引用：4不再出现, 3不再出现, 5在第12 → 淘汰4(最远)，换入2
- 访问1时(第11次)：内存{2,3,5}，未来：2不再出现, 3不再出现, 5在第12 → 淘汰2或3，换入1

**缺页次数 = 8**

---

### 物理块数 = 4

#### FIFO

| 访问 | 4 | 3 | 2 | 1 | 4 | 3 | 5 | 4 | 3 | 2 | 1 | 5 |
|------|---|---|---|---|---|---|---|---|---|---|---|---|
| 缺页 | ✓ | ✓ | ✓ | ✓ | × | × | ✓ | ✓ | × | ✓ | ✓ | ✓ |

**缺页次数 = 10** ← 注意！物理块从3→4，FIFO缺页反而从11→10……这个例子不是Belady异常。

> **Belady异常示例页面走向**: 1,2,3,4,1,2,5,1,2,3,4,5
> - 3块时缺页=9；4块时缺页=10 ← 这就是Belady异常！

#### OPT

| 访问 | 4 | 3 | 2 | 1 | 4 | 3 | 5 | 4 | 3 | 2 | 1 | 5 |
|------|---|---|---|---|---|---|---|---|---|---|---|---|
| 缺页 | ✓ | ✓ | ✓ | ✓ | × | × | ✓ | × | × | × | ✓ | × |

**缺页次数 = 7**

---

### 结果汇总

| 算法 | 3块缺页 | 4块缺页 |
|------|---------|---------|
| FIFO | 11 | 10 |
| OPT | 8 | 7 |

# 题型四：文件系统计算 🔥🔥

## 4.1 混合索引最大文件大小

### 例题（期末真题原型）

**题目**：某文件管理采用直接索引和多级索引混合方式。文件索引表共10项，前8项是直接索引，第9项是一级间接索引，第10项是二级间接索引。物理块大小2KB，每个索引项占4B。求最大文件大小。

**解答**：

**第一步**：计算每个物理块可存放的索引项数
- 块大小 = 2KB = 2048B
- 每索引项 = 4B
- 每块索引项数 = 2048 / 4 = **512个**

**第二步**：分别计算各级索引可寻址的数据量

| 索引类型 | 块数 | 计算方式 | 数据量 |
|----------|------|----------|--------|
| 直接索引(8项) | 8 | 8×1 | 8 × 2KB = **16KB** |
| 一级间接(第9项) | 512 | 512×1 | 512 × 2KB = **1MB** |
| 二级间接(第10项) | 512×512=262144 | 512×512 | 262144 × 2KB = **512MB** |

**第三步**：求和
- 最大文件 = 16KB + 1MB + 512MB ≈ **513MB**

**对应公式**：
```
最大文件 = 直接项×块大小 + (块大小/索引大小)¹×块大小 + (块大小/索引大小)²×块大小
```

---

### 变式训练

**题目**：索引表12项，前10项直接，第11项一级间接，第12项二级间接。块大小4KB，索引项4B。求最大文件。

**解答**：
- 每块索引项数 = 4096 / 4 = **1024个**
- 直接(10项): 10 × 4KB = **40KB**
- 一级间接: 1024 × 4KB = **4MB**
- 二级间接: 1024 × 1024 × 4KB = **4GB**
- 总计: 40KB + 4MB + 4GB ≈ **4GB**

## 4.2 目录改进计算

### 例题

**题目**：文件目录采用改进前FCB方式，每个FCB 48B（含文件名6B），物理块512B。目录共128个文件。若采用改进方式（分解为符号目录项和基本目录项，符号目录项含文件名6B和inode号2B），求改进前后的平均磁盘访问次数。

**解答**：

**改进前**：
- 每块存放FCB数 = 512 / 48 = 10个
- 128个文件需要的块数 = 128 / 10 = 13块（向上取整）
- 平均访问磁盘次数 = (1 + 13) / 2 = **7次**

**改进后**：
- 符号目录项大小 = 6 + 2 = 8B
- 每块存放符号项数 = 512 / 8 = 64个
- 符号目录占块数 = 128 / 64 = 2块
- 基本目录项大小 = 48 - 6 = 42B
- 每块存放基本项数 = 512 / 42 = 12个
- 基本目录占块数 = 128 / 12 = 11块（向上取整）

平均访问 = 查符号目录 + 查基本目录一次
- = (1 + 2) / 2 + 1
- = 1.5 + 1
- = **2.5次**

**改进效果明显**：从平均7次降到2.5次！

## 4.3 磁盘调度计算

### 例题

**题目**：磁盘访问请求序列(柱面号)：55, 58, 39, 18, 90, 160, 150, 38, 184。当前磁头在100号柱面。分别用FCFS、SSTF、SCAN(向柱面号增大方向)、C-SCAN计算磁头移动总距离。

**解答**：

**FCFS**（按请求顺序）：
100→55→58→39→18→90→160→150→38→184
= 45+3+19+21+72+70+10+112+146 = **498柱面**

**SSTF**（每次选最近）：
100→90→58→55→39→38→18→150→160→184
= 10+32+3+16+1+20+132+10+24 = **248柱面**

**SCAN**（向增大方向扫描）：
先向右：100→150→160→184→（到头）
再向左：→90→58→55→39→38→18
= 50+10+24+(184-18) = 50+10+24+166 = **250柱面**

**C-SCAN**（单向，到头跳回）：
向右：100→150→160→184
跳回0→18→38→39→55→58→90
= 50+10+24+184(跳到0)+18+20+1+16+3+32 = **358柱面**

**对比**：

| 算法 | 移动总距离 | 特点 |
|------|-----------|------|
| FCFS | 498 | 简单公平，效率最低 |
| **SSTF** | **248** | 最短，但可能饥饿 |
| SCAN | 250 | 类似SSTF但避免饥饿 |
| C-SCAN | 358 | 更公平，但距离长 |

# 题型五：银行家算法（死锁避免）🔥🔥🔥

## 算法背景

银行家算法由Dijkstra提出，用于**死锁避免**。核心思想是：当进程请求资源时，系统通过试探分配并执行安全性检查，判断分配后系统是否仍处于安全状态。只有当分配后仍然安全时，才真正授予资源。

适用场景：每个进程在开始前声明其最大资源需求量。

---

## 数据结构（明确说明每个数组的含义）

| 数据结构 | 维度 | 含义 | 说明 |
|----------|------|------|------|
| **Available[m]** | 长度为m的一维数组 | 每种资源的当前可用数量 | Available[j] = k 表示资源j还剩k个 |
| **Max[n][m]** | n×m矩阵 | 每个进程对每种资源的最大需求 | Max[i][j] = k 表示进程i最多需要k个资源j |
| **Allocation[n][m]** | n×m矩阵 | 每个进程当前已分配的资源数 | Allocation[i][j] = k 表示进程i已分配了k个资源j |
| **Need[n][m]** | n×m矩阵 | 每个进程还需要的资源数 | Need[i][j] = Max[i][j] - Allocation[i][j] |
| **Work[m]** | 长度为m的一维数组 | 安全性算法中的工作向量 | 模拟资源可用数量的变化过程 |
| **Finish[n]** | 长度为n的一维数组 | 完成标志 | Finish[i] = true 表示进程i可安全完成 |

**核心关系**：

```
Need[i][j] = Max[i][j] - Allocation[i][j]

   ┌─────────────────────────────────────────────┐
   │  任何时刻：Allocation[i] + Need[i] = Max[i]  │
   │  所有进程：ΣAllocation[i] + Available = 资源总数 │
   └─────────────────────────────────────────────┘
```

---

## 安全性算法（逐步描述）

```
步骤1: 初始化
       Work = Available                    // 工作向量初始为可用资源
       Finish[i] = false  (i = 0..n-1)     // 所有进程未完成

步骤2: 寻找一个进程i，满足：
       Finish[i] == false
       Need[i] <= Work                     // 逐资源比较，每个分量都≤

步骤3: 如果找到这样的进程i：
       Work = Work + Allocation[i]         // 回收该进程已分配的所有资源
       Finish[i] = true
       回到步骤2                           // 继续寻找下一个

       如果没有找到这样的进程i：
       进入步骤4

步骤4: 如果所有 Finish[i] == true (i = 0..n-1)
       则系统处于安全状态（存在安全序列）
       否则系统处于不安全状态
```

**安全序列**：上述算法找到进程的顺序就是一个安全序列（可能有多个）。

---

## 资源请求算法

```
设进程i发出请求 Request[i]，长度为m的资源向量

步骤1: 检查 Request[i] <= Need[i]
       若任一资源超过 → 报错（进程请求超过其最大声明需求）

步骤2: 检查 Request[i] <= Available
       若任一资源不足 → 进程等待（资源不够）

步骤3: 试探分配
       Available = Available - Request[i]
       Allocation[i] = Allocation[i] + Request[i]
       Need[i] = Need[i] - Request[i]

步骤4: 调用安全性算法检查当前状态
       若安全 → 真正分配（试探分配生效）
       若不安全 → 回滚试探分配，进程等待
```

---

## 例题1：判断安全状态并找安全序列

**题目**：系统有5个进程P0~P4，3类资源A/B/C。
- 资源总数：A=10, B=5, C=7
- 当前Available = (3, 3, 2)
- Max矩阵和Allocation矩阵如下：

**Max矩阵**：

| 进程 | A | B | C |
|------|---|---|---|
| P0 | 7 | 5 | 3 |
| P1 | 3 | 2 | 2 |
| P2 | 9 | 0 | 2 |
| P3 | 2 | 2 | 2 |
| P4 | 4 | 3 | 3 |

**Allocation矩阵**：

| 进程 | A | B | C |
|------|---|---|---|
| P0 | 0 | 1 | 0 |
| P1 | 2 | 0 | 0 |
| P2 | 3 | 0 | 2 |
| P3 | 2 | 1 | 1 |
| P4 | 0 | 0 | 2 |

**问题**：
(1) 计算Need矩阵
(2) 判断当前是否是安全状态
(3) 如果是，给出一个安全序列

---

**解答**：

**(1) 计算Need矩阵**

Need = Max - Allocation

| 进程 | Max | Allocation | Need |
|------|-----|-----------|------|
| P0 | (7,5,3) | (0,1,0) | **(7,4,3)** |
| P1 | (3,2,2) | (2,0,0) | **(1,2,2)** |
| P2 | (9,0,2) | (3,0,2) | **(6,0,0)** |
| P3 | (2,2,2) | (2,1,1) | **(0,1,1)** |
| P4 | (4,3,3) | (0,0,2) | **(4,3,1)** |

**(2)(3) 安全性检查**

初始：Work = Available = (3, 3, 2)，所有Finish = false

```
Work = (3,3,2)

寻找满足 Need[i] <= Work 且 Finish[i]==false 的进程：

  P0: (7,4,3) <= (3,3,2)? No  (7>3, 4>3)
→ P1: (1,2,2) <= (3,3,2)? Yes  (1≤3, 2≤3, 2≤2)
  Work = (3,3,2) + Allocation[1](2,0,0) = (5,3,2)
  Finish[1] = true

  P0: (7,4,3) <= (5,3,2)? No  (7>5)
  P2: (6,0,0) <= (5,3,2)? No  (6>5)
→ P3: (0,1,1) <= (5,3,2)? Yes  (0≤5, 1≤3, 1≤2)
  Work = (5,3,2) + Allocation[3](2,1,1) = (7,4,3)
  Finish[3] = true

  P0: (7,4,3) <= (7,4,3)? Yes  (7≤7, 4≤4, 3≤3)
→ 或P4: (4,3,1) <= (7,4,3)? Yes  (4≤7, 3≤4, 1≤3)
  (两条路径均可，先选P0)
  Work = (7,4,3) + Allocation[0](0,1,0) = (7,5,3)
  Finish[0] = true

  (若之前选P4: Work = (7,4,3)+(0,0,2) = (7,4,5), Finish[4]=true)
  (这里按P0→P4→P2继续)

→ P4: (4,3,1) <= (7,5,3)? Yes  (4≤7, 3≤5, 1≤3)
  Work = (7,5,3) + Allocation[4](0,0,2) = (7,5,5)
  Finish[4] = true

→ P2: (6,0,0) <= (7,5,5)? Yes  (6≤7, 0≤5, 0≤5)
  Work = (7,5,5) + Allocation[2](3,0,2) = (10,5,7)
  Finish[2] = true

全部 Finish = [T, T, T, T, T] → 系统处于安全状态！
```

**安全序列**：P1 → P3 → P0 → P4 → P2

**验证**：最终Work = (10,5,7) = 系统资源总数 ✓

> **注意**：安全序列不唯一。如果Step 3选P4而非P0，可得安全序列 P1→P3→P4→P0→P2。

# 题型六：段页式地址转换 🔥🔥

## 原理回顾

**段页式存储管理** = 分段 + 分页的结合。

- **分段**（用户视角）：按逻辑功能划分（代码段、数据段等），每段是独立的逻辑单元
- **分页**（物理视角）：将内存划分为固定大小的页框，消除外部碎片

**逻辑地址结构**：

```
逻辑地址 = 段号 + 段内页号 + 页内偏移
            ↑         ↑          ↑
         查段表     查页表      块内定位
```

**地址转换流程**：

```
逻辑地址
   │
   ├── 段号 ──→ 查段表 ──→ 段长越界检查 → 段内页表始址
   │                                    ↓
   ├── 页号 ──→ 查页表 ──→ 缺页检查 → 物理块号
   │                                    ↓
   └── 偏移 ──→ 页内偏移 ────────────→ 拼成物理地址
                                                ↓
                                   物理地址 = 块号×页大小 + 偏移
```

---

## 解题公式

**从逻辑地址提取各字段**（按地址格式位运算）：

```
假设:
  N位逻辑地址 = a位段号 + b位页号 + c位偏移  (a+b+c=N)
  页大小 = 2^c 字节

段号     = 逻辑地址 >> (b + c)              // 取高a位
段内页号 = (逻辑地址 >> c) & ((1<<b) - 1)    // 取中间b位
页内偏移 = 逻辑地址 & ((1<<c) - 1)           // 取低c位
```

**等效算术公式**：

```
段内地址  = 逻辑地址 % (2^(b+c))            // 去掉段号部分
段内页号  = 段内地址 / 页大小                 // 取整
页内偏移  = 段内地址 % 页大小                 // 取余

物理地址  = 物理块号 × 页大小 + 页内偏移
```

**地址空间计算**：

| 参数 | 公式 |
|------|------|
| 每段最大页数 | 2^b |
| 每页大小 | 2^c = 页大小 |
| 每段最大长度 | 2^b × 2^c = 2^(b+c) |
| 最大段数 | 2^a |
| 虚拟地址空间 | 2^a × 2^b × 2^c = 2^N |

---

## 例题3：完整段页式转换

**题目**：某段页式系统，逻辑地址16位，划分为段号4位 + 页号4位 + 页内偏移8位，页大小256B。进程段表如下：

| 段号 | 段长度 | 页表始址（物理内存地址） |
|------|--------|------------------------|
| 0 | 12KB | 0x100 |
| 1 | 8KB | 0x200 |
| 2 | 4KB | 0x300 |

各段页表内容（部分）：

| 段号 | 页表始址 | 页0 | 页1 | 页2 | 页3 | ... |
|------|---------|-----|-----|-----|-----|-----|
| 0 | 0x100 | 块5 | 块7 | 块9 | 块12 | ... |
| 1 | 0x200 | 块3 | 块6 | 块8 | 块1 | ... |
| 2 | 0x300 | 块4 | 块10 | 块15 | 块11 | ... |

给定逻辑地址 **0x2345**，求物理地址。

---

**解答**：

**第一步：拆分逻辑地址**

0x2345 转换为二进制（16位）：

```
0x2345 = 0010 0011 0100 0101
        │段号│ │页号│ │偏移│
        0010   0011   01000101
        =2     =3     =69(0x45)
```

即：段号=**2**，段内页号=**3**，页内偏移=**69**

**第二步：查段表**

```
段2长度 = 4KB = 4096B
页大小 = 256B
段2页数 = 4096 / 256 = 16页

页号3 < 16 → 段内地址合法 ✓

段2页表始址 = 0x300
```

**第三步：查页表**

```
段2页表[页号3] = 物理块号 11
```

**第四步：计算物理地址**

```
物理地址 = 物理块号 × 页大小 + 页内偏移
         = 11 × 256 + 69
         = 2816 + 69
         = 2885
         = 0xB45
```

**完整转换流程**：

```
逻辑地址 0x2345 (16位)
   │
   ├─ 高4位=0010=2 ──→ 段表[2]: 页表始址=0x300, 段长=4KB(16页)
   │                            ↓ 页号3 < 16 ✓
   ├─ 中4位=0011=3 ──→ 查页表2[3]: 物理块号=11
   │                            ↓
   └─ 低8位=01000101=69 ──→ 页内偏移=69
                                ↓
                    物理地址 = 11×256+69 = 2885 = 0xB45
```

## 例题4：段页式地址转换

**题目**：某系统逻辑地址32位，划分为段号8位 + 页号12位 + 页内偏移12位，页大小4KB。进程段表：
- 段0：长度2MB，页表始址0x1000
- 段1：长度1MB，页表始址0x2000

段0的页表中第3号页项存物理块号15。

逻辑地址 **0x00003000** 对应的物理地址是多少？

---

**解答**：

**第一步：拆分逻辑地址**

0x00003000 转换为二进制（32位）：

```
0x00003000 = 0000 0000 0000 0000 0011 0000 0000 0000
            │   段号(8位)   │ │  页号(12位)  │ │偏移(12位)│
            0000 0000 = 0     0000 0011 0000 = 3   0000 0000 0000 = 0
```

也可以直接观察十六进制：

```
0x00003000
    ↑↑       ↑↑↑
   段号8位   页号12位   偏移12位
   0x00=0    0x003=3   0x000=0
```

即：段号=**0**，段内页号=**3**，页内偏移=**0**

**第二步：查段表**

```
段0长度 = 2MB = 2×1024×1024 = 2097152B
页大小  = 4KB = 4096B
段0页数 = 2MB / 4KB = 512页

页号3 < 512 → 段内地址合法 ✓

段0页表始址 = 0x1000
```

**第三步：查页表**

```
段0页表[页号3] = 物理块号 15
```

**第四步：计算物理地址**

```
物理地址 = 物理块号 × 页大小 + 页内偏移
         = 15 × 4096 + 0
         = 61440
         = 0xF000
```

**验证**：0xF000 = 15 × 4096 = 61440 ✓

## 练习1

**题目**：接例题3系统（16位地址，段号4位+页号4位+偏移8位，页大小256B，同一套段表和页表），逻辑地址 **0x0345**，求物理地址。

**解答**：

**第一步：拆分逻辑地址**

0x0345 = 0000 0011 0100 0101（16位）

```
  段号(高4位)      页号(中间4位)     偏移(低8位)
  0000 = 0          0011 = 3          01000101 = 0x45 = 69
```

即：段号=**0**，段内页号=**3**，页内偏移=**69**

**第二步：查段表**

```
段0长度 = 12KB = 12288B
页大小  = 256B
段0页数 = 12288 / 256 = 48页

页号3 < 48 → 段内地址合法 ✓

段0页表始址 = 0x100
```

**第三步：查页表**

```
段0页表[页号3] = 物理块号 12
```

**第四步：计算物理地址**

```
物理地址 = 12 × 256 + 69
         = 3072 + 69
         = 3141
         = 0xC45
```

**验证**：0xC45 = 12 × 256 + 4 × 16 + 5 = 3072 + 64 + 5 = 3141 ✓
